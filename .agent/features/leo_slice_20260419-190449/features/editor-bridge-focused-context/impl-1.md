# Impl iteration 1 — F08 editor-bridge-focused-context

## Summary

Added CM6 `focusSnapshotField` + `updateListener` extension factory that tracks cursor/selection on every editor transaction, plus a pure `EditorBridge` orchestrator that registers Obsidian workspace listeners (`active-leaf-change` / `file-open` / `editor-change`) via `registerEvent`, debounces emissions at 300ms through the shared `debounce` util, and pushes assembled `FocusedContext` payloads into a `FocusedContextChannel` in-memory push sink. Concrete Obsidian resolution lives in a separate `WorkspaceFocusProbe` so the bridge core is unit-testable in node. Wired into `LeoPlugin.onload` / `onunload`.

## Files touched

- `src/util/debounce.ts` — shared trailing-debounce util with `cancel` / `flush` / `pending` (AC2, AC7).
- `src/editor/types.ts` — `FocusedContext`, `FocusedContextSink`, `NULL_FOCUSED_CONTEXT`.
- `src/editor/focusedContext.ts` — CM6 `focusSnapshotField` (StateField) + `createFocusedContextExtension` (updateListener wrapper) + `readFocusedContextFromView` (cursor/selection/viewport assembly).
- `src/editor/editorBridge.ts` — framework-agnostic bridge: registers extension, attaches/detaches workspace listeners via `registerEvent`, debounced emission, structured `editor.focus` log line per emission.
- `src/editor/workspaceFocusProbe.ts` — Obsidian-specific `EditorFocusProbe` that resolves the active `MarkdownView`, reads its CM6 `EditorView` (via `editor.cm`), and assembles `FocusedContext`.
- `src/editor/focusedContextChannel.ts` — in-memory push channel (`FocusedContextSink` + `subscribe` / `current`) for downstream consumers (F09 indicator, F10 `AgentController`).
- `src/main.ts` — constructs `FocusedContextChannel` + `WorkspaceFocusProbe` + `EditorBridge`; starts the bridge in `onload` and disposes it in `onunload`.
- `package.json` — added `@codemirror/state@6.5.0` and `@codemirror/view@6.38.6` as devDependencies (types; runtime stays `external` in esbuild, provided by Obsidian).
- `pnpm-lock.yaml` — regenerated by `pnpm install` for the two new devDeps.

## Deviations from feature.md

None. The scope bullet mentioning a `StateField` is satisfied literally by `focusSnapshotField` (state), while emission is triggered from `EditorView.updateListener` per AC1. The spec's "AgentController in-memory push channel" is realized as `FocusedContextChannel` (subscribable sink) — `AgentController` itself arrives in F10 and will subscribe there.

## Assumptions

- Obsidian's `Editor.cm` (CM6 `EditorView`) is accessed via a non-public but widely-used cast (`editor as unknown as { cm?: EditorView }`). This is the standard plugin-community pattern for reaching the underlying CM6 view; architecture §3.4 and code-style §CodeMirror 6 both assume direct CM6 access.
- `workspace.on('editor-change')` fires for the same edit that CM6's `updateListener` handles; we rely on the shared 300ms debounce to collapse both paths into a single emission per window (AC4).
- `performance.now()` is available in both the Obsidian host and the node test environment for the micro-latency budget check (AC6).
