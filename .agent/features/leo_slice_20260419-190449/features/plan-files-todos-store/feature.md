# F23 — Plan files & TodoWrite store

## Purpose

Deliver the Phase-2 foundation that later plan-mode features stack on: an on-disk `PlanStore` writing plan markdown to `<vault>/.leo/plans/<slug>.md` with a lazily-generated two-word kebab slug cached per session and retried up to 10 times on filename collision per [FR-PLAN-06](../../context.md#fr-plan-06), a path-traversal guard plus vault-root-relative resolution of the configurable `plansDirectory` with fallback to the default on violation per [NFR-REL-08](../../context.md#nfr-rel-08), and an in-memory `TodoStore` keyed by `agentId ?? sessionId` exposed to the agent via a `TodoWrite` tool whose schema (`{id, content, status: "pending"|"in-progress"|"completed", priority?, activeForm?}`) and prompt text copied verbatim from plan.md §3.3 satisfy [FR-PLAN-01](../../context.md#fr-plan-01) and [FR-PLAN-02](../../context.md#fr-plan-02). Unit coverage for slug generation, the path-traversal guard, and the default-directory fallback is in scope per [NFR-TEST-07](../../context.md#nfr-test-07). This slice is store-only: mode transitions, approval dialog, session resume, and stale-todo reminders are explicitly later features.

## Scope

### In scope

- `PlanStore` module (agent layer per [architecture §3.2](../../../../architecture/architecture.md#32-agent-layer)) owning `readPlan() / writePlan(content) / currentSlug() / resetSlug()` against `<vault>/<plansDirectory>/<slug>.md`, routing all FS I/O through the [architecture §3.4 VaultAdapter](../../../../architecture/architecture.md#34-adapters) seam per [FR-PLAN-06](../../context.md#fr-plan-06).
- Two-word kebab slug generator — lowercase `a-z`-only dictionary pair joined with `-` — lazily produced on the first plan-file access of a session and cached in memory for the session per [FR-PLAN-06](../../context.md#fr-plan-06); collision detection via `VaultAdapter.exists()` with up to 10 retries generating a fresh pair each time; on the 11th failure the store throws a typed `PlanSlugExhausted` error and logs `plan.slug.collision-exhausted` via the [F01 Logger](../plugin-bootstrap-logging/feature.md).
- Path-traversal guard on every `PlanStore` FS call: a vault-relative `resolvePlanPath(slug)` helper rejects `..`, absolute paths, and any result that, after normalisation, escapes `<vault>/<plansDirectory>/`; violations throw a typed `PlanPathEscape` error and emit `plan.path.traversal-rejected` per [FR-PLAN-06](../../context.md#fr-plan-06), [NFR-REL-08](../../context.md#nfr-rel-08).
- `plansDirectory` setting resolved relative to the vault root; a configured value that escapes the vault, contains `..`, or is absolute is rejected at load time and the store falls back to the default `.leo/plans/`, logging `plan.dir.fallback` with the offending input per [NFR-REL-08](../../context.md#nfr-rel-08).
- `TodoStore` module holding an in-memory `Map<string, Todo[]>` keyed by `agentId ?? sessionId` per [FR-PLAN-01](../../context.md#fr-plan-01); todos are not persisted to disk (resume is [F26](../../features-index.md)'s job); "all-completed" submission stores an empty list but returns the original `newTodos` verbatim so the model sees what it just finished, matching the [plan.md §3 invariant recorded in context.md](../../context.md#fr-plan-01).
- Typed `Todo` schema `{id: string, content: string, status: "pending" | "in-progress" | "completed", priority?: "low" | "medium" | "high", activeForm?: string}` validated by a single [Zod](../../../../standards/code-style.md#zod--tool-schemas) schema shared by the `TodoWrite` tool input and the `TodoStore` surface; `content` and (when present) `activeForm` must be non-empty.
- `TodoWrite` tool registered against [F16 tool-registry-builtin-read](../../features-index.md)'s `ToolRegistry` with the **verbatim description text from plan.md §3.3** (copied into `src/tools/builtin/todoWrite.ts`) per [FR-PLAN-02](../../context.md#fr-plan-02), `requiresConfirmation: false`, input `{newTodos: Todo[]}`, output `ToolResult<{todos: Todo[]}>` returning the submitted `newTodos` verbatim.
- Structured log events (`plan.slug.generated`, `plan.slug.collision`, `plan.slug.collision-exhausted`, `plan.path.traversal-rejected`, `plan.dir.fallback`, `plan.write`, `plan.read`, `todo.write`) through the [F01 Logger](../plugin-bootstrap-logging/feature.md) per [NFR-LOG-04](../../context.md#nfr-log-04); no plan or todo content logged above `debug` per [code-style — Logging](../../../../standards/code-style.md#logging).
- Vitest coverage of: two-word kebab slug shape; collision-retry loop up to 10 times producing distinct slugs; 11th collision surfacing `PlanSlugExhausted`; path-traversal guard rejecting `..`, absolute paths, and symlink-escape-like inputs; `plansDirectory` fallback to default on escaping config; `TodoWrite` Zod happy / reject paths; all-completed submission returns `newTodos` verbatim while internal state becomes empty per [NFR-TEST-07](../../context.md#nfr-test-07).

### Out of scope

- Plan-mode transitions (`EnterPlanMode` / `ExitPlanMode` tools, permission-system flag, write-tool blocking while `mode === 'plan'`, mode-transition attachments, subagent prohibition) per [FR-PLAN-03](../../context.md#fr-plan-03), [FR-PLAN-04](../../context.md#fr-plan-04), [FR-PLAN-05](../../context.md#fr-plan-05), [FR-PLAN-08](../../context.md#fr-plan-08), [NFR-REL-07](../../context.md#nfr-rel-07) → ship with [F24 plan-mode-permissions](../../features-index.md).
- Plan approval dialog (Approve / Edit / Reject, result-message variants, focus-trap, `planWasEdited` disk sync) per [FR-PLAN-07](../../context.md#fr-plan-07), [FR-UI-09](../../context.md#fr-ui-09) → ships with [F25 plan-approval-dialog](../../features-index.md).
- Plan / todo session resume (todo rehydration from transcript; plan-content recovery fallback chain `file_snapshot` → `ExitPlanMode` tool_use → user attachments) per [FR-PLAN-09](../../context.md#fr-plan-09) → ships with [F26 plan-session-resume](../../features-index.md).
- Stale-todo reminder injected at turn boundaries, rate-limited, wrapped in `<system-reminder>` tags per [FR-PLAN-03](../../context.md#fr-plan-03) → ships with [F24 plan-mode-permissions](../../features-index.md).
- Subagent plan-file path variant `.leo/plans/<slug>-agent-<agentId>.md` — treated as vacuous in Phase 2 since Leo has no subagent execution model yet (flagged in [context.md open questions](../../context.md#open-questions)); revisited by F24 if/when subagents land.

## Acceptance criteria

1. `PlanStore.currentSlug()` returns a two-word kebab slug (`^[a-z]+-[a-z]+$`) lazily on first call within a session, caches it for the lifetime of the `PlanStore` instance, and `PlanStore.writePlan(content)` writes the body to `<vault>/<plansDirectory>/<slug>.md` via the `VaultAdapter`; the next `readPlan()` round-trips the content byte-for-byte. ([FR-PLAN-06](../../context.md#fr-plan-06))
2. On slug-generation collision against an existing file, the generator retries with a fresh two-word pair up to 10 times; on the 11th collision it throws a typed `PlanSlugExhausted` error and emits the `plan.slug.collision-exhausted` log event via the [F01 Logger](../plugin-bootstrap-logging/feature.md). ([FR-PLAN-06](../../context.md#fr-plan-06), [NFR-TEST-07](../../context.md#nfr-test-07))
3. Every `PlanStore` FS call runs its argument through `resolvePlanPath(slug)`; inputs containing `..`, starting with `/`, or normalising outside `<vault>/<plansDirectory>/` throw a typed `PlanPathEscape` error, emit `plan.path.traversal-rejected`, and perform no I/O. ([FR-PLAN-06](../../context.md#fr-plan-06), [NFR-REL-08](../../context.md#nfr-rel-08), [NFR-TEST-07](../../context.md#nfr-test-07))
4. `plansDirectory` is resolved relative to vault root at `Plugin.onload`; a configured value that is absolute, contains `..`, or normalises outside the vault is rejected, the store falls back to the default `.leo/plans/`, and `plan.dir.fallback` logs the offending input alongside the default substitution. ([NFR-REL-08](../../context.md#nfr-rel-08), [NFR-TEST-07](../../context.md#nfr-test-07))
5. `TodoStore` keys todos by `agentId ?? sessionId` in an in-memory `Map<string, Todo[]>`; todos are never written to disk; a `TodoWrite` call whose `newTodos` are all `status: "completed"` stores an empty list while the tool returns the original `newTodos` verbatim so the model can observe the completion set. ([FR-PLAN-01](../../context.md#fr-plan-01))
6. The `TodoWrite` tool is registered with [F16 tool-registry-builtin-read](../../features-index.md) using a single Zod schema for `Todo` (`{id, content, status: "pending"|"in-progress"|"completed", priority?, activeForm?}`); the tool `description` is the **verbatim prompt text from plan.md §3.3** and is checked byte-for-byte in a fixture assertion so drift is caught at CI. ([FR-PLAN-01](../../context.md#fr-plan-01), [FR-PLAN-02](../../context.md#fr-plan-02))
7. Vitest suite covers: two-word kebab shape on happy path; collision-retry loop producing 10 distinct slugs then throwing on the 11th; path-traversal guard rejecting `..`, absolute paths, and escaping-after-join inputs; `plansDirectory` default-fallback on escape with `plan.dir.fallback` log; `TodoWrite` Zod happy / reject paths; all-completed submission empties internal state while returning `newTodos` verbatim; verbatim-prompt-text byte-for-byte assertion against plan.md §3.3. ([FR-PLAN-01](../../context.md#fr-plan-01), [FR-PLAN-02](../../context.md#fr-plan-02), [FR-PLAN-06](../../context.md#fr-plan-06), [NFR-REL-08](../../context.md#nfr-rel-08), [NFR-TEST-07](../../context.md#nfr-test-07))

## Implementation notes

- [Architecture §3.2 Agent Layer](../../../../architecture/architecture.md#32-agent-layer) — places `PlanStore` and `TodoStore` alongside `ToolRegistry` / `SkillsStore`; this slice delivers those two agent-layer modules.
- [Architecture §3.4 Adapters — VaultAdapter](../../../../architecture/architecture.md#34-adapters) — all plan-file IO routes through `VaultAdapter`, never `app.vault.adapter` directly, per the Obsidian-plugin-pattern contract.
- [Architecture §4 Key Contracts](../../../../architecture/architecture.md#4-key-contracts) — pins the `ToolSpec` / `ToolResult` types the `TodoWrite` tool implements.
- [Architecture §6 State Ownership](../../../../architecture/architecture.md#6-state-ownership) — plan file on disk at `<vault>/.leo/plans/<slug>.md`; todos in-memory only per [FR-PLAN-01](../../context.md#fr-plan-01).
- [Architecture §7 Error Handling](../../../../architecture/architecture.md#7-error-handling-strategy) — typed `PlanPathEscape` / `PlanSlugExhausted` surface as tool errors and structured log events, never silent.
- [Architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed) — places modules under `src/tools/builtin/todoWrite.ts`, `src/storage/PlanStore.ts`, `src/agent/TodoStore.ts`; fixtures in `tests/unit/`.
- [Architecture §10 Concurrency & Lifecycle Rules](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules) — session-scoped slug cache clears on `Plugin.onunload`; no timers held open.
- [Architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules) — `FR-PLAN-*` attaches to agent-layer + tool-registry modules; this slice binds the store tier.
- [Tech stack — Storage Layout](../../../../standards/tech-stack.md#storage-layout) — fixes the `<vault>/.leo/` tree; plan files live beside `conversations/`, `skills/`, `logs/`.
- [Tech stack — Agent / Tool / Skill / MCP Wiring](../../../../standards/tech-stack.md#agent--tool--skill--mcp-wiring) — pins `@langchain/core/tools` `tool()` + Zod for the `TodoWrite` registration shape.
- [Tech stack — Dependencies — Production](../../../../standards/tech-stack.md#dependencies--production) — names `zod` as the validation dep used by the shared `Todo` schema.
- [Code style — TypeScript](../../../../standards/code-style.md#typescript) — strict mode, named exports only, `readonly` on `Todo` fields, `as const` status union (no `enum`).
- [Code style — Zod & Tool Schemas](../../../../standards/code-style.md#zod--tool-schemas) — one Zod schema per external shape; `z.infer` for TS type; `.describe(...)` on fields (LLM reads them).
- [Code style — Obsidian Plugin Patterns](../../../../standards/code-style.md#obsidian-plugin-patterns) — route FS through `VaultAdapter`; `Notice` for user-surface errors, `Logger` for diagnostics.
- [Code style — Async & Concurrency](../../../../standards/code-style.md#async--concurrency) — collision-retry loop bounded at 10; no bare network; `AbortSignal` plumbed through `TodoWrite.invoke(ctx)` per the `ToolCtx` contract.
- [Code style — Error Handling](../../../../standards/code-style.md#error-handling) — Fail Fast on path escape; typed errors, no swallow.
- [Code style — Logging](../../../../standards/code-style.md#logging) — structured `plan.*` / `todo.*` events, no plan or todo content above `debug`.
- [Code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw) — Vitest unit suite with `VaultAdapter` faked in-memory; no real FS; `fake-indexeddb` unused here.
- [Best practices — Core Principles](../../../../standards/best-practices.md#core-principles) — Single Responsibility: `PlanStore` is I/O + slug; `TodoStore` is in-memory map; `TodoWrite` is tool-registry adapter.
- [Best practices — Testing & Quality Gates](../../../../standards/best-practices.md#testing--quality-gates) — happy plus edge coverage (collision 10→11 boundary; traversal vectors; fallback path) seeded deterministically with an injected slug-word dictionary, no clocks or sleeps.

## Open questions

- **Subagent plan-file variant (`.leo/plans/<slug>-agent-<agentId>.md`) — global [open questions](../../context.md#open-questions) "Plan mode in subagent contexts"**: Leo has no subagent execution model in Phase 2; this feature ships the main-agent path only. Flagged for F24 to revisit once subagent scope is decided.
- **`plansDirectory` outside the vault — global [open questions](../../context.md#open-questions) "`plansDirectory` configurability"**: [NFR-REL-08](../../context.md#nfr-rel-08) mandates vault-root-relative resolution with fallback; this feature picks the strict reading (any escape → fallback to default, no `safeStorage` route). Verifier to confirm the SRS intent allows no out-of-vault opt-in.
- **Two-word kebab dictionary source — plan.md §2.2 is silent on provenance**: this feature assumes a small bundled adjective+noun word-list shipped in `src/tools/builtin/todoWrite.ts`; verifier to confirm no external dep (e.g. `random-words`) is preferred.
- **`plan-file-write` tool name vs whitelisted path — global [open questions](../../context.md#open-questions) "Plan-file writes allowed in plan mode"**: [FR-PLAN-05](../../context.md#fr-plan-05) lists `plan-file-write` in the plan-mode allowlist; this feature exposes `PlanStore.writePlan` as a module-level API rather than a registered tool — whether F24 adds a distinct `WritePlan` `ToolSpec` or whitelists a `create_note`/`edit_note` path restricted to `<plansDirectory>/` is deferred to F24.
