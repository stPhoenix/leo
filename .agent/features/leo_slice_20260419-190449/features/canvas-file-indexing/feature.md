# F36 — Canvas file indexing

## Purpose

Deliver the phase-4 `.canvas` parsing tier that extends the [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) → [F28 chunking-metadata](../chunking-metadata/feature.md) → [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md) pipeline to the Obsidian Canvas format called out in [FR-IDX-05](../../context.md#fr-idx-05) as a phase-4 addition on top of the v1 markdown-only baseline. Canvas files are plain JSON whose top-level `nodes[]` array carries heterogeneous nodes (`type: "text" | "file" | "link" | "group"`); this feature parses that JSON and emits one synthetic [F28](../chunking-metadata/feature.md)-shaped `Chunk` per text-bearing node with metadata `{path, node_id, heading_path: ["canvas"], frontmatter_tags: [], inline_tags: []}`, reusing the [F28](../chunking-metadata/feature.md) inline-tag extractor on `text`-node bodies so `#tag` markers written inside a canvas text node flow into `inline_tags` the same way they do for markdown. The emitted chunks are handed straight to the shared `processPath` seam exposed by [F27](../vault-indexer-dirty-queue/feature.md), so the downstream embed → IndexedDB path in [F29](../embeddings-indexeddb-store/feature.md) is reused byte-for-byte. Markdown-first behaviour from [F27](../vault-indexer-dirty-queue/feature.md) is preserved: this slice *opens up* the existing `TFile.extension !== 'md'` filter to additionally accept `'canvas'` rather than replacing it, keeping PDFs / images / binaries out of the queue.

## Scope

### In scope

- `CanvasChunker` pure module (no IO, no Obsidian singletons bound — all host input passed as arguments) exposing `chunk(input: CanvasChunkerInput): Chunk[]` per [architecture §3.3 Domain / Core](../../../../architecture/architecture.md#33-domain--core-pure) and [architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points); `CanvasChunkerInput = { path: string; source: string }` where `source` is the raw `.canvas` JSON string read by the adapter.
- Canvas file format parsing per the Obsidian `.canvas` JSON schema referenced in [architecture §3.4 Adapters](../../../../architecture/architecture.md#34-adapters): `JSON.parse(source)` then iterate the top-level `nodes[]` array. For each node, inspect `node.type` and extract:
  - `type === "text"`: use `node.text` verbatim as the chunk `text` and run the [F28 inline-tag extractor](../chunking-metadata/feature.md) against the markdown body so `#tag` tokens inside the text node land in `inline_tags` (frontmatter tags are always `[]` — canvas has no frontmatter).
  - `type === "file"`: use `node.file` (the embedded vault-relative target) as the chunk `text` preceded by a `"file: "` prefix so the embedding captures the link target without the adapter needing to re-read the referenced file — cross-file indexing stays owned by [F27](../vault-indexer-dirty-queue/feature.md)'s own enqueue of the referenced `.md`. `inline_tags: []`.
  - `type === "link"`: use `node.url` as the chunk `text` preceded by a `"link: "` prefix. `inline_tags: []`.
  - Nodes with `label` but no primary body (e.g. group nodes with a label) contribute the `label` string as the chunk `text` preceded by a `"label: "` prefix.
- Synthetic chunk metadata shape per [FR-IDX-07](../../context.md#fr-idx-07)-compatible contract: `{path: string, node_id: string, heading_path: readonly string[] = ["canvas"], frontmatter_tags: readonly string[] = [], inline_tags: readonly string[], text: string, line_start: 0, line_end: 0}` — `path` is vault-relative, `node_id = node.id` (the stable UUID-like string Obsidian writes for every canvas node), `heading_path` is the literal `["canvas"]` marker so downstream [F31 rag-cosine-search](../rag-cosine-search/feature.md) and [F35 graphrag-boosts](../graphrag-boosts/feature.md) can recognize canvas rows without a separate type flag, and `line_start`/`line_end` are fixed `0` since canvas nodes have no meaningful line geometry. All arrays `readonly` per [code style — TypeScript](../../../../standards/code-style.md#typescript).
- Non-text / unknown-type node skip: nodes whose `type` is neither `"text"` / `"file"` / `"link"` / group-with-label, or whose body fields are empty after trimming, are silently skipped (never emit an empty-text chunk). A single `indexer.canvas.skip-node` debug-level log entry per skipped node through [F01 Logger](../plugin-bootstrap-logging/feature.md).
- Malformed JSON resilience: a `try / catch` around `JSON.parse(source)` returns `[]` on failure plus a single warning-level `indexer.canvas.parse-error` log event through [F01 Logger](../plugin-bootstrap-logging/feature.md) per [code style — Error Handling](../../../../standards/code-style.md#error-handling); the indexer never throws into [F27](../vault-indexer-dirty-queue/feature.md)'s drain loop. Same degradation path for missing / non-array `nodes` field and for an empty-array `nodes: []`.
- Inline-tag reuse: markdown-node inline-tag extraction is pulled directly from [F28 chunking-metadata](../chunking-metadata/feature.md) via the same exported pure helper — no reimplementation of the `#tag` regex or `#`-strip / dedupe normalization so canvas and markdown tag semantics stay identical.
- Integration point with [F27](../vault-indexer-dirty-queue/feature.md): the existing `processPath(path)` seam is extension-dispatched — when `TFile.extension === 'canvas'`, the adapter reads the raw JSON string via `VaultAdapter.read` and invokes `CanvasChunker.chunk({path, source})` in place of [F28](../chunking-metadata/feature.md)'s markdown chunker; when `TFile.extension === 'md'`, the existing markdown chunker is invoked. The seam dispatch is a two-line switch introduced in this slice; [F27](../vault-indexer-dirty-queue/feature.md)'s markdown-only enqueue filter is relaxed to also accept `'canvas'` per [FR-IDX-05](../../context.md#fr-idx-05) phase-4 addition.
- Deterministic output ordering: chunks emitted in `nodes[]` array order (stable across runs on the same JSON source); same `(path, source)` pair yields byte-identical `Chunk[]` — Vitest snapshot covers this invariant per [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw).
- Vitest unit coverage per [NFR-TEST-01](../../context.md#nfr-test-01) and [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw): valid canvas JSON with mixed `text` / `file` / `link` / `group-with-label` nodes; unknown `type` silently skipped; empty-text / empty-file / empty-link body silently skipped; inline tags extracted from a text node's body; malformed JSON returns `[]` + warning log; `nodes` missing / `nodes: []` returns `[]`; deterministic snapshot on the same input; `heading_path === ["canvas"]` invariant asserted on every emitted chunk.

### Out of scope

- Chunker internals (heading-based segmentation, sliding-window fallback, frontmatter-tag normalization) → [F28 chunking-metadata](../chunking-metadata/feature.md); this feature reuses only the pure inline-tag helper from F28 and implements its own canvas-node walk.
- Embedding the emitted chunks through `EmbeddingClient` and upserting vectors + chunk metadata + `IndexHeader` into IndexedDB per [FR-IDX-08](../../context.md#fr-idx-08), [FR-IDX-09](../../context.md#fr-idx-09) → [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md); this feature ends at the `Chunk[]` return value handed back through the shared `processPath` seam.
- Graph-link edges between canvas nodes (e.g. `edges[]` from→to adjacency mirrored into [F34 graph-cache-symmetric](../graph-cache-symmetric/feature.md)) → not in v1; only `nodes[]` is parsed here. `metadataCache.resolvedLinks` continues to own canvas-to-markdown link edges on the [F34](../graph-cache-symmetric/feature.md) side through Obsidian's own resolution.
- `.canvas` file UI (status-bar counter separation, dedicated icon) → [F30 indexer-ui-controls](../indexer-ui-controls/feature.md) already covers the generic indexer status surface; this slice contributes only the chunker tier and the seam dispatch.
- Non-canvas phase-4+ formats (PDFs, images, per [FR-IDX-05](../../context.md#fr-idx-05) "deferred post-v1") — this slice does not widen the filter beyond `'md' | 'canvas'`.

## Acceptance criteria

1. `.canvas` files enqueued through [F27](../vault-indexer-dirty-queue/feature.md) after the phase-4 filter relaxation are parsed by `CanvasChunker.chunk({path, source})` into one `Chunk` per text-bearing node in `nodes[]` document order; each chunk carries `{path, node_id: node.id, heading_path: ["canvas"], frontmatter_tags: [], inline_tags, text, line_start: 0, line_end: 0}`. ([FR-IDX-05](../../context.md#fr-idx-05))
2. Nodes with `type` outside `{"text", "file", "link"}` and group nodes without a `label` field are silently skipped — no empty-text chunk ever emitted; `CanvasChunker.chunk` on a `nodes: []` array returns `[]`; non-text node-skips log `indexer.canvas.skip-node` at `debug` level only. (feature-specific)
3. Malformed canvas JSON (any `JSON.parse` throw, missing `nodes` key, non-array `nodes`) yields an empty `Chunk[]` result plus exactly one warning-level `indexer.canvas.parse-error` log event through [F01 Logger](../plugin-bootstrap-logging/feature.md); `CanvasChunker.chunk` never throws into the [F27](../vault-indexer-dirty-queue/feature.md) drain loop. (feature-specific)
4. Inline tags inside a canvas `type: "text"` node's body are extracted via the same pure helper [F28 chunking-metadata](../chunking-metadata/feature.md) uses (leading `#` stripped, trimmed, deduped first-seen order); frontmatter tags remain `[]` for every canvas chunk (canvas format has no frontmatter). ([FR-IDX-05](../../context.md#fr-idx-05))
5. Resulting `Chunk[]` flows through the same [F27](../vault-indexer-dirty-queue/feature.md) `processPath` seam and is embedded + stored by [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md) with no special-casing in the downstream pipeline — an integration Vitest asserts that a `.canvas` file's chunks land in the same IndexedDB `vectors` store keyed on `{path, line_start: 0, line_end: 0}` (disambiguated from other nodes of the same file by `node_id`). (feature-specific)
6. `CanvasChunker.chunk` is a pure function — zero IO, zero Obsidian-singleton access, same input yields byte-identical `Chunk[]` (Vitest snapshot asserts determinism); `heading_path === ["canvas"]` is a byte-identical-array invariant on every emitted chunk. ([NFR-TEST-01](../../context.md#nfr-test-01))
7. Vitest suite covers: mixed `text` / `file` / `link` / group-with-label nodes; unknown-type skip; empty-body skip; inline-tag extraction from a text node; malformed JSON → `[]` + warning log; missing / empty `nodes` → `[]`; deterministic snapshot; executable under `vitest run` with no vault or network fixture. ([NFR-TEST-01](../../context.md#nfr-test-01))

## Dependencies

- [F28 chunking-metadata](../chunking-metadata/feature.md) — reuses the pure inline-tag extraction helper and the canonical `Chunk` shape; this feature adds a *sibling* canvas-chunker module behind the same `processPath` seam rather than extending F28's markdown-only chunker.
- [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) — relaxes its markdown-only `enqueueDirty` filter to additionally accept `TFile.extension === 'canvas'` and dispatches the `.canvas` branch of the `processPath(path)` seam to `CanvasChunker` (the `.md` branch still calls [F28](../chunking-metadata/feature.md)).
- [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md) — unchanged; canvas chunks flow through the same embed + IndexedDB upsert path with `{path, node_id, line_start: 0, line_end: 0}` keying.
- Drives requirement [FR-IDX-05](../../context.md#fr-idx-05) (phase-4 addition); touches glossary entries [context.md Glossary — Dirty queue](../../context.md#glossary) (canvas paths now flow through it) and [features-index.md](../../features-index.md) note that F36 is an independent phase-4 vertical slice although FR-IDX-05's v1 baseline is already covered by [F27](../vault-indexer-dirty-queue/feature.md)'s markdown-only filter.

## Implementation notes

- [Architecture §3.3 Domain / Core](../../../../architecture/architecture.md#33-domain--core-pure) — `CanvasChunker` is a pure module; `CanvasChunkerInput → Chunk[]` with zero IO, identical discipline to [F28](../chunking-metadata/feature.md).
- [Architecture §3.4 Adapters](../../../../architecture/architecture.md#34-adapters) — raw `.canvas` JSON string is read via `VaultAdapter.read`; this feature never touches `app.vault` directly.
- [Architecture §5.4 Lazy Indexing](../../../../architecture/architecture.md#54-lazy-indexing) — extends the `processPath → chunk → embed → upsert` row at the `chunk` step only for canvas inputs.
- [Architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points) — a second `Chunker` implementation behind the same seam; swappable without touching [F27](../vault-indexer-dirty-queue/feature.md) / [F29](../embeddings-indexeddb-store/feature.md).
- [Architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed) — module at `src/indexer/CanvasChunker.ts`; test at `tests/unit/canvas-chunker.test.ts`.
- [Architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules) — binds the phase-4 slice of `FR-IDX-05` to `CanvasChunker` alongside F27's v1 filter.
- [Tech stack — Platform APIs](../../../../standards/tech-stack.md#platform-apis) — reads `TFile.extension === 'canvas'` through Obsidian's `TFile` API surface exposed by [F27](../vault-indexer-dirty-queue/feature.md).
- [Code style — TypeScript](../../../../standards/code-style.md#typescript) — strict, `readonly` arrays, discriminated union on `node.type`, no `any`.
- [Code style — Error Handling](../../../../standards/code-style.md#error-handling) — `JSON.parse` wrapped in typed `try/catch`; malformed input short-circuits to `[]` with a single warning log.
- [Code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw) — unit-only coverage; zero vault / network fixtures.
- [Code style — Imports & Module Boundaries](../../../../standards/code-style.md#imports--module-boundaries) — core module imports no adapter or UI code; only the shared inline-tag helper from [F28](../chunking-metadata/feature.md).
- [Code style — Logging](../../../../standards/code-style.md#logging) — `indexer.canvas.parse-error` at `warn`, `indexer.canvas.skip-node` at `debug`; no node-body content above `debug`.
- [Best practices — Core Principles](../../../../standards/best-practices.md#core-principles) — Single Responsibility: canvas-JSON → `Chunk[]` only; no embedding, persistence, or orchestration.

## Open questions

- **Group-node body handling** — Obsidian canvas group nodes can carry both a `label` and child-node IDs; v1 of this slice emits a single `label:`-prefixed chunk per group but never unrolls children separately (children are already emitted as their own `nodes[]` entries). Verifier to confirm the one-group-one-label rule is correct.
- **`link` / `file` prefix strings (`"link: "` / `"file: "` / `"label: "`)** — Prefixes are introduced here so the embedded vector has a textual cue of the node kind; the SRS does not pin wording. Proposal: keep lowercased `"<kind>: "` prefixes; verifier to confirm or adjust.
- **`line_start`/`line_end` fixed to `0`** — The canonical [FR-IDX-07](../../context.md#fr-idx-07) shape requires integer line fields; canvas nodes have no line geometry. Proposal: both fields `0` and downstream uses `node_id` for disambiguation. Verifier to confirm that [F29](../embeddings-indexeddb-store/feature.md) composite-key `{path}#{line_start}-{line_end}` accepts the dup-prefix collision resolved by also keying on `node_id`.
