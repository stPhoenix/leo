# F27 — VaultIndexer dirty queue & header

## Purpose

Deliver the lazy, resumable VaultIndexer spine every later RAG / GraphRAG / canvas feature plugs into: on `Plugin.onload` it loads the Index Header `{model, dim, version}` and, on mismatch against settings, prompts the user "reindex now / later / revert model" per [FR-IDX-01](../../context.md#fr-idx-01); diffs the current vault state against the persisted header-side file manifest to compute the initial dirty set per [FR-IDX-02](../../context.md#fr-idx-02); installs `vault.on('create')` / `vault.on('modify')` / `vault.on('delete')` / `vault.on('rename')` listeners that push affected paths into a persisted `DirtyQueue` per [FR-IDX-03](../../context.md#fr-idx-03); drains that queue lazily via an idle timer (default 30s) or synchronously on-demand when a RAG query fires, never both at once per [FR-IDX-04](../../context.md#fr-idx-04); filters non-markdown inputs up-front so canvas / PDF / image / binary paths never enter the queue per [FR-IDX-05](../../context.md#fr-idx-05); yields to the main thread via `requestIdleCallback`-chunked loops so the editor is never blocked per [FR-IDX-12](../../context.md#fr-idx-12) and [NFR-PERF-02](../../context.md#nfr-perf-02); and mirrors the queue to `<vault>/.leo/index/queue.json` so the initial index is resumable across Obsidian restarts per [NFR-PERF-04](../../context.md#nfr-perf-04). It is the orchestrator-only tier — chunking ([F28](../../features-index.md)), embedding + IndexedDB persistence ([F29](../../features-index.md)), UI ([F30](../../features-index.md)), exclude list ([F32](../../features-index.md)), graph cache ([F34](../../features-index.md)), and canvas parsing ([F36](../../features-index.md)) are explicitly downstream.

## Scope

### In scope

- `VaultIndexer` module (adapter layer per [architecture §3.4](../../../../architecture/architecture.md#34-adapters)) owning `init()` / `enqueueDirty(path)` / `processDueWork(signal)` / `queryOnDemand(signal)` / `shutdown()` plus the in-memory `model / dim / version` triple loaded from `IndexHeader` at startup per [architecture §6 State Ownership — Index Header](../../../../architecture/architecture.md#6-state-ownership).
- `IndexHeader` read path at `Plugin.onload`: read `<vault>/.leo/index/header.json` debug mirror per [tech stack — Storage Layout](../../../../standards/tech-stack.md#storage-layout); if the `{model, dim, version}` triple does not match the configured provider `embedding model` / `dim` / schema version, surface a blocking `Notice` with three inline choices — **now** (triggers a full reindex by marking every markdown file dirty), **later** (defers; chat and RAG queries short-circuit until resolved), **revert model** (writes the old `{model, dim}` back into settings and leaves the index untouched) — per [FR-IDX-01](../../context.md#fr-idx-01).
- Initial vault-diff sweep on `init()`: enumerate every `app.vault.getMarkdownFiles()` path, compare against the manifest inside `IndexHeader` (path + `mtime` + size), and push any new / modified / removed path into the `DirtyQueue`; dispatched through [F01 Logger](../plugin-bootstrap-logging/feature.md) as a single `indexer.diff.complete` structured event per [FR-IDX-02](../../context.md#fr-idx-02), [NFR-LOG-04](../../context.md#nfr-log-04).
- `DirtyQueue` listener registration via `this.registerEvent(app.vault.on('create' | 'modify' | 'delete' | 'rename', ...))` per [architecture §3.4 VaultAdapter](../../../../architecture/architecture.md#34-adapters) and [code style — Obsidian Plugin Patterns](../../../../standards/code-style.md#obsidian-plugin-patterns); `rename` emits a delete-of-old-path + create-of-new-path pair so downstream stores can drop stale vectors per [FR-IDX-03](../../context.md#fr-idx-03). Listeners are auto-disposed on `Plugin.onunload`.
- Lazy drain policy per [FR-IDX-04](../../context.md#fr-idx-04): an idle timer with configurable `indexerIdleMs` (default 30000) calls `processDueWork(signal)` when the queue is non-empty; a synchronous `queryOnDemand(signal)` path is exposed for RAG query callers so a query triggers an immediate drain of any pending dirty entries before searching. Mutual exclusion is enforced — a single `AbortController` + in-flight flag guarantees at most one drain active per [code style — Async & Concurrency](../../../../standards/code-style.md#async--concurrency) and [architecture §10](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules).
- Markdown-only filter: `enqueueDirty(path)` early-returns unless `path.endsWith('.md')` (and is not a `.canvas` / binary file per `TFile.extension`) per [FR-IDX-05](../../context.md#fr-idx-05); canvas is explicitly deferred to [F36 canvas-file-indexing](../../features-index.md).
- Off-main-thread yielding inside `processDueWork`: iterate paths in a `requestIdleCallback`-scheduled loop with a soft `timeRemaining()` budget; after each path the loop re-schedules itself via `requestIdleCallback` so the editor stays responsive — a pure-function `chunkIteration(paths, budget)` util is unit-tested. `requestIdleCallback` is polyfilled with `setTimeout(fn, 1)` for tests per [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw). Implements [FR-IDX-12](../../context.md#fr-idx-12) and [NFR-PERF-02](../../context.md#nfr-perf-02).
- `DirtyQueue` persistence at `<vault>/.leo/index/queue.json` written through `VaultAdapter` per [architecture §3.4](../../../../architecture/architecture.md#34-adapters): set-backed dedupe, atomic temp-file rename write on every enqueue / drain-tick (debounced via the shared util per [code style — Async & Concurrency](../../../../standards/code-style.md#async--concurrency)), loaded on `init()` before the diff sweep so a partial first-index survives `Plugin.onunload` or Obsidian restart per [NFR-PERF-04](../../context.md#nfr-perf-04).
- Seams for downstream features: `processPath(path)` is a single injectable callback returning `Promise<void>`; [F28 chunking-metadata](../../features-index.md) + [F29 embeddings-indexeddb-store](../../features-index.md) wire their pipeline behind it, and [F32 rag-exclude-list](../../features-index.md) wraps it with a glob filter.
- Structured log events `indexer.header.match` / `indexer.header.mismatch` / `indexer.header.user-choice` / `indexer.diff.complete` / `indexer.enqueue` / `indexer.queue.persisted` / `indexer.drain.start` / `indexer.drain.tick` / `indexer.drain.complete` / `indexer.skip.non-markdown` through the [F01 Logger](../plugin-bootstrap-logging/feature.md) per [NFR-LOG-04](../../context.md#nfr-log-04); no file content logged above `debug` per [code style — Logging](../../../../standards/code-style.md#logging).
- Vitest coverage for: header match vs mismatch with each of the three user choices routed; diff sweep against a faked `VaultAdapter` manifest (new / modified / deleted / renamed); listener fan-out including rename→delete+create pair; markdown-only filter rejecting `.canvas` / `.pdf` / `.png`; lazy idle drain under `vi.useFakeTimers`; on-demand drain pre-empting the idle timer; mutual exclusion (no concurrent drains); `requestIdleCallback` yielding budget respected; `queue.json` persistence + reload across simulated `init()` reruns; abort during drain releases in `finally`.

### Out of scope

- Heading-based / fixed-size chunking + chunk metadata `{path, line_start, line_end, heading_path, frontmatter_tags, inline_tags}` per [FR-IDX-06](../../context.md#fr-idx-06), [FR-IDX-07](../../context.md#fr-idx-07) → ships with [F28 chunking-metadata](../../features-index.md).
- Embedding via `EmbeddingClient` + IndexedDB upsert of vectors / chunks / `IndexHeader` persistence + corruption-rebuild prompt per [FR-IDX-08](../../context.md#fr-idx-08), [FR-IDX-09](../../context.md#fr-idx-09), [NFR-DATA-02](../../context.md#nfr-data-02), [NFR-REL-03](../../context.md#nfr-rel-03) → ships with [F29 embeddings-indexeddb-store](../../features-index.md). This feature only reads the header mirror and writes the queue mirror.
- Indexer UI: status-bar progress widget, "Leo: Re-index vault" palette command, reindex-on-model-switch visible settings toggle, no-index "Index vault" empty-state CTA per [FR-IDX-13](../../context.md#fr-idx-13), [FR-IDX-14](../../context.md#fr-idx-14) → ships with [F30 indexer-ui-controls](../../features-index.md); this slice exposes only a blocking `Notice` for header mismatch.
- Exclude list (glob patterns applied to RAG results and indexing work) per [FR-RAG-08](../../context.md#fr-rag-08) → ships with [F32 rag-exclude-list](../../features-index.md); markdown-only is the only filter here.
- Symmetric graph cache built from `metadataCache.resolvedLinks` + `metadataCache.on('resolved')` incremental updates per [FR-IDX-10](../../context.md#fr-idx-10), [FR-IDX-11](../../context.md#fr-idx-11) → ships with [F34 graph-cache-symmetric](../../features-index.md).
- `.canvas` JSON node parsing per [architecture §3.4](../../../../architecture/architecture.md#34-adapters) → ships with [F36 canvas-file-indexing](../../features-index.md); canvas paths are rejected by the markdown-only filter in this slice.

## Acceptance criteria

1. On `Plugin.onload`, `VaultIndexer.init()` reads `<vault>/.leo/index/header.json` and compares `{model, dim, version}` to the configured provider embedding model / dim / schema version; on match it logs `indexer.header.match` and proceeds; on mismatch it surfaces a blocking `Notice` offering **now / later / revert model**, routes the user choice through [F01 Logger](../plugin-bootstrap-logging/feature.md) as `indexer.header.user-choice`, and for **now** marks every `app.vault.getMarkdownFiles()` path dirty. ([FR-IDX-01](../../context.md#fr-idx-01))
2. After header resolution, `init()` runs a single vault-diff sweep against the manifest in `IndexHeader` (`path` + `mtime` + `size`), pushing every new / modified / removed markdown path into the `DirtyQueue` in a single pass, then emits `indexer.diff.complete` with `{added, modified, removed}` counts only (no paths above `debug`). ([FR-IDX-02](../../context.md#fr-idx-02))
3. `VaultIndexer` registers `vault.on('create' | 'modify' | 'delete' | 'rename')` through `Plugin.registerEvent` so listeners auto-dispose on `onunload`; each event funnels into `enqueueDirty(path)`, and `rename` emits a delete-of-`oldPath` + create-of-`newPath` pair so downstream stores can drop stale entries. ([FR-IDX-03](../../context.md#fr-idx-03))
4. `enqueueDirty(path)` rejects any path whose `TFile.extension !== 'md'` (explicitly including `.canvas`, `.pdf`, `.png`, `.jpg`, binaries), logs `indexer.skip.non-markdown` with `{extension}`, and never writes such paths to `queue.json`. ([FR-IDX-05](../../context.md#fr-idx-05))
5. Drain runs lazily: an idle timer (default 30 000 ms, `indexerIdleMs` configurable) fires `processDueWork(signal)` when the queue is non-empty; a RAG query caller invoking `queryOnDemand(signal)` pre-empts the timer and drains synchronously before returning; a single in-flight flag + `AbortController` guarantees at most one drain active at a time, and aborts release the flag in `finally`. ([FR-IDX-04](../../context.md#fr-idx-04))
6. Inside `processDueWork`, work iterates via `requestIdleCallback`-scheduled ticks: each tick processes paths until `deadline.timeRemaining() < 5ms` or the queue empties, then re-schedules; the pure-function `chunkIteration(paths, budget)` util is covered by Vitest, and integration tests assert the editor's `requestAnimationFrame` callbacks are never starved for > 16 ms. ([FR-IDX-12](../../context.md#fr-idx-12), [NFR-PERF-02](../../context.md#nfr-perf-02))
7. `DirtyQueue` mirrors to `<vault>/.leo/index/queue.json` (atomic temp-rename write, debounced) on every enqueue and drain-tick; on `init()`, the queue is rehydrated from that file **before** the vault-diff sweep runs, so a partial first-index survives `Plugin.onunload` / Obsidian restart and the union of (persisted queue ∪ diff result) is the starting dirty set. ([NFR-PERF-04](../../context.md#nfr-perf-04))
8. Vitest suite (using in-memory `VaultAdapter` / `IndexHeader` / `Logger` fakes, `fake-indexeddb`, `vi.useFakeTimers`) covers: header match / mismatch paths with all three user choices; diff sweep across added / modified / removed / renamed; listener fan-out including rename pair; markdown-only rejection; idle-timer drain; on-demand pre-emption; mutual exclusion under concurrent callers; `requestIdleCallback` yielding budget; `queue.json` persistence + reload across simulated restarts; abort-during-drain releases state in `finally`. ([NFR-PERF-02](../../context.md#nfr-perf-02), [NFR-PERF-04](../../context.md#nfr-perf-04))

- [Architecture §5.1 Plugin Startup](../../../../architecture/architecture.md#51-plugin-startup) — shows `Plug->>VI: init, verify Index Header` and `VI-->>Plug: mismatch? notify user` — this feature realises that row.
- [Architecture §5.4 Lazy Indexing](../../../../architecture/architecture.md#54-lazy-indexing) — pins the `Vault→VaultIndexer→DirtyQueue→(idle 30s OR on-demand)→flush` flow implemented here, minus the chunk/embed/upsert arrows (deferred to F28/F29).
- [Architecture §6 State Ownership](../../../../architecture/architecture.md#6-state-ownership) — fixes `Index Header` ownership on `VaultIndexer` with the disk mirror at `.leo/index/header.json` this feature reads.
- [Architecture §7 Error Handling Strategy](../../../../architecture/architecture.md#7-error-handling-strategy) — LM Studio unreachable pauses indexing; this feature honors the `ProviderManager` readiness flag when scheduling drain ticks.
- [Architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points) — `Chunker` / `VectorStore` interfaces swap behind the `processPath` seam exposed here.
- [Architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed) — modules at `src/indexer/VaultIndexer.ts`, `src/indexer/DirtyQueue.ts`, `src/indexer/IndexHeader.ts`; tests at `tests/unit/dirtyQueue.test.ts`.
- [Architecture §10 Concurrency & Lifecycle Rules](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules) — "Indexer runs off-path: own DirtyQueue + idle loop. Never shares a worker with the chat path." is the invariant realised here.
- [Architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules) — binds `FR-IDX-*` to `VaultIndexer` / `DirtyQueue` / `IndexHeader`.

- [Tech stack — Storage Layout](../../../../standards/tech-stack.md#storage-layout) — pins `<vault>/.leo/index/header.json` (and sibling `queue.json` added here) as the on-disk debug + resume mirror.

## Open questions

- **Reindex-on-model-switch surface (FR-IDX-01 vs FR-IDX-13 / FR-IDX-14)** — this slice handles the startup header-mismatch path with a blocking `Notice`; a user changing the embedding model mid-session from the settings tab should also trigger the same prompt, but the visible settings-side UI belongs to [F30 indexer-ui-controls](../../features-index.md). Verifier to confirm the split.
- **`queue.json` schema forward-compatibility** — this feature introduces `<vault>/.leo/index/queue.json` which the SRS does not name explicitly (only `header.json` appears in [tech stack — Storage Layout](../../../../standards/tech-stack.md#storage-layout)). Proposal: `{ version: 1, paths: string[] }` with version-gated reads; verifier to confirm or rename.
- **On-demand drain latency budget (FR-IDX-04 vs NFR-PERF-03)** — synchronous drain before a RAG query could push end-to-end latency above the 200 ms budget [NFR-PERF-03](../../context.md#nfr-perf-03) on large dirty sets. Proposal: cap synchronous drain to `min(queueLength, 32)` entries, defer the tail to the idle path; verifier to pin a number.
- **`requestIdleCallback` availability in Electron renderer** — Obsidian's renderer supports `requestIdleCallback`; the SRS does not fix a fallback for environments lacking it. This feature polyfills via `setTimeout(fn, 1)` for tests and falls back silently at runtime if `typeof requestIdleCallback === 'undefined'`; verifier to confirm fallback is acceptable per [FR-IDX-12](../../context.md#fr-idx-12).
- **Exclude-list interplay with persisted `queue.json` ([FR-RAG-08](../../context.md#fr-rag-08) global [open questions](../../context.md#open-questions))** — if the exclude-list filter ships in [F32](../../features-index.md) after this slice, a previously-persisted queue may contain now-excluded paths. Proposal: F32 drains with filter on load; flagged for F32 detail.
