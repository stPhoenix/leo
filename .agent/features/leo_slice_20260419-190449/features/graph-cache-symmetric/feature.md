# F34 ‚Äî Symmetric graph cache

## Purpose

Deliver the `GraphCache` adapter every later GraphRAG feature plugs into: a symmetric in-memory adjacency map built once from Obsidian's [`app.metadataCache.resolvedLinks`](../../../../standards/tech-stack.md#platform-apis) ‚Äî merged forward-plus-back so every edge `a‚Üíb` implies `b‚Üía` ‚Äî and kept fresh via the [`metadataCache.on('resolved')`](../../../../standards/tech-stack.md#platform-apis) event hook so create / modify / rename / delete round-trips in the vault propagate into the adjacency without a full rebuild per [FR-IDX-10](../../context.md#fr-idx-10) and [FR-IDX-11](../../context.md#fr-idx-11). The cache exposes a pure read surface (`neighbors(path): ReadonlySet<string>`, `has(path): boolean`, `size(): number`) consumed by [F35 graphrag-boosts](../../features-index.md) for 1-hop / 2-hop neighbor lookup and eventually by context-assembly callers; it owns adjacency state only ‚Äî no scoring math ([F35](../../features-index.md)), no cosine search ([F31](../rag-cosine-search/feature.md)), no canvas parsing ([F36](../../features-index.md)), no UI. It sits in the adapter tier per [architecture ¬ß3.4 Adapters ‚Äî GraphCache](../../../../architecture/architecture.md#34-adapters) and [architecture ¬ß6 State Ownership ‚Äî Graph adjacency](../../../../architecture/architecture.md#6-state-ownership), rebuildable from `metadataCache` on demand because state is purely in-memory.

## Scope

### In scope

- `GraphCache` adapter module mounted at `src/graph/GraphCache.ts` per [architecture ¬ß9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed); owns a single `Map<string, Set<string>>` undirected adjacency keyed by vault-relative markdown path per [architecture ¬ß3.4 Adapters ‚Äî GraphCache](../../../../architecture/architecture.md#34-adapters) and [architecture ¬ß6 State Ownership](../../../../architecture/architecture.md#6-state-ownership).
- Initial build on `init()` from `app.metadataCache.resolvedLinks` per [tech stack ‚Äî Retrieval Layer](../../../../standards/tech-stack.md#retrieval-layer): iterate the outer `source ‚Üí { target ‚Üí count }` shape and, for every edge, upsert **both** `source‚Üítarget` **and** `target‚Üísource` into the adjacency so the stored graph is symmetric (forward plus back merged) regardless of which side Obsidian resolved per [FR-IDX-10](../../context.md#fr-idx-10).
- Read surface: pure `neighbors(path): ReadonlySet<string>` returning an empty frozen set on miss, `has(path): boolean`, `size(): number`, `snapshot(): ReadonlyMap<string, ReadonlySet<string>>` for test fixtures per [architecture ¬ß3.3 Domain / Core (pure)](../../../../architecture/architecture.md#33-domain--core-pure); never returns a mutable reference (all reads returned via `ReadonlySet` / `ReadonlyMap` views per [code style ‚Äî TypeScript](../../../../standards/code-style.md#typescript)).
- Incremental update on `this.registerEvent(app.metadataCache.on('resolved', ...))` per [code style ‚Äî Obsidian Plugin Patterns](../../../../standards/code-style.md#obsidian-plugin-patterns) and [architecture ¬ß3.4 Adapters](../../../../architecture/architecture.md#34-adapters): `resolved` fires after Obsidian has recomputed links for any subset of files; the handler rebuilds adjacency for the affected paths by (a) subtracting the old `source‚Üí*` and reciprocal `*‚Üísource` edges for every path whose `resolvedLinks` changed, then (b) inserting the new symmetric edges from the fresh `resolvedLinks` slice per [FR-IDX-11](../../context.md#fr-idx-11).
- File lifecycle coverage via the same `resolved` hook: `create` / `modify` / `rename` / `delete` all trigger Obsidian to re-emit `resolved` once metadata converges; this feature registers exactly one `resolved` listener and uses the event's current `resolvedLinks` snapshot as the source of truth ‚Äî it does not separately subscribe to `vault.on('create'|'modify'|'rename'|'delete')`, following [tech stack ‚Äî Retrieval Layer](../../../../standards/tech-stack.md#retrieval-layer) which pins `metadataCache` as the single graph source per [FR-IDX-11](../../context.md#fr-idx-11).
- Rename handling inside the `resolved` diff: when a path disappears from every source's link set and a new path appears pointing to the same targets, both paths route through the same subtract-then-insert loop; the cache never carries a stale node because every edge touching the old path is dropped when its reciprocal slot's `resolvedLinks` is recomputed per [FR-IDX-11](../../context.md#fr-idx-11).
- Empty-adjacency invariant: a node with zero neighbors after update has its `Set` removed from the `Map` (not left as an empty set) so `has(path)` returns `false` for orphaned nodes; `neighbors(path)` returns a shared frozen empty set singleton for misses to avoid allocation on the hot path per [code style ‚Äî Async & Concurrency](../../../../standards/code-style.md#async--concurrency).
- Listener auto-disposal via `Plugin.registerEvent` so `onunload` releases the `resolved` hook per [code style ‚Äî Obsidian Plugin Patterns](../../../../standards/code-style.md#obsidian-plugin-patterns) and [architecture ¬ß10 Concurrency & Lifecycle Rules](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules); `shutdown()` clears the internal `Map` so a warm plugin reload rebuilds from scratch on the next `init()`.
- Idempotence: running `init()` twice yields the identical adjacency snapshot; replaying the same `resolved` payload is a no-op on a freshly-built cache per [best practices ‚Äî Core Principles](../../../../standards/best-practices.md#core-principles).
- Structured log events through [F01 Logger](../plugin-bootstrap-logging/feature.md) per [NFR-LOG-04](../../context.md#nfr-log-04): `graph.build.complete` (`{nodeCount, edgeCount}`), `graph.resolved.tick` (`{pathsTouched, edgesAdded, edgesRemoved}`), `graph.shutdown` (`{nodeCount}`); no `path` fields above `debug` per [code style ‚Äî Logging](../../../../standards/code-style.md#logging).
- Vitest unit coverage per [code style ‚Äî Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw): symmetry (forward-only `resolvedLinks` fixture produces symmetric adjacency); idempotent `init()`; `resolved` incremental diff (edges added / removed / unchanged); rename path (old node drops when reciprocal sources update); delete path (node removed when it has no neighbors); empty-adjacency invariant (miss returns shared frozen empty set, same reference across calls); `snapshot()` returns a deep read-only view; `shutdown()` clears the map and unsubscribes the listener.

### Out of scope

- 1-hop / 2-hop / tag-shared boost scoring math (1.5√ó / 1.2√ó / 1.1√ó additive) over RAG results per [FR-RAG-02](../../context.md#fr-rag-02) / [FR-RAG-03](../../context.md#fr-rag-03) / [FR-RAG-04](../../context.md#fr-rag-04) ‚Üí ships with [F35 graphrag-boosts](../../features-index.md); this feature exposes only the adjacency read surface those boosts traverse.
- Cosine similarity search, top-K min-heap, same-file overlap merge, and `RAGEngine.query` plumbing per [FR-RAG-01](../../context.md#fr-rag-01) / [FR-RAG-06](../../context.md#fr-rag-06) / [FR-RAG-07](../../context.md#fr-rag-07) ‚Üí owned by [F31 rag-cosine-search](../rag-cosine-search/feature.md); this feature never invokes `Scorer.cosine` or `VectorStore.getAll`.
- `.canvas` JSON node parsing and any canvas-sourced edges in the adjacency per [FR-IDX-05](../../context.md#fr-idx-05) ‚Üí owned by [F36 canvas-file-indexing](../../features-index.md); canvas targets that appear in `resolvedLinks` are stored verbatim as path strings but never inspected here.

## Acceptance criteria

1. On `GraphCache.init()`, the adapter reads `app.metadataCache.resolvedLinks` once and populates an in-memory `Map<string, Set<string>>` where, for every `source ‚Üí { target ‚Üí count }` entry, both `source‚Üítarget` and `target‚Üísource` are inserted so the resulting adjacency is symmetric (forward-plus-back merged) regardless of which side Obsidian resolved. ([FR-IDX-10](../../context.md#fr-idx-10))
2. `neighbors(path): ReadonlySet<string>` returns the stored adjacency set for `path` or a shared frozen empty set on miss; `has(path): boolean` returns `false` for orphaned / unknown paths; `size(): number` returns the count of nodes with at least one neighbor; `snapshot(): ReadonlyMap<string, ReadonlySet<string>>` returns a deep read-only view suitable for test assertions. ([FR-IDX-10](../../context.md#fr-idx-10))
3. A single `metadataCache.on('resolved')` listener is registered via `Plugin.registerEvent` at `init()` and auto-disposes on `Plugin.onunload` per [architecture ¬ß10](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules); on each `resolved` tick the handler subtracts stale reciprocal edges for every path whose link set changed and inserts the fresh symmetric edges from the current `resolvedLinks` snapshot. ([FR-IDX-11](../../context.md#fr-idx-11))
4. `create` / `modify` / `rename` / `delete` round-trips in the vault propagate into the adjacency through the same `resolved` hook (no separate `vault.on(*)` subscription): after a rename, neither the old path nor stale reciprocal edges remain; after a delete, the node is removed from the `Map` once it has zero neighbors; the Vitest fixture exercises all four lifecycle paths against a faked `metadataCache`. ([FR-IDX-11](../../context.md#fr-idx-11))
5. Idempotence: calling `init()` twice produces a byte-identical `snapshot()`; replaying the same `resolved` payload on a fresh cache is a no-op (`edgesAdded === 0 && edgesRemoved === 0` in the `graph.resolved.tick` log event). ([FR-IDX-10](../../context.md#fr-idx-10), [FR-IDX-11](../../context.md#fr-idx-11))
6. Structured log events `graph.build.complete` / `graph.resolved.tick` / `graph.shutdown` emit through the [F01 Logger](../plugin-bootstrap-logging/feature.md) with counts only (`nodeCount`, `edgeCount`, `pathsTouched`, `edgesAdded`, `edgesRemoved`); no `path` payload fields above `debug` per [NFR-LOG-04](../../context.md#nfr-log-04) and [code style ‚Äî Logging](../../../../standards/code-style.md#logging). ([FR-IDX-11](../../context.md#fr-idx-11))
7. `shutdown()` clears the internal `Map`, unsubscribes the `resolved` listener via the `Plugin.registerEvent` handle, and leaves `size() === 0`; a subsequent `init()` rebuilds from the current `resolvedLinks` without referencing any prior state. ([FR-IDX-10](../../context.md#fr-idx-10), [FR-IDX-11](../../context.md#fr-idx-11))
8. Vitest suite (using an in-memory faked `MetadataCache` with a mutable `resolvedLinks` field and a synchronous `on('resolved')` event bus) covers: symmetry from forward-only input; idempotent `init()`; incremental `resolved` diff (added / removed / unchanged edges); rename-drops-stale-edges; delete-removes-orphaned-node; `neighbors` miss returns the shared frozen empty set (same reference); `snapshot()` is deep read-only; `shutdown()` unsubscribes the listener. ([FR-IDX-10](../../context.md#fr-idx-10), [FR-IDX-11](../../context.md#fr-idx-11))

## Open questions

- **Canvas / non-markdown targets in adjacency ([FR-IDX-05](../../context.md#fr-idx-05) vs [FR-IDX-10](../../context.md#fr-idx-10) and the [context.md Graph boost on non-indexed sources open question](../../context.md#open-questions))** ‚Äî `resolvedLinks` can surface `.canvas` / PDF / image targets before [F36 canvas-file-indexing](../../features-index.md) ships. Proposal: store every path verbatim here (adjacency is format-agnostic) and let [F35 graphrag-boosts](../../features-index.md) skip non-indexed neighbors when composing the boost; verifier to confirm or push a filter down into this feature.
- **`MetadataCache.on('resolved')` diff granularity** ‚Äî Obsidian's `resolved` event does not publish a diff; it signals "cache is now consistent after a change burst." This feature currently recomputes every affected path's reciprocal edges by comparing the fresh `resolvedLinks` slice against the previous snapshot held in the cache. Proposal: keep a shadow `Map<string, Set<string>>` of the last-seen forward edges per path for cheap delta computation; verifier to confirm the O(Œî) update cost is acceptable on a 10 k vault per [NFR-PERF-02](../../context.md#nfr-perf-02) or recommend a full rebuild instead.
- **Tag-shared boost prerequisites ([FR-RAG-04](../../context.md#fr-rag-04))** ‚Äî the tag-shared additive boost in [F35 graphrag-boosts](../../features-index.md) reads chunk tags from [F28 chunking-metadata](../../features-index.md), not from this adjacency. This feature therefore owns link edges only, not a tag index; verifier to confirm that split or pull a tag‚Üípaths index into `GraphCache` alongside adjacency.
