# F43 — Autocompaction threshold engine

## Purpose

Deliver the Layer-2 autocompaction pass that sits behind [F42 microcompaction](../compaction-microcompact/feature.md) in the query-loop order fixed by [compact.md §1](../../../../srs/compact.md#1-overview): when `tokenCountWithEstimation(messages) >= autoCompactThreshold`, Leo issues a single summarization LLM call using the verbatim prompt from [compact.md §10](../../../../srs/compact.md#10-summarization-prompts), consumes the `<summary>` block per [compact.md §10 "Summary Post-Processing"](../../../../srs/compact.md#10-summarization-prompts), and rebuilds the outgoing message array in the `boundaryMarker → summaryMessages → messagesToKeep → attachments → hookResults` order fixed by [compact.md §11](../../../../srs/compact.md#11-post-compaction-message-assembly) — all threshold constants, post-compact budgets, and retry limits sourced from [compact.md §3](../../../../srs/compact.md#3-constants-and-thresholds). The summarization call runs with a 30-second keep-alive ping and up to `MAX_COMPACT_STREAMING_RETRIES = 2` streaming retries per [compact.md §7 Phase 3 Path B "Keep-alive" / "Streaming retry"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) and [compact.md §3 "Retry and Streaming"](../../../../srs/compact.md#3-constants-and-thresholds). It satisfies the top-level layered-compaction requirement at [FR-COMPACT-01](../../context.md#fr-compact-01), the constants/threshold binding at [FR-COMPACT-03](../../context.md#fr-compact-03), the verbatim-prompt binding at [FR-COMPACT-04](../../context.md#fr-compact-04), the post-compact assembly order at [FR-COMPACT-05](../../context.md#fr-compact-05), and the keep-alive + ≤2 streaming retries + ≤3 PTL retries latency NFR at [NFR-PERF-07](../../context.md#nfr-perf-07). PTL retry, circuit breaker, and the ContextAnalyzer pipeline split out to [F44](../../features-index.md), [F45](../../features-index.md), and [F46](../../features-index.md) respectively — this slice covers the happy-path threshold engine, the summarization call, and the post-compact assembly.

## Scope

### In scope

- `autoCompactIfNeeded(messages, context, cacheSafeParams, querySource?, tracking?, snipTokensFreed?)` entry point per [compact.md §6](../../../../srs/compact.md#6-layer-2-autocompaction); returns a `CompactionResult` or `null` when the decision function says skip.
- `shouldAutoCompact(messages, model, querySource?, snipTokensFreed?)` threshold decision per [compact.md §6](../../../../srs/compact.md#6-layer-2-autocompaction) using [F41](../token-estimator-3tier/feature.md)'s `tokenCountWithEstimation`; returns `true` iff `tokenCount - snipTokensFreed >= autoCompactThreshold` with `autoCompactThreshold = effectiveContextWindow - 13_000` and `effectiveContextWindow = contextWindow - min(maxOutputTokensForModel, 20_000)` per [compact.md §3 "Threshold Calculations"](../../../../srs/compact.md#3-constants-and-thresholds).
- Constant bindings from [compact.md §3 "Context Window"](../../../../srs/compact.md#3-constants-and-thresholds), [§3 "Compaction Thresholds"](../../../../srs/compact.md#3-constants-and-thresholds), [§3 "Post-Compaction Budgets"](../../../../srs/compact.md#3-constants-and-thresholds), and [§3 "Retry and Streaming"](../../../../srs/compact.md#3-constants-and-thresholds) exposed as module-level `const` values (`MODEL_CONTEXT_WINDOW_DEFAULT=200_000`, `COMPACT_MAX_OUTPUT_TOKENS=20_000`, `AUTOCOMPACT_BUFFER_TOKENS=13_000`, `POST_COMPACT_MAX_FILES_TO_RESTORE=5`, `POST_COMPACT_TOKEN_BUDGET=50_000`, `POST_COMPACT_MAX_TOKENS_PER_FILE=5_000`, `POST_COMPACT_MAX_TOKENS_PER_SKILL=5_000`, `POST_COMPACT_SKILLS_TOKEN_BUDGET=25_000`, `MAX_COMPACT_STREAMING_RETRIES=2`).
- `contextWindow` resolution using the priority order from [compact.md §3](../../../../srs/compact.md#3-constants-and-thresholds): (1) model string `[1m]` suffix → 1,000,000; (2) provider capability metadata (`max_input_tokens` from LM Studio's `/v1/models` if present); (3) default 200,000. Environment-variable + beta-header paths are out of scope (see below).
- `getCompactPrompt(customInstructions?)` prompt builder returning the verbatim text from [compact.md §10](../../../../srs/compact.md#10-summarization-prompts) — `NO_TOOLS_PREAMBLE` + `BASE_COMPACT_PROMPT` + `DETAILED_ANALYSIS_INSTRUCTION` + optional `Additional Instructions:\n{customInstructions}` + `NO_TOOLS_TRAILER` — assembled by string concatenation from constants that carry the exact section text; a Vitest snapshot binds the rendered prompt byte-for-byte to [compact.md §10](../../../../srs/compact.md#10-summarization-prompts).
- Summarization streaming call via [F02 ProviderManager](../provider-lmstudio-core/feature.md)'s existing `stream` surface with `querySource='compact'`, tools disabled, thinking disabled, `maxOutputTokens = min(COMPACT_MAX_OUTPUT_TOKENS, providerMaxOutput)`, and a system prompt `"You are a helpful AI assistant tasked with summarizing conversations."` per [compact.md §7 Phase 3 Path B](../../../../srs/compact.md#7-layer-3-full-conversation-compaction). Leo does not implement the cache-sharing fork (Path A) in v1 — see Open questions.
- Pre-API message preparation per [compact.md §7 Phase 3 Path B](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) and [compact.md §15 "Image/Document Stripping" / "Attachment Stripping"](../../../../srs/compact.md#15-api-invariant-preservation): `getMessagesAfterCompactBoundary` → append the generated `summaryRequest` user message → `stripReinjectedAttachments` (drop `skill_discovery` / `skill_listing`) → `stripImagesFromMessages` (replace `image` / `document` blocks with `[image]` / `[document]` text markers) → `normalizeMessagesForAPI` (merge streaming chunks, ensure valid ordering).
- 30-second keep-alive signal during the summarization call per [compact.md §7 Phase 3 Path B "Keep-alive"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) — Leo emits a `keepAlive.tick` structured log event every 30s while the stream is open; implemented as a `setInterval` guarded by the call's `AbortController` so the tick stops on abort or completion.
- Streaming retry loop of up to `MAX_COMPACT_STREAMING_RETRIES = 2` attempts with exponential backoff per [compact.md §7 Phase 3 Path B "Streaming retry"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) and [compact.md §3 "Retry and Streaming"](../../../../srs/compact.md#3-constants-and-thresholds); the counter resets on successful completion and is telemetered via `tengu_compact_streaming_retry` fields per [compact.md §17](../../../../srs/compact.md#17-telemetry-events).
- `formatCompactSummary(summary)` post-processing per [compact.md §10 "Summary Post-Processing"](../../../../srs/compact.md#10-summarization-prompts): strip `<analysis>…</analysis>`, replace `<summary>…</summary>` content with `Summary:\n{content}`, collapse multiple blank lines to a double newline, trim.
- Post-compact message assembly per [compact.md §11](../../../../srs/compact.md#11-post-compaction-message-assembly): `buildPostCompactMessages(result)` returns `[boundaryMarker, ...summaryMessages, ...(messagesToKeep ?? []), ...attachments, ...hookResults]` in that exact order; the `boundaryMarker` is a `SystemCompactBoundaryMessage` with `compact_metadata: { trigger: 'auto', pre_tokens }` per [compact.md §2 "SystemCompactBoundaryMessage"](../../../../srs/compact.md#2-architecture).
- Post-compact attachments generated in parallel per [compact.md §7 Phase 4](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) and [compact.md §11 "Attachment Types"](../../../../srs/compact.md#11-post-compaction-message-assembly), each budget-capped using [compact.md §3 "Post-Compaction Budgets"](../../../../srs/compact.md#3-constants-and-thresholds): (a) file restorations — re-read up to `POST_COMPACT_MAX_FILES_TO_RESTORE = 5` most-recently-accessed files via [F16](../tool-registry-builtin-read/feature.md)'s `read_note`, each capped at `POST_COMPACT_MAX_TOKENS_PER_FILE = 5_000` tokens (estimated via F41), with the sum across all attachments capped at `POST_COMPACT_TOKEN_BUDGET = 50_000`; (b) skill attachments — re-inject content for invoked skills with per-skill cap `POST_COMPACT_MAX_TOKENS_PER_SKILL = 5_000` and total cap `POST_COMPACT_SKILLS_TOKEN_BUDGET = 25_000`; (c) plan attachment (Leo-specific) pulled from [F23](../plan-files-todos-store/feature.md)'s plan file if present; (d) plan-mode attachment from [F24](../plan-mode-permissions/feature.md) if `mode === 'plan'`; (e) deferred-tools / agent-listing / MCP-instruction deltas deferred to [F51+](../../features-index.md) — emit empty arrays for those in v1.
- Recursion guards from [compact.md §6 "Should Auto-Compact Decision"](../../../../srs/compact.md#6-layer-2-autocompaction): skip when `querySource === 'compact'` (the summarization call itself) to prevent reentry; skip when user disabled auto-compact via a forthcoming Advanced-section setting (see Open questions).
- `tengu_compact` telemetry event per [compact.md §17](../../../../srs/compact.md#17-telemetry-events) carrying `{preCompactTokenCount, postCompactTokenCount, truePostCompactTokenCount, autoCompactThreshold, isAutoCompact, querySource, compactionInputTokens, compactionOutputTokens, compactionTotalTokens}`, emitted via [F01 Logger](../plugin-bootstrap-logging/feature.md); `tengu_compact_streaming_retry` event on each retry; `tengu_compact_failed` event on failure (consumed later by [F45](../../features-index.md)'s circuit-breaker counter).
- Vitest coverage for: threshold formula at the boundary (just above / just below `autoCompactThreshold`), verbatim-prompt snapshot, post-compact assembly order, per-file / per-skill / total budget caps, keep-alive tick emission, streaming-retry exponential-backoff schedule up to 2 attempts, recursion guard on `querySource='compact'`, `formatCompactSummary` round-trips.

### Out of scope

- Circuit-breaker (3 consecutive failure disable) → [F45](../../features-index.md), per [compact.md §6 "Auto-Compact Execution" step 2](../../../../srs/compact.md#6-layer-2-autocompaction) and [NFR-REL-06](../../context.md#nfr-rel-).
- Session memory compaction (Layer 4) — Leo marks it out of scope per [context.md Out of scope](../../context.md#out-of-scope) and [compact.md §8](../../../../srs/compact.md#8-layer-4-session-memory-compaction); the cheap "try SM first" short-circuit at [compact.md §6 "Auto-Compact Execution" step 5](../../../../srs/compact.md#6-layer-2-autocompaction) is therefore skipped and the call goes straight to the full summarization path.
- Partial compaction (Layer 3 sibling) per [compact.md §9](../../../../srs/compact.md#9-partial-compaction) — no UI requirement exists; see [context.md Open questions](../../context.md#open-questions) "Partial compaction trigger & UI".
- Cache-sharing fork (Path A) per [compact.md §7 Phase 3 Path A](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) — LM Studio does not expose a `runForkedAgent` equivalent in v1; the direct streaming path (Path B) is the only one implemented. See Open questions.
- Environment-variable overrides (`DISABLE_COMPACT`, `CLAUDE_CODE_MAX_CONTEXT_TOKENS`, `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE`, etc.) per [compact.md §18](../../../../srs/compact.md#18-environment-variable-overrides) — Leo does not consume process env for user config; Advanced-section settings land later.
- Feature-flag surface (`tengu_cobalt_raccoon` reactive-only mode, `CONTEXT_COLLAPSE`, `REACTIVE_COMPACT`, `tengu_compact_cache_prefix`, `tengu_compact_streaming_retry`) per [compact.md §19](../../../../srs/compact.md#19-feature-flags) — Leo ships with streaming retry always-on and no remote-config plumbing.
- Hooks integration (pre-compact / post-compact / session-start) per [compact.md §16](../../../../srs/compact.md#16-hooks-integration) — no Leo hook surface exists; `hookResults` is always `[]` in v1.
- Token-warning UI, status line, `/context` command, category breakdown, grid — all deferred to [F46](../../features-index.md), [F47](../../features-index.md), [F48](../../features-index.md).
- Post-compact cleanup (`runPostCompactCleanup`) per [compact.md §12](../../../../srs/compact.md#12-post-compaction-cleanup) — Leo has no prompt-cache or classifier-approval caches to clear in v1; `resetMicrocompactState` is invoked as the only relevant hook, deferred into F42's module; the other entries are Claude-Code-internal and skipped.
- Image / file attachments in the pre-summary messages — the `[image]` / `[document]` marker substitution is wired but only exercised when [F49](../../features-index.md) lands; v1 still strips per [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation) as a no-op on markdown-only vaults.

## Acceptance criteria

1. `shouldAutoCompact(messages, model, querySource?, snipTokensFreed?)` returns `true` iff `tokenCountWithEstimation(messages) - (snipTokensFreed ?? 0) >= autoCompactThreshold` where `autoCompactThreshold = effectiveContextWindow - AUTOCOMPACT_BUFFER_TOKENS` and `effectiveContextWindow = contextWindow - min(maxOutputTokensForModel, 20_000)`; `contextWindow` resolves via the priority `[1m]` suffix → provider `max_input_tokens` → `MODEL_CONTEXT_WINDOW_DEFAULT = 200_000`. A Vitest matrix asserts the boundary at `autoCompactThreshold - 1` (false) and `autoCompactThreshold` (true), plus the `[1m]`-model 1,000,000 branch and the default-200,000 branch. ([FR-COMPACT-01](../../context.md#fr-compact-01), [FR-COMPACT-03](../../context.md#fr-compact-03), [compact.md §3 "Threshold Calculations"](../../../../srs/compact.md#3-constants-and-thresholds), [compact.md §6](../../../../srs/compact.md#6-layer-2-autocompaction))
2. `autoCompactIfNeeded` short-circuits and returns `null` when `querySource === 'compact'` (summarization reentry guard) and when `shouldAutoCompact` returns `false`, without ever calling `ProviderManager.stream`; Vitest asserts zero stream invocations across those branches. ([FR-COMPACT-01](../../context.md#fr-compact-01), [compact.md §6 "Should Auto-Compact Decision"](../../../../srs/compact.md#6-layer-2-autocompaction))
3. The summarization prompt emitted by `getCompactPrompt(customInstructions?)` is byte-identical to the concatenation `NO_TOOLS_PREAMBLE + BASE_COMPACT_PROMPT + DETAILED_ANALYSIS_INSTRUCTION + (customInstructions ? "\n\nAdditional Instructions:\n" + customInstructions : "") + NO_TOOLS_TRAILER` as defined in [compact.md §10](../../../../srs/compact.md#10-summarization-prompts); a Vitest snapshot pins each section's text verbatim to the spec and fails if any whitespace or punctuation drifts. No prompt text is restated in this feature doc or in source comments — the source holds the constant, the spec holds the authoritative reference. ([FR-COMPACT-04](../../context.md#fr-compact-04), [compact.md §10](../../../../srs/compact.md#10-summarization-prompts))
4. The summarization call's request payload matches [compact.md §7 Phase 3 Path B](../../../../srs/compact.md#7-layer-3-full-conversation-compaction): system prompt `"You are a helpful AI assistant tasked with summarizing conversations."`; tools disabled; thinking disabled; `maxOutputTokens = min(COMPACT_MAX_OUTPUT_TOKENS, providerMaxOutput)` with `COMPACT_MAX_OUTPUT_TOKENS = 20_000`; `querySource = 'compact'`. A Vitest + `msw` fixture captures the outbound body and asserts all four fields. ([FR-COMPACT-03](../../context.md#fr-compact-03), [FR-COMPACT-04](../../context.md#fr-compact-04), [compact.md §7 Phase 3](../../../../srs/compact.md#7-layer-3-full-conversation-compaction))
5. Pre-API message preparation applies, in order: `getMessagesAfterCompactBoundary` → append the summary-request user message → `stripReinjectedAttachments` (remove `skill_discovery` / `skill_listing`) → `stripImagesFromMessages` (replace `image` / `document` blocks with `[image]` / `[document]` text markers, including nested inside `tool_result.content`) → `normalizeMessagesForAPI`; Vitest fixtures cover each transformation in isolation and end-to-end. ([FR-COMPACT-04](../../context.md#fr-compact-04), [FR-COMPACT-07](../../context.md#fr-compact-07), [compact.md §7 Phase 3 Path B](../../../../srs/compact.md#7-layer-3-full-conversation-compaction), [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation))
6. The returned `CompactionResult` assembled via `buildPostCompactMessages(result)` produces messages in exactly this order: `boundaryMarker → summaryMessages → (messagesToKeep ?? []) → attachments → hookResults`; a Vitest integration test with fixture inputs asserts the array equals the concatenation in that exact order — no reordering, no interleaving. The `boundaryMarker` is a `SystemCompactBoundaryMessage` carrying `{trigger: 'auto', pre_tokens: preCompactTokenCount}` and the summary user message carries `{isCompactSummary: true, isVisibleInTranscriptOnly: true}`. ([FR-COMPACT-05](../../context.md#fr-compact-05), [compact.md §11](../../../../srs/compact.md#11-post-compaction-message-assembly), [compact.md §2](../../../../srs/compact.md#2-architecture))
7. Post-compact file attachments honour the [compact.md §3 "Post-Compaction Budgets"](../../../../srs/compact.md#3-constants-and-thresholds) caps: at most `POST_COMPACT_MAX_FILES_TO_RESTORE = 5` files are re-attached (ordered by most-recently-read), each capped at `POST_COMPACT_MAX_TOKENS_PER_FILE = 5_000` tokens (estimated via [F41](../token-estimator-3tier/feature.md), truncated at byte offset where the estimator crosses the cap), and the running sum across file + skill + plan + plan-mode attachments is capped at `POST_COMPACT_TOKEN_BUDGET = 50_000` tokens (further attachments are dropped once the budget is exhausted). Files already visible in preserved messages are excluded. Vitest asserts each cap at its boundary and one token over. ([FR-COMPACT-03](../../context.md#fr-compact-03), [FR-COMPACT-05](../../context.md#fr-compact-05), [compact.md §3 "Post-Compaction Budgets"](../../../../srs/compact.md#3-constants-and-thresholds), [compact.md §7 Phase 4](../../../../srs/compact.md#7-layer-3-full-conversation-compaction))
8. Post-compact skill attachments honour `POST_COMPACT_MAX_TOKENS_PER_SKILL = 5_000` per skill and `POST_COMPACT_SKILLS_TOKEN_BUDGET = 25_000` total; skills invoked during the pre-compact session are re-attached in invocation order until either cap is hit, then remaining skills are dropped silently. Vitest asserts both caps with a six-skill / 30_000-token fixture. ([FR-COMPACT-03](../../context.md#fr-compact-03), [FR-COMPACT-05](../../context.md#fr-compact-05), [compact.md §3 "Post-Compaction Budgets"](../../../../srs/compact.md#3-constants-and-thresholds), [compact.md §7 Phase 4](../../../../srs/compact.md#7-layer-3-full-conversation-compaction))
9. The summarization call runs with a 30-second keep-alive ping (`setInterval(fn, 30_000)`) that emits a `keepAlive.tick` structured log event on each firing; the interval is cleared on completion, error, or abort via the shared `AbortController`. A Vitest test with fake timers asserts the tick fires at 30s/60s while the stream is pending and stops immediately on abort. ([NFR-PERF-07](../../context.md#nfr-perf-07), [compact.md §7 Phase 3 Path B "Keep-alive"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction))
10. Streaming failures retry up to `MAX_COMPACT_STREAMING_RETRIES = 2` times with exponential backoff (1s, 2s). The third consecutive failure surfaces the error, emits `tengu_compact_failed {reason: 'no_streaming_response'}`, and returns `null` (the circuit-breaker counter increment ships with [F45](../../features-index.md)). Vitest + `msw` simulates three consecutive stream errors and asserts exactly two retries before failure. ([NFR-PERF-07](../../context.md#nfr-perf-07), [compact.md §3 "Retry and Streaming"](../../../../srs/compact.md#3-constants-and-thresholds), [compact.md §7 Phase 3 Path B "Streaming retry"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction), [compact.md §17 `tengu_compact_failed`](../../../../srs/compact.md#17-telemetry-events))
11. `formatCompactSummary(summary)` strips the `<analysis>…</analysis>` block entirely, extracts the `<summary>…</summary>` content and prefixes it with `Summary:\n`, collapses runs of blank lines to a single double-newline, and trims leading/trailing whitespace; if no `<summary>` block is present the function throws, triggering the `tengu_compact_failed {reason: 'no_summary'}` telemetry event. Vitest round-trips seven fixtures (analysis only, summary only, both, neither, nested angle-brackets, multiple blank-line runs, trailing whitespace). ([FR-COMPACT-04](../../context.md#fr-compact-04), [compact.md §10 "Summary Post-Processing"](../../../../srs/compact.md#10-summarization-prompts), [compact.md §17 `tengu_compact_failed`](../../../../srs/compact.md#17-telemetry-events))
12. API invariants from [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation) are preserved across the entire autocompact pass: every `tool_result` in the outgoing post-compact messages has a matching `tool_use` in a kept assistant message; assistant messages sharing a `message.id` stay adjacent; the first outgoing API message after the boundary has role `user`. Vitest reuses [F42](../compaction-microcompact/feature.md)'s invariant fixtures to confirm the autocompact pass does not re-introduce violations (shared pairing/continuity checker). ([FR-COMPACT-07](../../context.md#fr-compact-07), [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation))
13. End-to-end latency budget: the full autocompact call (threshold check → prompt build → keep-alive ping schedule → stream → summary parse → attachment assembly) is wired through a single `AbortController` propagated from [F10 AgentController](../agent-controller-core/feature.md); cancellation during summarization clears the keep-alive interval, aborts the stream, and returns `null`. A Vitest test asserts abort-during-stream cleans up the interval timer and the stream reader within one microtask. ([NFR-PERF-07](../../context.md#nfr-perf-07), [compact.md §20 "User Abort Handling"](../../../../srs/compact.md#20-error-handling))

## Implementation notes

- [Architecture §3.2 Agent Layer](../../../../architecture/architecture.md#32-agent-layer) — autocompact lives alongside `AgentRunner`, `ContextAssembler`, `Truncator`, and F42's microcompact in the agent layer.
- [Architecture §3.3 Domain / Core (pure)](../../../../architecture/architecture.md#33-domain--core-pure) — the threshold decision, prompt builder, `formatCompactSummary`, and assembly functions are pure; IO is confined to `ProviderManager.stream` and the attachment readers.

## Open questions

- **User-disable setting for auto-compact**: [compact.md §6 "Should Auto-Compact Decision"](../../../../srs/compact.md#6-layer-2-autocompaction) lists `autoCompactEnabled === false` as a skip condition. Leo has no such setting yet; should v1 expose an Advanced-section toggle (default enabled), or ship always-on and surface the toggle later with F03? Proposing always-on for v1.
- **Cache-sharing fork (Path A)**: [compact.md §7 Phase 3 Path A](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) assumes the provider supports `runForkedAgent` with identical cache-key parameters to hit the prompt cache. LM Studio does not expose this; [F38](../../features-index.md) cloud providers may (OpenAI/Anthropic). Skip Path A entirely in v1 and revisit when a provider that supports cache-sharing lands? Proposing yes — v1 is Path B only.
- **`maxOutputTokensForModel` resolution for LM Studio**: [compact.md §3 "Threshold Calculations"](../../../../srs/compact.md#3-constants-and-thresholds) clamps by `min(maxOutputTokensForModel, 20_000)`. LM Studio's `/v1/models` response does not always include a `max_tokens` field; fall back to `MAX_OUTPUT_TOKENS_DEFAULT = 32_000` or the provider's configured `maxTokens` setting from F03? Proposing the F03 setting as the fallback source.
- **File-restoration source of truth**: [compact.md §7 Phase 4](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) restores "most recently accessed files". In Claude Code that is tracked in a `FileReadState`. Leo has no equivalent yet — does the post-compact attachment pass scan pre-compact `tool_result.content` for `read_note` invocations and pick the latest 5 unique paths, or does [F10 AgentController](../agent-controller-core/feature.md) maintain a running LRU? Proposing a new LRU map in AgentController seeded by every `read_note` tool use.
- **Skill invocation tracking**: [compact.md §7 Phase 4 step 5](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) attaches "invoked skill content". [F22](../skills-picker-active-skill/feature.md) tracks the active skill per thread but not the full invocation history. Does v1 re-attach only the currently-active skill's system prompt, or the historical list? Proposing currently-active only (matches single-skill-per-thread assumption in F22).
- **Summary-output validation**: [compact.md §10 "Expected Output Format"](../../../../srs/compact.md#10-summarization-prompts) expects `<analysis>` + `<summary>` blocks. If the model returns only `<summary>`, does `formatCompactSummary` succeed, or should it warn and fail? Proposing succeed with a debug-level log; only fail when neither block appears.
- **Keep-alive pulse mechanism for LM Studio**: [compact.md §7 Phase 3 Path B "Keep-alive"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction) says "session activity signals to prevent WebSocket idle timeouts." LM Studio uses plain SSE over HTTP, not WebSocket — does Leo still need a 30s ping, or is the SSE reader's own TCP keep-alive sufficient? Proposing emit the log tick regardless (cheap + observable), but do not send a wire-level ping unless a specific provider needs it.
- **Interaction with [FR-AGENT-08](../../context.md#fr-agent-) pre-compaction fallback**: F10 ships a pre-compaction truncator (oldest history → RAG). Does autocompact run before or after that truncator? [compact.md §1](../../../../srs/compact.md#1-overview) puts "history snip" ahead of autocompact; history snip in Leo is F10's truncator — document the ordering in F10's follow-up.
- **Post-compact assembly when attachments budget is zero**: If every attachment category is empty (no recent files, no plan, no skill), the post-compact messages collapse to `[boundaryMarker, summary]`. Is that acceptable or should a stub attachment be inserted? Proposing acceptable — matches the upstream contract in [compact.md §11](../../../../srs/compact.md#11-post-compaction-message-assembly).
