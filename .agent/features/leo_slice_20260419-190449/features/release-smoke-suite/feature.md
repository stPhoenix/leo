# F57 — Release smoke suite & CM6 manual-integration checklist

## Purpose

Close the two release-gate Testing NFRs — [NFR-TEST-03](../../context.md#nfr-test-03) "CM6 code validated via manual integration in dev vault" and [NFR-TEST-04](../../context.md#nfr-test-04) "Release smoke: load → index tiny vault → RAG question → agent edit → accept" — with a single vertical slice that ships (a) a deterministic scripted Vitest integration smoke exercising the full `load → index tiny vault → RAG question → agent edit → accept` round-trip through [F20 tool-edit-note-with-lock](../tool-edit-note-with-lock/feature.md) and [F33 rag-tag-filter-search-vault](../rag-tag-filter-search-vault/feature.md) against a bundled 10-note fixture vault, (b) a committed manual-integration checklist (`tests/smoke/CM6-CHECKLIST.md`) covering every CM6 interaction surface — edit lock, readonly decoration, blocked-keystroke `Notice`, grouped `EditorTransaction` single-hop undo, 3s post-edit highlight, Focused Context on cursor / selection / viewport — exactly matching the FR/NFR clauses the unit suites cannot assert, and (c) a `pnpm smoke` npm script wiring both gates into CI and the release ritual. No new user-facing surface is introduced; the slice is pure test harness + documentation, consuming the public APIs of every upstream feature without touching their internals.

## Scope

### In scope

- **Tiny-vault fixture** (`tests/smoke/fixtures/tinyVault/`) — a committed 10-note markdown vault with deterministic frontmatter tags, inline `[[wikilinks]]`, one heading hierarchy deep enough to exercise [F28 chunking-metadata](../chunking-metadata/feature.md)'s chunk boundary, and one note designated as the "target" for the agent-edit hop; seeded identically on every run per [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw).
- **Scripted smoke harness** at `tests/smoke/release.smoke.test.ts` driving the five [NFR-TEST-04](../../context.md#nfr-test-04) phases in order under one `describe` block: (1) `Plugin.onload` from [F01 plugin-bootstrap-logging](../plugin-bootstrap-logging/feature.md) resolves without throw, (2) [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) drains the tiny fixture to completion under a fake-timer `requestIdleCallback` pump, (3) [F33 rag-tag-filter-search-vault](../rag-tag-filter-search-vault/feature.md) `search_vault` / `RAGEngine.query` returns a non-empty `RAGHit[]` for a canned question, (4) [F20 tool-edit-note-with-lock](../tool-edit-note-with-lock/feature.md) applies an `edit_note` via [F18 edit-lock-transactions](../edit-lock-transactions/feature.md) `EditorBridge.withLock`, (5) the inline `[Accept]` click leaves the buffer mutated and the 3s highlight scheduled — per [NFR-TEST-04](../../context.md#nfr-test-04).
- **Fixture LM Studio provider** via `msw` per [NFR-TEST-02](../../context.md#nfr-test-02) + [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw): stubs `/v1/chat/completions` with a canned agent response that emits exactly one `edit_note` `tool_call` targeting the designated tiny-vault note, plus `/v1/embeddings` returning deterministic 1024-dim vectors so [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md) seeds predictably.
- **Fake Obsidian harness** composing the same test doubles already used by [F04 chat-sidebar-view](../chat-sidebar-view/feature.md), [F18 edit-lock-transactions](../edit-lock-transactions/feature.md), and [F33 rag-tag-filter-search-vault](../rag-tag-filter-search-vault/feature.md) Vitest suites: `fake-indexeddb` for the vector store, stubbed `MarkdownView` / `Editor` / `EditorTransaction` for the active-note edit hop, stubbed `app.workspace` / `Vault` wired to the tinyVault fixture, and the existing [F01](../plugin-bootstrap-logging/feature.md) `Logger` captured into a ring buffer for assertion.
- **Pass/fail gate** asserting exactly these post-conditions on a successful run: `RAGHit[].length >= 1`, `EditorBridge.withLock` called exactly once on the active-note path, `VaultAdapter.modify` called zero times on the active-note path (per [F20 AC 2](../tool-edit-note-with-lock/feature.md)), `Editor.undo` called zero times (Accept is a no-op per [F20 AC 5/6](../tool-edit-note-with-lock/feature.md)), inline `[Accept] [Reject]` rendered into the `InlineConfirmation` region of [F04](../chat-sidebar-view/feature.md) (never a native `Modal` per [FR-UI-08](../../context.md#fr-ui-08)), and the log ring buffer contains `tool.invoke.ok {toolId: "edit_note"}` + `edit_note.accept` + `agent.turn.rag.hits` in order — all sourced verbatim from the upstream feature ACs, never restated.
- **CM6 manual-integration checklist** at `tests/smoke/CM6-CHECKLIST.md` per [NFR-TEST-03](../../context.md#nfr-test-03) — one committed markdown checklist covering the six CM6 invariants the unit suites cannot prove in a real renderer: edit-lock readonly decoration blocks keystrokes on the locked range ([FR-EDIT-06](../../context.md#fr-edit-06)), blocked-keystroke `Notice` surfaces ([FR-EDIT-06](../../context.md#fr-edit-06)), grouped `EditorTransaction` collapses to a single native Ctrl/Cmd-Z hop ([FR-EDIT-05](../../context.md#fr-edit-05), [FR-EDIT-09](../../context.md#fr-edit-09)), 3s post-edit highlight flashes and clears ([FR-EDIT-08](../../context.md#fr-edit-08)), Focused Context recomputes on cursor / selection / viewport change ([FR-EDIT-01](../../context.md#fr-edit-01), [FR-EDIT-02](../../context.md#fr-edit-02), [FR-EDIT-03](../../context.md#fr-edit-03)), lock released on every exit path per [F18](../edit-lock-transactions/feature.md) ([FR-EDIT-07](../../context.md#fr-edit-07), [NFR-REL-04](../../context.md#nfr-rel-04)). Each item renders as `- [ ] <clause> — reproduction steps — expected result — linked FR/NFR` and is checked off by a human releaser against a dev vault before the build is shipped.
- **`pnpm smoke` npm script** per [tech stack — Runtime & Build](../../../../standards/tech-stack.md#runtime--build) running the Vitest smoke harness only (excluded from `pnpm test` default run so local TDD stays fast, excluded from `pnpm bench` so perf runs stay clean), plus a `smoke` CI job that runs it on every PR and fails on a single assertion breach.
- **Release-ritual document** at `tests/smoke/RELEASE.md` pinning the two-step release gate: step 1 `pnpm smoke` must pass green ([NFR-TEST-04](../../context.md#nfr-test-04)), step 2 the `CM6-CHECKLIST.md` must have every box ticked by the releaser against a freshly-built plugin loaded into a dev Obsidian vault ([NFR-TEST-03](../../context.md#nfr-test-03)); a `git commit -s` on a completed checklist is the human signal.
- **Structured smoke-run log events** per [NFR-LOG-04](../../context.md#nfr-log-04) via the [F01 Logger](../plugin-bootstrap-logging/feature.md): `smoke.phase.start` / `smoke.phase.ok` / `smoke.phase.fail` (`{phase, durationMs}`) on every one of the five phases, plus a single `smoke.run.summary` (`{passed, failed, totalMs}`) at the end; query text / path / tool-args never log above `debug` per [NFR-LOG-01](../../context.md#nfr-log-01) and [code style — Logging](../../../../standards/code-style.md#logging).

## Acceptance criteria

1. `tests/smoke/fixtures/tinyVault/` ships a deterministic 10-note markdown fixture with frontmatter tags, inline wikilinks, and one designated target note; two invocations with the same seed yield byte-identical outputs, a Vitest fixture asserts the note count is exactly 10, and the fixture is consumed unmodified by both the indexer and the RAG phases. ([NFR-TEST-04](../../context.md#nfr-test-04))
2. `tests/smoke/release.smoke.test.ts` drives the five phases in order — load, index, RAG question, agent edit, accept — inside one `describe` block with sub-`it` phases; each phase emits `smoke.phase.start` / `smoke.phase.ok` / `smoke.phase.fail` via [F01 Logger](../plugin-bootstrap-logging/feature.md) carrying `{phase, durationMs}`, and the suite fails the run on the first phase-fail without skipping downstream phases (so the CI log always shows which phase broke). ([NFR-TEST-04](../../context.md#nfr-test-04))
3. The RAG phase asserts `RAGHit[].length >= 1` on a canned question whose embedding the stubbed `/v1/embeddings` `msw` handler returns deterministically; [F33](../rag-tag-filter-search-vault/feature.md) `RAGEngine.query` is invoked exactly once with the turn's `AbortSignal` threaded through per its [AC 6](../rag-tag-filter-search-vault/feature.md). ([NFR-TEST-04](../../context.md#nfr-test-04))
4. The agent-edit phase asserts [F20 tool-edit-note-with-lock](../tool-edit-note-with-lock/feature.md) routes through [F18 edit-lock-transactions](../edit-lock-transactions/feature.md)'s [`EditorBridge.withLock(range, fn)`](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules) exactly once on the active-note path, calls `VaultAdapter.modify` zero times (per [F20 AC 2](../tool-edit-note-with-lock/feature.md)), and the mutation lands inside a single grouped [`EditorTransaction`](../../../../standards/tech-stack.md#platform-apis) per [FR-EDIT-05](../../context.md#fr-edit-05). ([NFR-TEST-04](../../context.md#nfr-test-04))
5. The accept phase asserts the inline `[Accept] [Reject]` renders into the `InlineConfirmation` region from [F04 chat-sidebar-view](../chat-sidebar-view/feature.md) — never a native Obsidian `Modal` per [FR-UI-08](../../context.md#fr-ui-08) — and clicking `[Accept]` leaves the buffer mutated, calls `Editor.undo` zero times (Accept is a no-op per [F20 AC 5/6](../tool-edit-note-with-lock/feature.md)), and the log ring buffer contains `tool.invoke.ok {toolId: "edit_note"}` followed by `edit_note.accept`. ([NFR-TEST-04](../../context.md#nfr-test-04))
6. `tests/smoke/CM6-CHECKLIST.md` is committed to the repo with one `- [ ]` row per CM6 invariant — readonly decoration ([FR-EDIT-06](../../context.md#fr-edit-06)), blocked-keystroke `Notice` ([FR-EDIT-06](../../context.md#fr-edit-06)), single-hop undo on grouped `EditorTransaction` ([FR-EDIT-05](../../context.md#fr-edit-05), [FR-EDIT-09](../../context.md#fr-edit-09)), 3s post-edit highlight ([FR-EDIT-08](../../context.md#fr-edit-08)), Focused Context recompute on cursor / selection / viewport ([FR-EDIT-01](../../context.md#fr-edit-01), [FR-EDIT-02](../../context.md#fr-edit-02), [FR-EDIT-03](../../context.md#fr-edit-03)), lock released on every exit path ([FR-EDIT-07](../../context.md#fr-edit-07), [NFR-REL-04](../../context.md#nfr-rel-04)) — each with reproduction steps, expected result, and a linked FR/NFR anchor. ([NFR-TEST-03](../../context.md#nfr-test-03))
7. A `pnpm smoke` npm script is added to `package.json` wiring the Vitest smoke harness only (not `pnpm test`, not `pnpm bench`); a `smoke` CI job runs `pnpm smoke` on every PR and fails on a single assertion breach; `tests/smoke/RELEASE.md` pins the two-step release gate (`pnpm smoke` green + `CM6-CHECKLIST.md` ticked) as the release ritual. ([NFR-TEST-03](../../context.md#nfr-test-03), [NFR-TEST-04](../../context.md#nfr-test-04))
8. No public API of [F01](../plugin-bootstrap-logging/feature.md) / [F18](../edit-lock-transactions/feature.md) / [F20](../tool-edit-note-with-lock/feature.md) / [F27](../vault-indexer-dirty-queue/feature.md) / [F29](../embeddings-indexeddb-store/feature.md) / [F31](../rag-cosine-search/feature.md) / [F33](../rag-tag-filter-search-vault/feature.md) changes; a Vitest type-compat snapshot asserts the exported surfaces are byte-identical before and after F57 lands. ([NFR-TEST-04](../../context.md#nfr-test-04))
9. Structured smoke-run log events `smoke.phase.start` / `smoke.phase.ok` / `smoke.phase.fail` / `smoke.run.summary` emit via [F01 Logger](../plugin-bootstrap-logging/feature.md) with `{phase?, durationMs, passed?, failed?, totalMs?}` fields only; query text, note paths, tool args, and RAG hit content never appear at `info` or above per [NFR-LOG-01](../../context.md#nfr-log-01). ([NFR-TEST-04](../../context.md#nfr-test-04))

## Implementation notes

- [Architecture §1 Architectural Principles](../../../../architecture/architecture.md#1-architectural-principles) — the smoke respects "Fail-safe editor ops" and "Observable" by asserting lock release and structured log events on every phase.

- [Architecture §3.2 Agent Layer](../../../../architecture/architecture.md#32-agent-layer) — the smoke drives the `AgentRunner` turn loop through the same `tool_call → lookup → invoke → tool_result` path [F10](../agent-controller-core/feature.md) / [F16](../tool-registry-builtin-read/feature.md) already realise.
- [Architecture §3.4 Adapters](../../../../architecture/architecture.md#34-adapters) — the smoke harness wraps the adapters ([`EditorBridge`](../../../../architecture/architecture.md#34-adapters), `VaultAdapter`, `VectorStore`, `ProviderManager`) with Vitest spies to count call counts on each phase.
- [Architecture §5.2 Chat Turn (no tools)](../../../../architecture/architecture.md#52-chat-turn-no-tools) + [§5.3 Chat Turn (with tool call + confirmation)](../../../../architecture/architecture.md#53-chat-turn-with-tool-call--confirmation) — canonical sequence the smoke replays phase-by-phase.
- [Architecture §10 Concurrency & Lifecycle Rules](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules) — `AbortController` threaded through every phase; smoke asserts teardown leaves no orphan timers.
- [Tech stack — Runtime & Build](../../../../standards/tech-stack.md#runtime--build) — `pnpm smoke` script and Vitest harness choice; the smoke is excluded from the default `pnpm test` run.
- [Tech stack — Testing](../../../../standards/tech-stack.md#testing) — Vitest + `msw` + `fake-indexeddb` stack reused verbatim from the unit suites.
- [Code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw) — `msw` handlers for `/v1/chat/completions` and `/v1/embeddings`; `fake-indexeddb` for the vector store; deterministic fixture seed.
- [Code style — Logging](../../../../standards/code-style.md#logging) — `smoke.*` events ship counts and milliseconds only; content stays at `debug`.
- [Code style — Async & Concurrency](../../../../standards/code-style.md#async--concurrency) — `AbortSignal` threaded through every phase; no orphan timers survive the run.
- [Code style — LangGraph / Agent Layer](../../../../standards/code-style.md#langgraph--agent-layer) — the smoke asserts typed `ToolResult{ok:true,data}` on the happy `edit_note` path.

## Open questions

- **`pnpm smoke` CI runner stability** — inherits the [F50 perf-scale-10k-vault open question on CI runner noise](../perf-scale-10k-vault/feature.md). The smoke asserts hard booleans (call counts, log events, buffer content) so `performance.now()` noise should not affect it, but fake-timer drift on shared runners might; verifier to confirm the smoke runs clean on GitHub-hosted CI or requires a dedicated self-hosted runner per [NFR-TEST-04](../../context.md#nfr-test-04).
- **Manual CM6 checklist vs real-Obsidian Playwright E2E** — [NFR-TEST-03](../../context.md#nfr-test-03) phrases the requirement as "manual integration in dev vault" so a committed checklist satisfies it, but a future Playwright harness against a real Obsidian build could automate it. Proposing manual checklist in v1 (per literal SRS wording) with a Playwright follow-up deferred; verifier to confirm the manual-only interpretation or promote automation now.
- **Release ritual sign-off** — `tests/smoke/RELEASE.md` pins a two-step gate but does not specify who signs off (maintainer? release manager?). Proposing the `git commit -s` of a ticked `CM6-CHECKLIST.md` is the human signal; verifier to confirm versus a GitHub release-draft comment or a CHANGELOG entry.
- **Tiny-vault size** — the SRS phrase "tiny vault" is not quantified. Proposing 10 notes as a minimum that still exercises chunking + multi-note RAG + graph adjacency; verifier to confirm or tighten to 5 (faster run) or extend to 20 (richer RAG signal).
- **Smoke-level coverage for MCP / Plan / Skills / Context** — the canonical [NFR-TEST-04](../../context.md#nfr-test-04) ladder stops at "agent edit → accept" and does not name MCP / Plan / Skills / Context paths. Proposing F57 covers only the canonical ladder per its literal reading; extensions are deferred to follow-up rows once the respective features land. Verifier to confirm or extend the ladder now to cover MCP ([NFR-TEST-05](../../context.md#nfr-test-05) already owned by F51) and context-command ([NFR-TEST-08](../../context.md#nfr-test-08) already owned by F47).
