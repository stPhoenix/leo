# F30 — Index status bar + reindex command + empty-state CTA

## Purpose

Surface the vault-indexer's operational controls to the user: a persistent status-bar progress widget rendering files-remaining plus the current file while [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) is draining per [FR-IDX-14](../../context.md#fr-idx-14); a palette-reachable "Leo: Re-index vault" command that clears the Index Header manifest and re-enqueues every markdown file through F27's `queryOnDemand`/enqueue seam per [FR-IDX-13](../../context.md#fr-idx-13); a reindex-on-model-switch confirmation that fires when the user changes the embedding model from the F03 Provider section (parallel to F27's startup header-mismatch path) per [FR-IDX-13](../../context.md#fr-idx-13); and a no-index "Index vault" CTA surfaced from the ChatView empty-thread region when the header manifest is absent or the first-run flag is set so the user can kick off the initial sweep without leaving the chat. This slice owns only the visible controls — the indexing engine, chunker, and vector store stay behind F27/F28/F29.

## Scope

### In scope

- Status-bar entry added via `Plugin.addStatusBarItem()` per [tech stack — Platform APIs](../../../../standards/tech-stack.md#platform-apis) and [architecture §3.1 UI Layer](../../../../architecture/architecture.md#31-ui-layer-react-mounted-inside-obsidian-views), subscribed to F27's `indexer.drain.start` / `indexer.drain.tick` / `indexer.drain.complete` event stream; renders `Indexing: <n> files left - <basename>` during drain, empty (DOM-removed) when the queue is idle, keyed with `role="status" aria-live="polite"` and the `database` icon via `setIcon("database")` per [FR-IDX-14](../../context.md#fr-idx-14).
- "Leo: Re-index vault" command registered through `Plugin.addCommand({id: "leo-reindex-vault", name: "Leo: Re-index vault"})` per [code style — Obsidian Plugin Patterns](../../../../standards/code-style.md#obsidian-plugin-patterns) and reachable from Obsidian's Hotkeys UI per [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md); callback confirms via inline `Notice` (Cancel / Re-index now), on confirm clears the Index Header manifest, enumerates `app.vault.getMarkdownFiles()`, fans every path through F27's `enqueueDirty`, and triggers an immediate `queryOnDemand` drain so progress shows right away per [FR-IDX-13](../../context.md#fr-idx-13).
- Reindex-on-model-switch confirmation: subscribes to the settings store `embedding model` change event (written through F03's persistence API) and, when the new value diverges from the persisted `IndexHeader.model`, surfaces the same blocking `Notice` contract F27 uses at startup (**now** / **later** / **revert**); **now** routes through the re-index command; **revert** writes the old model value back through F03's settings store per [FR-IDX-13](../../context.md#fr-idx-13).
- No-index empty-state CTA mounted in the ChatView empty-thread region (reusing the F04 chat shell subtree) when either the first-run flag from [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md) is set or `IndexHeader` is absent: renders a one-line prompt "Your vault isn't indexed yet" with an "Index vault" button that dispatches the same `leo-reindex-vault` command handler via `app.commands.executeCommandById` per [code style — Obsidian Plugin Patterns](../../../../standards/code-style.md#obsidian-plugin-patterns); auto-unmounts on first drain-complete.
- Status-bar throttling via `requestAnimationFrame` coalescing drain-tick renders to one paint per frame, with a collapsed-truncation variant when the status-bar available width forces a shorter label per [tech stack — UI Layer](../../../../standards/tech-stack.md#ui-layer) and [code style — Styling (Tailwind + Obsidian)](../../../../standards/code-style.md#styling-tailwind--obsidian) (Obsidian CSS variables only, zero colour literals).
- Structured log events `indexer.ui.reindex-command` / `indexer.ui.model-switch-prompt` / `indexer.ui.empty-state-cta` / `indexer.ui.status-bar-throttled` via the [F01 Logger](../plugin-bootstrap-logging/feature.md) per [code style — Logging](../../../../standards/code-style.md#logging); no file paths above `debug`.
- Vitest coverage: status-bar mount/unmount on drain events (`vi.useFakeTimers`), command invocation enqueues every markdown file and triggers `queryOnDemand`, model-switch confirmation `now`/`later`/`revert` branches, empty-state CTA mounts when header absent and unmounts on drain-complete, `rAF` throttling coalesces multiple tick events per [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw).

### Out of scope

- Indexer engine, dirty queue, listener fan-out, header match/mismatch at startup, `requestIdleCallback` drain scheduling → [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md).
- Heading-based / fixed-size chunking and chunk metadata → [F28 chunking-metadata](../chunking-metadata/feature.md).
- `EmbeddingClient`, IndexedDB `VectorStore`, corruption detection + rebuild prompt → [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md).
- RAG search, context-pack assembly, per-query ranking, exclude list → [F31 rag-search-context-pack](../../features-index.md) and [F32 rag-exclude-list](../../features-index.md).

## Acceptance criteria

1. A status-bar entry registered via `Plugin.addStatusBarItem()` renders `Indexing: <n> files left - <basename>` whenever [F27](../vault-indexer-dirty-queue/feature.md)'s drain loop is active, updates on each `indexer.drain.tick`, and DOM-removes itself on `indexer.drain.complete`; updates are `rAF`-throttled so multiple ticks per frame coalesce to a single paint. ([FR-IDX-14](../../context.md#fr-idx-14))
2. A command "Leo: Re-index vault" is registered with `Plugin.addCommand({id: "leo-reindex-vault"})`, appears in the command palette, is reachable from Obsidian's Hotkeys UI via the shared helper from [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md), and on confirm clears the `IndexHeader` manifest, re-enqueues every `app.vault.getMarkdownFiles()` path through [F27](../vault-indexer-dirty-queue/feature.md)'s `enqueueDirty`, and triggers an immediate `queryOnDemand` drain. ([FR-IDX-13](../../context.md#fr-idx-13))
3. When the user changes the embedding model in the F03 Provider section, a blocking `Notice` surfaces with **now / later / revert model** choices matching [F27](../vault-indexer-dirty-queue/feature.md)'s startup contract; **now** invokes the re-index command handler, **later** defers and leaves the index untouched, **revert** writes the old `{model, dim}` back into settings through F03's persistence API. ([FR-IDX-13](../../context.md#fr-idx-13))
4. When the first-run flag from [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md) is set OR `IndexHeader` is absent, the ChatView empty-thread region renders an "Index vault" CTA button that dispatches `leo-reindex-vault` via `app.commands.executeCommandById`; the CTA auto-unmounts on the first `indexer.drain.complete` and never mounts when an index already exists. ([FR-IDX-13](../../context.md#fr-idx-13), [FR-IDX-14](../../context.md#fr-idx-14))
5. The status-bar entry is keyed `role="status" aria-live="polite"`, uses `setIcon("database")`, and styles the label exclusively through Obsidian CSS variables (`--text-muted`, `--background-modifier-border`) with zero colour literals per [code style — Styling (Tailwind + Obsidian)](../../../../standards/code-style.md#styling-tailwind--obsidian). ([FR-IDX-14](../../context.md#fr-idx-14))
6. Structured log events `indexer.ui.reindex-command` / `indexer.ui.model-switch-prompt` / `indexer.ui.empty-state-cta` / `indexer.ui.status-bar-throttled` are emitted through the [F01 Logger](../plugin-bootstrap-logging/feature.md) on each respective action, with no file paths above `debug`. ([FR-IDX-13](../../context.md#fr-idx-13), [FR-IDX-14](../../context.md#fr-idx-14))
7. Vitest suite (with `vi.useFakeTimers`, in-memory `VaultAdapter` fake, stubbed F27 event bus) covers: status-bar mount/update/unmount across `drain.start`/`drain.tick`/`drain.complete`, `rAF` throttling coalescing ticks, re-index command re-enqueues every markdown file, model-switch confirmation `now`/`later`/`revert` branches, empty-state CTA mount when header absent and auto-unmount on drain-complete, command-palette registration via `Plugin.addCommand`. ([FR-IDX-13](../../context.md#fr-idx-13), [FR-IDX-14](../../context.md#fr-idx-14))

- Drives requirements [FR-IDX-13](../../context.md#fr-idx-13), [FR-IDX-14](../../context.md#fr-idx-14); glossary anchor [context.md Glossary — Index header](../../context.md#glossary).
- Downstream: [F31 rag-search-context-pack](../../features-index.md) benefits from the empty-state CTA preventing first-query-no-results; [F38 cloud-providers-safestorage](../../features-index.md) may reuse the model-switch confirmation pattern when cloud-provider embedding models ship.

## Open questions

- **Status-bar collapsed label format** — Obsidian status bars have no fixed width; truncation rules ("Indexing: 142" vs "Idx 142") are undefined by the SRS. Proposal: full label by default, basename only when available width < 140 px (measured via `ResizeObserver` on the status-bar item); verifier to confirm.
- **Empty-state CTA placement inside ChatView** — the F04 chat-sidebar-view slice ships the empty-thread region without example prompts (those defer to F04's later polish). This feature mounts the "Index vault" CTA there; if F04 later adds example prompts, they should sit below the CTA. Verifier to confirm stacking order.
- **Model-switch confirmation vs F27 startup path** — F27 owns the startup header-mismatch `Notice`; this feature owns the mid-session settings-change path. Both should share a single `ReindexPromptService` helper so copy and log events stay in lockstep. Proposal: extract the helper here and wire F27 to consume it in a follow-up; verifier to confirm the split is acceptable for Phase 3.
- **Re-index idempotency on rapid double-click** — if the user fires "Leo: Re-index vault" twice in quick succession, the second invocation should be a no-op while the first is still enumerating. Proposal: gate the command handler on an in-flight flag that clears on the next `drain.complete`; verifier to pin the exact debounce window.
