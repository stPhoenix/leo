# F33 — Tag filter + `search_vault` tool

## Purpose

Extend the [F31 `RAGEngine.query`](../rag-cosine-search/feature.md) contract with an optional `tags?: string[]` input that restricts results to chunks whose note carries **any** requested tag per [FR-RAG-05](../../context.md#fr-rag-05), and expose that capability to the agent as a first-class built-in `search_vault(query, tags?)` [`ToolSpec`](../tool-registry-builtin-read/feature.md) on the [F16 `ToolRegistry`](../tool-registry-builtin-read/feature.md) so the LLM can route explicit "find me notes about X tagged Y" turns through the same retrieval seam; simultaneously wire the [F10 `AgentRunner`](../agent-controller-core/feature.md) pre-prompt grounding `RAGEngine.query` call it currently stubs out so every user turn is grounded in top-K hits before the system prompt is assembled per [FR-AGENT-02](../../context.md#fr-agent-02). The tag predicate reads the [`frontmatter_tags` + `inline_tags`](../chunking-metadata/feature.md) arrays already written onto every [F28 chunk](../chunking-metadata/feature.md) and the [F29 `vectors` row](../embeddings-indexeddb-store/feature.md), so no new storage work is needed; the predicate fires **after** the embed step and **during** the linear scan so tag-filtered rows never consume top-K slots per [FR-RAG-06](../../context.md#fr-rag-06), and composes cleanly under the [F32 exclude-list](../rag-exclude-list/feature.md) pre-filter and [F35 graph/tag boosts](../../features-index.md) that layer on top of [F31 `Scorer.cosine`](../rag-cosine-search/feature.md). The tool surface and the pre-prompt grounding together realise [FR-AGENT-02](../../context.md#fr-agent-02)'s "Query RAGEngine before constructing the LLM prompt" clause and [FR-RAG-05](../../context.md#fr-rag-05)'s tag-filter contract in a single slice; cosine search itself stays in [F31](../rag-cosine-search/feature.md), exclude-list filtering stays in [F32](../rag-exclude-list/feature.md), graph boosts stay in [F35](../../features-index.md), and skill `allowedTools` gating stays in [F22](../skills-picker-active-skill/feature.md).

## Scope

### In scope

- Extend the [F31 `RAGEngine.query`](../rag-cosine-search/feature.md) signature to `query(text: string, opts?: {k?: number; tags?: string[]; signal?: AbortSignal}): Promise<RAGHit[]>` per [architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules); `tags` defaults to `undefined` (no filter) and an **empty array** `[]` is treated as "no filter" (strict identity with undefined) so the tool path can pass `tags ?? []` without changing semantics per [FR-RAG-05](../../context.md#fr-rag-05); `RAGHit` shape stays `{path, line_start, line_end, score}` per [FR-RAG-06](../../context.md#fr-rag-06).
- `TagMatcher.matches(chunkTags: {frontmatter: string[]; inline: string[]}, requested: string[]): boolean` pure predicate per [architecture §3.3 Domain / Core](../../../../architecture/architecture.md#33-domain--core-pure): returns `true` when `requested` is empty or when the union `frontmatter ∪ inline` shares **any** entry with `requested` (set intersection non-empty), mounted at `src/rag/TagMatcher.ts` per [architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed); tag normalisation (lowercase, strip leading `#`, dedupe) is applied once to `requested` on entry so scan-loop cost stays `O(1)` per row per [NFR-PERF-03](../../context.md#nfr-perf-03).
- Tag-filter injection point in the [F31](../rag-cosine-search/feature.md) linear-scan path per [FR-RAG-05](../../context.md#fr-rag-05) + [FR-RAG-06](../../context.md#fr-rag-06): after the [F32 exclude-list](../rag-exclude-list/feature.md) pre-scan filter and **before** `Scorer.cosine`, `if (!TagMatcher.matches(row.tags, normalisedTags)) continue`; filtered rows never score and never occupy a top-K heap slot; empty `tags` makes the guard a no-op with strict identity vs the F31 path per [code style — TypeScript](../../../../standards/code-style.md#typescript).
- `search_vault` built-in `ToolSpec` per [FR-AGENT-04](../../context.md#fr-agent-04) + [FR-RAG-05](../../context.md#fr-rag-05) registered at `Plugin.onload` into the [F16 `ToolRegistry`](../tool-registry-builtin-read/feature.md) alongside `read_note`: `{id: "search_vault", description: "Semantic search over the vault; optional tag filter", schema: z.object({query: z.string().min(1), tags: z.array(z.string()).optional()}), requiresConfirmation: false, source: "builtin", invoke}` per [code style — Zod & Tool Schemas](../../../../standards/code-style.md#zod--tool-schemas); `invoke(input, ctx)` calls `ragEngine.query(input.query, {tags: input.tags, signal: ctx.signal})` and returns `{ok: true, data: {hits: RAGHit[]}} | {ok: false, error}` per [code style — LangGraph / Agent Layer](../../../../standards/code-style.md#langgraph--agent-layer).
- [F10 `AgentRunner`](../agent-controller-core/feature.md) pre-prompt grounding `RAGEngine.query` call per [FR-AGENT-02](../../context.md#fr-agent-02) + [architecture §5.2 Chat Turn (no tools)](../../../../architecture/architecture.md#52-chat-turn-no-tools): replaces F10's stub hits-provider with a real call `const hits = await ragEngine.query(userMessage, {signal})` issued **before** `ContextAssembler` composes the system prompt so the hits feed the `RAGHits` slot F10 already reserved per [FR-AGENT-03](../../context.md#fr-agent-03); no `tags` passed on the pre-prompt path (tag filter is tool-driven only per [FR-RAG-05](../../context.md#fr-rag-05)); unavailable store returns `[]` per [F31](../rag-cosine-search/feature.md) so the turn degrades gracefully per [NFR-REL-03](../../context.md#nfr-rel-03).
- Tag normalisation per [FR-IDX-07](../../context.md#fr-idx-07) + [F28 chunking-metadata](../chunking-metadata/feature.md) conventions: matching is case-insensitive, leading `#` stripped from both requested and stored tags, nested tag paths (`area/work`) match as literal strings (no hierarchical prefix matching in v1 per [FR-RAG-05](../../context.md#fr-rag-05)); duplicate entries in `requested` deduped via `new Set<string>()` before the scan loop.
- `search_vault` result payload per [FR-AGENT-04](../../context.md#fr-agent-04): strictly `{hits: RAGHit[]}` with each hit `{path, line_start, line_end, score}` — no `text`, no `chunkId`, no `heading_path`, downstream `ContextAssembler` re-reads prose via [VaultAdapter](../../../../architecture/architecture.md#34-adapters) per [F31](../rag-cosine-search/feature.md); empty store / unavailable / zero matches all resolve to `{ok: true, data: {hits: []}}` (never a tool-error) so the LLM can reason about "no matches" per [code style — LangGraph / Agent Layer](../../../../standards/code-style.md#langgraph--agent-layer).
- Structured log events per [NFR-LOG-04](../../context.md#nfr-log-04) via the [F01 Logger](../plugin-bootstrap-logging/feature.md): `rag.query.tag-filter` (`{requested: number, kept: number, dropped: number}` — counts only, never tag strings above `debug`), `tool.invoke.start` / `tool.invoke.ok` / `tool.invoke.error` for `search_vault` inherited from [F16](../tool-registry-builtin-read/feature.md), `agent.turn.rag.ms` + `agent.turn.rag.hits` on the pre-prompt grounding call; query text and tag strings never log above `debug` per [NFR-LOG-01](../../context.md#nfr-log-01).
- Vitest coverage per [NFR-TEST-01](../../context.md#nfr-test-01) and [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw): pure `TagMatcher.matches` (empty requested = always true, single match, multi match, zero intersection, case-insensitive, `#`-strip, frontmatter-only, inline-only, both sources); tag-filter injection preserves F31 top-K ordering on unfiltered runs (byte-identity vs F31 snapshot); tag-filtered rows never enter the heap (regression guard on top-K slot count); `search_vault` tool happy path / empty-hits / abort-propagation / Zod-schema rejection of missing `query`; `AgentRunner` pre-prompt `RAGEngine.query` invoked exactly once per turn before `ContextAssembler` with `signal` threaded; unavailable-store path resolves to empty hits without throwing.

### Out of scope

- Query embedding + cosine similarity + top-K selection + same-file overlap merge → [F31 rag-cosine-search](../rag-cosine-search/feature.md); this feature only adds the `tags` opt + tool registration on top.
- Glob-pattern exclude-list filtering pre-scan per [FR-RAG-08](../../context.md#fr-rag-08) → [F32 rag-exclude-list](../rag-exclude-list/feature.md); composes before tag filter in the scan loop.
- 1-hop / 2-hop graph boosts and tag-shared **score boost** (1.1× additive) per [FR-RAG-02](../../context.md#fr-rag-02)–[FR-RAG-04](../../context.md#fr-rag-04) → [F35 graphrag-boosts](../../features-index.md); this feature does tag-**filtering**, not tag-**scoring**.
- Skill `allowedTools` gating of `search_vault` per [FR-SKILL-07](../../context.md#fr-skill-07) + [FR-AGENT-12](../../context.md#fr-agent-12) → [F22 skills-picker-active-skill](../skills-picker-active-skill/feature.md); `ToolRegistry.listFor(thread)` pass-through hook from [F16](../tool-registry-builtin-read/feature.md) applies no filter in this slice.
- Tool-confirmation UX, Allow-once / Allow-for-thread persistence → [F17 tool-confirmation-flow](../tool-confirmation-flow/feature.md); `search_vault` is read-only with `requiresConfirmation: false` so this slice bypasses the flow entirely per [FR-AGENT-04](../../context.md#fr-agent-04).
- Hierarchical tag prefix matching (`area` matching `area/work`) and tag autocomplete in the composer → deferred post-v1; [FR-RAG-05](../../context.md#fr-rag-05) specifies literal any-of match.

## Acceptance criteria

1. `RAGEngine.query(text, {tags})` returns only hits whose row's `frontmatter_tags ∪ inline_tags` intersects the normalised `tags` set (case-insensitive, leading `#` stripped); `tags` unset **or** `[]` is strictly equivalent to the [F31](../rag-cosine-search/feature.md) no-filter path (byte-identical result sequence). ([FR-RAG-05](../../context.md#fr-rag-05))
2. `TagMatcher.matches(chunkTags, requested): boolean` is a pure function: returns `true` when `requested` is empty, `true` when `frontmatter ∪ inline` shares any entry with `requested` after normalisation, `false` otherwise; `#Tag`, `#tag`, and `tag` all match the same row; never throws on malformed tag arrays (empty / undefined branches return safe defaults). ([FR-RAG-05](../../context.md#fr-rag-05))
3. Tag-filtered rows are rejected **before** `Scorer.cosine` and never occupy a top-K heap slot — a Vitest fixture seeding 100 rows where only 10 carry the requested tag asserts the returned `RAGHit[]` length is `min(10, DEFAULT_TOP_K)` and the heap comparison counter records ≤10 scoring operations. ([FR-RAG-05](../../context.md#fr-rag-05), [FR-RAG-06](../../context.md#fr-rag-06))
4. The built-in `search_vault` `ToolSpec` is registered at `Plugin.onload` via [F16 `ToolRegistry.register`](../tool-registry-builtin-read/feature.md) with `requiresConfirmation: false`, Zod schema `z.object({query: z.string().min(1), tags: z.array(z.string()).optional()})`, and `invoke(input, ctx)` that calls `ragEngine.query(input.query, {tags: input.tags, signal: ctx.signal})` and returns `{ok: true, data: {hits: RAGHit[]}}` on success, `{ok: false, error}` only on thrown exceptions. ([FR-AGENT-02](../../context.md#fr-agent-02), [FR-RAG-05](../../context.md#fr-rag-05))
5. `search_vault` with missing / non-string `query` is rejected by the Zod parse in the registry `invoke` wrapper and returns `{ok: false, error}` without touching `RAGEngine`; `tags: []` and `tags` unset both resolve the same way as a no-tag query (byte-identical hits). ([FR-AGENT-02](../../context.md#fr-agent-02))
6. `AgentRunner` invokes `ragEngine.query(userMessage, {signal})` exactly once per non-tool chat turn **before** `ContextAssembler` composes the system prompt; the returned `RAGHit[]` is fed into the `RAGHits` slot [F10](../agent-controller-core/feature.md) already reserves, and the turn's `AbortSignal` is threaded through; unavailable-store `[]` propagates as empty `RAGHits` without throwing. ([FR-AGENT-02](../../context.md#fr-agent-02))
7. Structured log events `rag.query.tag-filter` (`{requested, kept, dropped}` counts only, no tag strings above `debug`), `agent.turn.rag.ms`, and `agent.turn.rag.hits` are emitted via the [F01 Logger](../plugin-bootstrap-logging/feature.md); `tool.invoke.*` events inherit their shape from [F16](../tool-registry-builtin-read/feature.md). ([FR-RAG-05](../../context.md#fr-rag-05), [FR-AGENT-02](../../context.md#fr-agent-02))
8. Vitest suite covers: `TagMatcher` normalisation matrix (empty-requested, single-match, multi-match, zero-intersection, case-insensitive, `#`-strip, frontmatter-only, inline-only, both-sources); filter preserves F31 top-K ordering on unfiltered runs (snapshot byte-identity); filtered rows never enter heap (scoring-call counter ≤ matching row count); `search_vault` happy / empty-hits / Zod-rejection / abort-propagation; `AgentRunner` pre-prompt call invoked once with signal and fed into `ContextAssembler`. ([FR-RAG-05](../../context.md#fr-rag-05), [FR-AGENT-02](../../context.md#fr-agent-02))

## Open questions

- **Hierarchical tag matching (`area` ⊇ `area/work`)** — [FR-RAG-05](../../context.md#fr-rag-05) says "any requested tag" without specifying prefix semantics. This feature implements **literal string equality** (post-normalisation), matching Obsidian's own tag-query convention. Verifier to confirm literal-only is the v1 contract or flag a follow-up for hierarchical semantics.
- **`search_vault` result payload — hits vs flattened prose** — the tool returns `{hits: RAGHit[]}` (path+lines+score, no text) so the LLM sees the same shape `ContextAssembler` consumes; an alternative is to inline `VaultAdapter.read` slices per hit. Verifier to pick one — current design keeps the tool cheap and lets `ContextAssembler` own prose-loading per [F31](../rag-cosine-search/feature.md)'s narrow-payload rationale.
- **Pre-prompt RAG call timeout** — [NFR-PERF-03](../../context.md#nfr-perf-03) budgets the scan at ≤200 ms but the embed hop to LM Studio can add another 100–300 ms. If a slow provider stalls grounding, the turn waits. Proposing a soft 1 s timeout that resolves to `[]` and logs `agent.turn.rag.timeout`; verifier to confirm the budget or punt to [F50 perf-scale-10k-vault](../../features-index.md).
