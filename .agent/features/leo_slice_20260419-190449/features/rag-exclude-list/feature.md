# F32 — RAG exclude list

## Purpose

Wrap the [F31 `RAGEngine.query`](../rag-cosine-search/feature.md) scan seam and the [F27 `VaultIndexer.enqueueDirty`](../vault-indexer-dirty-queue/feature.md) admission seam with a single user-configurable glob-pattern exclude list so that notes under paths the user does not want in RAG (drafts, scratch, personal, attachments) are both (a) filtered out of top-K results before scoring and (b) rejected from the dirty queue so the embedding pipeline never wastes work on them per [FR-RAG-08](../../context.md#fr-rag-08). The feature owns a typed `ExcludeList` domain value (ordered array of glob patterns) loaded via [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md) under the Indexing section, a pure `ExcludeMatcher.matches(path: string): boolean` matcher composed from [`minimatch`](../../../../standards/tech-stack.md#retrieval-layer) (or equivalent pure glob matcher per [architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points)), and two narrow seams where the matcher is injected — one at the very top of `RAGEngine.query` before [F31](../rag-cosine-search/feature.md)'s cosine scan, and one inside `VaultIndexer.enqueueDirty` immediately after the markdown-only filter per [FR-IDX-05](../../context.md#fr-idx-05). A settings change invalidates the persisted [F27](../vault-indexer-dirty-queue/feature.md) queue by draining now-excluded paths and persists them out of `<vault>/.leo/index/queue.json` per the cross-feature [open question on queue reconciliation](../vault-indexer-dirty-queue/feature.md). No scoring math, no graph math, no re-embedding, no UI beyond a single settings textarea — those are explicitly downstream.

## Scope

### In scope

- `ExcludeList = readonly string[]` domain value (ordered, empty `[]` default) + pure `ExcludeMatcher.matches(path: string, patterns: readonly string[]): boolean` per [architecture §3.3 Domain / Core (pure)](../../../../architecture/architecture.md#33-domain--core-pure); no IO, no state, no side-effects per [best practices — Core Principles](../../../../standards/best-practices.md#core-principles).
- Glob matching backed by [`minimatch`](../../../../standards/tech-stack.md#retrieval-layer) (or an equivalent minimal pure glob library chosen behind a single adapter per [architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points)); anchored at the vault-relative path root; `**`, `*`, `?`, and `!`-negation semantics follow the library's defaults; empty patterns and whitespace-only patterns are dropped at load time.
- Query-time filter per [FR-RAG-08](../../context.md#fr-rag-08): `RAGEngine.query(text, opts)` calls `ExcludeMatcher.matches` against every row's `path` returned by [F31](../rag-cosine-search/feature.md)'s `VectorStore.getAll()` scan **before** cosine computation so excluded rows neither score nor occupy a top-K slot; `Scorer.cosine` is never invoked for a filtered row; the same-file overlap merge in [F31](../rag-cosine-search/feature.md) is unaffected.
- Indexer-time filter per the [FR-RAG-08 scope open question](../../context.md#open-questions): `VaultIndexer.enqueueDirty(path)` early-returns on `ExcludeMatcher.matches(path) === true` **after** the markdown-only filter from [F27](../vault-indexer-dirty-queue/feature.md) per [FR-IDX-05](../../context.md#fr-idx-05); a `indexer.skip.excluded` structured log event carries `{patternCount}` only (no `path` above `debug`) per [code style — Logging](../../../../standards/code-style.md#logging).
- Settings field mounted inside the Indexing section of [F03](../settings-tab-scaffold/feature.md) per [FR-UI-10](../../context.md#fr-ui-10): one multi-line textarea, one pattern per line, bound through Obsidian `loadData()` / `saveData()` per [tech stack — Platform APIs](../../../../standards/tech-stack.md#platform-apis); patterns trimmed, deduped, empty lines dropped; persisted shape is `{ excludeGlobs: readonly string[] }` slice of `<vault>/.leo/config.json` per [tech stack — Storage Layout](../../../../standards/tech-stack.md#storage-layout).
- Settings-change reconciliation: on pattern save, an `ExcludeListStore` singleton emits `excludeList.changed` on the shared event bus; [F27](../vault-indexer-dirty-queue/feature.md) subscribes and purges any newly-excluded path from the in-memory `DirtyQueue` + rewrites `queue.json` atomically so the persisted queue never contains excluded paths per the [F27 open question](../vault-indexer-dirty-queue/feature.md); [F31](../rag-cosine-search/feature.md) reads the current `ExcludeMatcher` on every `query` (no memoised stale reference).
- `ExcludeListStore` module mounted at `src/settings/ExcludeListStore.ts` per [architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed); exposes `list(): readonly string[]`, `set(patterns: readonly string[]): Promise<void>`, `matcher(): (path: string) => boolean`; internal `Matcher` cache rebuilt on every `set()` call so pattern compilation cost is amortised per [architecture §10 Concurrency & Lifecycle Rules](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules).
- Structured log events via the [F01 Logger](../plugin-bootstrap-logging/feature.md) per [NFR-LOG-04](../../context.md#nfr-log-04): `exclude.settings.loaded` (`{patternCount}`), `exclude.settings.changed` (`{before, after}`), `exclude.rag.filter` (`{rowsIn, rowsOut}`), `exclude.indexer.skip` (`{patternCount}`); path payloads never log above `debug` per [code style — Logging](../../../../standards/code-style.md#logging).
- Vitest coverage per [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw): `ExcludeMatcher.matches` matrix (exact, `**` recursive, `*` segment, negation, case sensitivity, empty-list no-op); `RAGEngine.query` with an injected matcher demonstrating filtered rows never reach `Scorer.cosine` (spy assertion) and top-K still honours `DEFAULT_TOP_K=10` after filtering; `VaultIndexer.enqueueDirty` rejecting excluded paths; settings-change purge rewriting `queue.json` through a faked `VaultAdapter`; empty `excludeGlobs` behaves as identity.

### Out of scope

- Full RAG scoring pipeline (`Scorer.cosine`, top-K min-heap, same-file merge, `AbortSignal` plumbing, `/v1/embeddings` adapter) per [FR-RAG-01](../../context.md#fr-rag-01), [FR-RAG-06](../../context.md#fr-rag-06), [FR-RAG-07](../../context.md#fr-rag-07) → owned by [F31 rag-cosine-search](../rag-cosine-search/feature.md); this feature only injects a pre-scan predicate.
- 1-hop / 2-hop graph boosts and tag-shared additive boost per [FR-RAG-02](../../context.md#fr-rag-02)–[FR-RAG-04](../../context.md#fr-rag-04) → owned by [F35 graphrag-boosts](../../features-index.md); this feature never composes with boost math.
- Optional tag filter at query time and the `search_vault(query, tags?)` tool registration per [FR-RAG-05](../../context.md#fr-rag-05) + [FR-AGENT-02](../../context.md#fr-agent-02) → owned by [F33 rag-tag-filter-search-vault](../../features-index.md).
- Settings-tab scaffold itself (seven collapsible sections, section ordering, first-run wizard, collapse-state persistence) per [FR-UI-10](../../context.md#fr-ui-10) → owned by [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md); this feature only mounts one textarea inside the existing Indexing section.
- Status-bar / reindex-on-model-switch / empty-state CTA per [FR-IDX-13](../../context.md#fr-idx-13), [FR-IDX-14](../../context.md#fr-idx-14) → owned by [F30 indexer-ui-controls](../indexer-ui-controls/feature.md); this feature never paints into the status bar.
- `.canvas` and non-markdown exclusions per [FR-IDX-05](../../context.md#fr-idx-05) → already handled by [F27](../vault-indexer-dirty-queue/feature.md)'s markdown-only filter; this feature's exclude runs **after** that gate.

## Acceptance criteria

1. `ExcludeMatcher.matches(path, patterns)` is a pure function (no IO, no state) that returns `true` iff any glob pattern in `patterns` matches `path`; empty `patterns` returns `false` for every input; the matcher uses [`minimatch`](../../../../standards/tech-stack.md#retrieval-layer) (or equivalent) semantics for `**`, `*`, `?`, and negation. ([FR-RAG-08](../../context.md#fr-rag-08))
2. `RAGEngine.query(text, opts)` consults `ExcludeListStore.matcher()` once per call and filters every row returned by `VectorStore.getAll()` **before** invoking `Scorer.cosine`; a Vitest spy confirms `Scorer.cosine` is never called for an excluded `path`; top-K still honours `DEFAULT_TOP_K=10` from [F31](../rag-cosine-search/feature.md) after filtering. ([FR-RAG-08](../../context.md#fr-rag-08))
3. `VaultIndexer.enqueueDirty(path)` early-returns when `ExcludeListStore.matcher()(path) === true` (after the `.md` filter from [F27](../vault-indexer-dirty-queue/feature.md)), never writes the path to `<vault>/.leo/index/queue.json`, and emits `indexer.skip.excluded` with `{patternCount}` only. ([FR-RAG-08](../../context.md#fr-rag-08), [FR-IDX-05](../../context.md#fr-idx-05))
4. Settings textarea in the Indexing section of [F03](../settings-tab-scaffold/feature.md) persists `excludeGlobs: readonly string[]` through Obsidian `saveData()`; on load, patterns are trimmed, deduped, and empty / whitespace-only lines dropped; the persisted slice lives inside `<vault>/.leo/config.json` per [tech stack — Storage Layout](../../../../standards/tech-stack.md#storage-layout). ([FR-RAG-08](../../context.md#fr-rag-08), [FR-UI-10](../../context.md#fr-ui-10))
5. On settings change, `ExcludeListStore.set(patterns)` emits `excludeList.changed`; [F27](../vault-indexer-dirty-queue/feature.md) drops every now-excluded path from the in-memory `DirtyQueue` and rewrites `queue.json` atomically through `VaultAdapter` so the persisted queue never contains excluded paths across a simulated restart. ([FR-RAG-08](../../context.md#fr-rag-08))
6. Empty `excludeGlobs = []` is a strict no-op: `RAGEngine.query` returns byte-identical hits to the [F31](../rag-cosine-search/feature.md) baseline fixture, and `VaultIndexer.enqueueDirty` admits every markdown path the [F27](../vault-indexer-dirty-queue/feature.md) baseline admits. ([FR-RAG-08](../../context.md#fr-rag-08))
7. Structured log events `exclude.settings.loaded` / `exclude.settings.changed` / `exclude.rag.filter` / `exclude.indexer.skip` emit through the [F01 Logger](../plugin-bootstrap-logging/feature.md) with counts and booleans only; no `path` field above `debug` per [NFR-LOG-04](../../context.md#nfr-log-04). ([FR-RAG-08](../../context.md#fr-rag-08))
8. Vitest suite covers: `ExcludeMatcher.matches` matrix (exact / `**` / `*` / `?` / negation / empty); `RAGEngine.query` spy asserting `Scorer.cosine` is never called on excluded rows; `VaultIndexer.enqueueDirty` rejecting excluded paths; settings-change purge rewriting `queue.json` through a faked `VaultAdapter`; empty-list identity. ([FR-RAG-08](../../context.md#fr-rag-08))

- [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) — supplies the `enqueueDirty(path)` admission seam and the `<vault>/.leo/index/queue.json` persistence this feature filters into and reconciles on settings change per the [F27 open question on exclude-list interplay](../vault-indexer-dirty-queue/feature.md); the markdown-only filter from [FR-IDX-05](../../context.md#fr-idx-05) runs first, this feature's predicate second.
- Drives requirement [FR-RAG-08](../../context.md#fr-rag-08); intersects with the [context.md Exclude-list scope open question](../../context.md#open-questions) (answered here as "applied at both query and indexing time").
- Upstream settings wiring from [F03 settings-tab-scaffold](../settings-tab-scaffold/feature.md) (Indexing section per [FR-UI-10](../../context.md#fr-ui-10)) and [F01 plugin-bootstrap-logging](../plugin-bootstrap-logging/feature.md) (`Logger` channel per [NFR-LOG-04](../../context.md#nfr-log-04)); downstream composition with [F33 rag-tag-filter-search-vault](../../features-index.md) (tag filter runs after exclude), [F35 graphrag-boosts](../../features-index.md) (boost math composed over the already-filtered top-K), and [F50 perf-scale-10k-vault](../../features-index.md) (exclude path cost must stay sub-ms on 10 k rows).

## Open questions

- **Glob library choice (`minimatch` vs `picomatch` vs `micromatch`)** — [tech stack — Retrieval Layer](../../../../standards/tech-stack.md#retrieval-layer) names no specific glob library. `minimatch` is the Obsidian-idiomatic choice (already transitively present via many community plugins) but `picomatch` is smaller and faster. Verifier to pin one for bundle-size predictability per [tech stack — Bundle Budget](../../../../standards/tech-stack.md#bundle-budget).
- **Negation-pattern ordering semantics** — `minimatch` and `picomatch` differ on how later `!pattern` entries override earlier matches. SRS [FR-RAG-08](../../context.md#fr-rag-08) says "glob patterns" without fixing ordering. Proposal: first-match-wins with negation as a second pass; verifier to confirm or switch to last-match-wins.
- **Retroactive deletion on pattern add** — when a user adds a pattern covering already-indexed paths, this feature purges them from `queue.json` but does not touch existing `vectors` rows in IndexedDB (they simply get filtered at query time). Proposal: leave vectors in place (cheap read, expensive re-index) and let [F29](../embeddings-indexeddb-store/feature.md)'s next full rebuild clean them up; verifier to decide whether an explicit "delete excluded vectors" sweep is required.
- **Settings UI placement inside Indexing section** — [F03](../settings-tab-scaffold/feature.md) ships an empty Indexing section scaffold; this feature mounts a single textarea into it. Verifier to confirm the placement does not collide with [F30 indexer-ui-controls](../indexer-ui-controls/feature.md)'s reindex-on-model-switch UI also inside Indexing.
