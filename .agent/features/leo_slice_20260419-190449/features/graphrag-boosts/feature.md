# F35 — 1-hop / 2-hop / tag-shared boosts

## Purpose

Compose the GraphRAG score-boost pass on top of [F31 rag-cosine-search](../rag-cosine-search/feature.md)'s raw cosine output using the symmetric adjacency exposed by [F34 graph-cache-symmetric](../graph-cache-symmetric/feature.md) and the chunk-tag metadata already emitted by [F28 chunking-metadata](../../features-index.md) so that chunks from notes linked to — or sharing tags with — the active note rise in the top-K per [FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03) and [FR-RAG-04](../../context.md#fr-rag-04). The slice introduces a pure `Scorer.applyBoosts(score, ctx)` composed over the existing `Scorer.cosine` math per [architecture §3.3 Domain / Core — Scorer](../../../../architecture/architecture.md#33-domain--core-pure) and a thin `GraphTraversal.neighbors1h/neighbors2h(activePath)` pair per [architecture §3.3 Domain / Core — GraphTraversal](../../../../architecture/architecture.md#33-domain--core-pure) that reads [F34](../graph-cache-symmetric/feature.md)'s `neighbors` surface, with default multipliers 1-hop = 1.5×, 2-hop = 1.2× (graph boost = max of the two when a chunk's note is reachable) and an additive tag-shared +0.1× overlay (so `final = cosine · graphBoost + tagSharedDelta` — the additive-with-graph-boost clause from [FR-RAG-04](../../context.md#fr-rag-04)). Boosts fire only when the caller supplies `activeNotePath`; absent that field the engine degrades to pure cosine so non-chat callers (e.g. [F33 `search_vault`](../rag-tag-filter-search-vault/feature.md)) remain unaffected. Exclude-list filtering ([F32](../../features-index.md)), tag-filter gating ([F33](../rag-tag-filter-search-vault/feature.md)), graph cache construction ([F34](../graph-cache-symmetric/feature.md)), and cosine search itself ([F31](../rag-cosine-search/feature.md)) are out of scope; this feature owns only the boost composition.

## Scope

### In scope

- Extend `RAGEngine.query(text, opts)` per [architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules) with an optional `activeNotePath?: string` and `activeNoteTags?: string[]` field on `opts`; when both present the engine runs the boost pass described below, otherwise the pre-existing [F31](../rag-cosine-search/feature.md) pure-cosine path ships verbatim per [FR-RAG-01](../../context.md#fr-rag-01).
- `GraphTraversal` pure module mounted at `src/rag/GraphTraversal.ts` per [architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed) with `neighbors1h(path, graph): ReadonlySet<string>` (direct read-through to [F34 `GraphCache.neighbors(path)`](../graph-cache-symmetric/feature.md)) and `neighbors2h(path, graph): ReadonlySet<string>` (union of `neighbors1h(n)` for every `n ∈ neighbors1h(path)`, with `path` itself and every 1-hop neighbor removed so the 2-hop set is disjoint from the 1-hop set and the active note).
- `Scorer.applyBoosts(rawScore, ctx): number` pure function added alongside [F31](../rag-cosine-search/feature.md) `Scorer.cosine` per [architecture §3.3 Domain / Core — Scorer](../../../../architecture/architecture.md#33-domain--core-pure); signature `{rawScore: number; chunkPath: string; chunkTags: ReadonlySet<string>; oneHop: ReadonlySet<string>; twoHop: ReadonlySet<string>; activeTags: ReadonlySet<string>; weights: {oneHop: number; twoHop: number; tagShared: number}}`; returns `rawScore · graphBoost + tagSharedDelta` where `graphBoost = 1.5` if `chunkPath ∈ oneHop`, else `1.2` if `chunkPath ∈ twoHop`, else `1.0`, and `tagSharedDelta = 0.1 · rawScore` when `chunkTags ∩ activeTags ≠ ∅` else `0` — encoding the additive-with-graph-boost semantics per [FR-RAG-04](../../context.md#fr-rag-04).
- Default multipliers `GRAPH_BOOST_1H = 1.5 as const`, `GRAPH_BOOST_2H = 1.2 as const`, `TAG_SHARED_BOOST = 1.1 as const` (delta encoded as `TAG_SHARED_BOOST - 1 = 0.1` per [FR-RAG-04](../../context.md#fr-rag-04)'s "additive with graph boost" clause) per [code style — TypeScript](../../../../standards/code-style.md#typescript); sourced from a settings record per [architecture §8 Extension Points — Graph boost tuning](../../../../architecture/architecture.md#8-extension-points) and overridable at `RAGEngine` construction.
- Boost pass injected into `RAGEngine.query` between [F31](../rag-cosine-search/feature.md)'s cosine scoring and the min-heap top-K selection so boosted rows compete for top-K slots (not post-top-K) per [FR-RAG-02](../../context.md#fr-rag-02)/[FR-RAG-03](../../context.md#fr-rag-03)/[FR-RAG-04](../../context.md#fr-rag-04); `chunkTags` resolved from the `frontmatter_tags ∪ inline_tags` already persisted per chunk row by [F28](../../features-index.md) and [F29](../embeddings-indexeddb-store/feature.md), identical normalisation to [F33](../rag-tag-filter-search-vault/feature.md) (case-insensitive, `#`-stripped, deduped).
- 1-hop and 2-hop neighbor sets computed **once per query** (not per row) via `GraphTraversal.neighbors1h/neighbors2h(activeNotePath, graphCache)` and passed into `Scorer.applyBoosts` by reference, avoiding O(N·k) graph traversal on the hot path per [NFR-PERF-03](../../context.md#nfr-perf-03).
- Disjoint-set invariant between 1-hop and 2-hop: `neighbors2h` subtracts `neighbors1h ∪ {activeNotePath}` so a note reachable both 1-hop and 2-hop receives only the higher 1.5× multiplier; 2-hop does **not** compound over 1-hop per [FR-RAG-02](../../context.md#fr-rag-02)/[FR-RAG-03](../../context.md#fr-rag-03).
- Absent-active-note degrade path: when `opts.activeNotePath` is `undefined` the engine skips `GraphTraversal` entirely and routes every row through `applyBoosts` with empty `oneHop`/`twoHop`/`activeTags` sets so the math collapses to `rawScore · 1.0 + 0` (byte-identical to [F31](../rag-cosine-search/feature.md) output) per [FR-RAG-01](../../context.md#fr-rag-01); non-chat callers like [F33 `search_vault`](../rag-tag-filter-search-vault/feature.md) remain unaffected.
- Graph-cache-unavailable degrade: if [F34 `GraphCache.size() === 0`](../graph-cache-symmetric/feature.md) (pre-init or post-shutdown) the boost pass skips traversal, treats `oneHop`/`twoHop` as empty, still applies the tag-shared additive if `activeNoteTags` are present, and emits one `rag.boost.graph-unavailable` log event per query via [F01 Logger](../plugin-bootstrap-logging/feature.md).
- Non-indexed neighbor paths handled per the [context.md Graph boost on non-indexed sources open question](../../context.md#open-questions): `oneHop`/`twoHop` store every path from [F34](../graph-cache-symmetric/feature.md) verbatim (including `.canvas`/PDF targets), but only chunks whose `path` appears in the live `VectorStore` scan can ever match, so non-indexed neighbors boost nothing and cause no error — the filter is implicit in the intersection with scored rows.
- Same-file overlap merge inherited from [F31](../rag-cosine-search/feature.md) runs **after** the boost pass + top-K selection so merged hits keep `score = max(a.score, b.score)` computed on **already-boosted** scores per [FR-RAG-07](../../context.md#fr-rag-07).
- Structured log events via [F01 Logger](../plugin-bootstrap-logging/feature.md) per [NFR-LOG-04](../../context.md#nfr-log-04): `rag.boost.start` (`{oneHopSize, twoHopSize, activeTagsSize}`), `rag.boost.applied` (`{rowsBoostedOneHop, rowsBoostedTwoHop, rowsBoostedTag}`), `rag.boost.graph-unavailable`, `rag.boost.no-active-note` (debug only); active note path and tag strings never log above `debug` per [code style — Logging](../../../../standards/code-style.md#logging).
- Vitest unit coverage per [NFR-TEST-01](../../context.md#nfr-test-01) and [code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw): pure `Scorer.applyBoosts` matrix (none / 1-hop only / 2-hop only / tag-only / 1-hop + tag / 2-hop + tag / active-note-self-reference), `GraphTraversal` disjoint-set invariant, `neighbors2h` excluding the active note and its 1-hop set, one-query-one-traversal call-count assertion, absent-active-note byte-identity vs [F31](../rag-cosine-search/feature.md), graph-cache-unavailable path, boost-then-top-K ordering (a rawScore=0.1 1-hop row beating a rawScore=0.14 non-neighbour).

### Out of scope

- Graph cache construction, symmetric adjacency, `metadataCache.on('resolved')` hook per [FR-IDX-10](../../context.md#fr-idx-10)/[FR-IDX-11](../../context.md#fr-idx-11) → owned by [F34 graph-cache-symmetric](../graph-cache-symmetric/feature.md); this feature only **reads** the `neighbors` surface.
- Cosine similarity math, `VectorStore.getAll()` linear scan, min-heap top-K, `RAGHit` shape, and the `≤200 ms` [NFR-PERF-03](../../context.md#nfr-perf-03) budget per [FR-RAG-01](../../context.md#fr-rag-01)/[FR-RAG-06](../../context.md#fr-rag-06)/[FR-RAG-07](../../context.md#fr-rag-07) → owned by [F31 rag-cosine-search](../rag-cosine-search/feature.md).
- Optional `tags?: string[]` **query-time filter** that drops chunks before scoring per [FR-RAG-05](../../context.md#fr-rag-05) and the `search_vault(query, tags?)` ToolSpec → owned by [F33 rag-tag-filter-search-vault](../rag-tag-filter-search-vault/feature.md); the tag-shared **boost** here is a distinct, additive score adjustment, never a filter.

## Acceptance criteria

1. `RAGEngine.query(text, {activeNotePath, activeNoteTags})` computes the 1-hop set once via `GraphTraversal.neighbors1h(activeNotePath, graphCache)`, the 2-hop set once via `GraphTraversal.neighbors2h` (disjoint from the 1-hop set and the active note), and passes both into `Scorer.applyBoosts` for every scored row **before** min-heap top-K selection. ([FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03))
2. `Scorer.applyBoosts` returns `rawScore · 1.5` for chunks whose `path ∈ oneHop`, `rawScore · 1.2` for chunks whose `path ∈ twoHop`, and `rawScore · 1.0` otherwise; the 2-hop set is recomputed once per query as `(⋃ neighbors1h(n) for n ∈ oneHop) \ (oneHop ∪ {activeNotePath})` so no chunk ever receives the 1.2× multiplier when a 1.5× multiplier also applies. ([FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03))
3. When a chunk's `frontmatter_tags ∪ inline_tags` (normalised case-insensitive, `#`-stripped, deduped) intersects `activeNoteTags`, the engine adds `0.1 · rawScore` **additively on top of** whichever graph multiplier the row received — matching the "additive with graph boost" clause so a 1-hop tag-shared row ends at `rawScore · 1.5 + 0.1 · rawScore = rawScore · 1.6`. ([FR-RAG-04](../../context.md#fr-rag-04))
4. When `opts.activeNotePath` is `undefined`, the boost pass collapses to `rawScore · 1.0 + 0` for every row and the engine's output is byte-identical to the [F31](../rag-cosine-search/feature.md) pure-cosine path; Vitest asserts the snapshot equality against a golden fixture. ([FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03), [FR-RAG-04](../../context.md#fr-rag-04))
5. When [F34 `GraphCache.size() === 0`](../graph-cache-symmetric/feature.md), the engine skips `GraphTraversal`, applies the tag-shared additive only (if `activeNoteTags` non-empty), and emits exactly one `rag.boost.graph-unavailable` log event per query via [F01 Logger](../plugin-bootstrap-logging/feature.md); no `Error` is thrown and top-K still returns a `RAGHit[]`. ([FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03))
6. The boost pass runs **before** the min-heap top-K selection so a low-raw-score 1-hop chunk can displace a higher-raw-score non-neighbour; Vitest proves this by seeding a row with `rawScore=0.10 ∧ oneHop` that beats a row with `rawScore=0.14 ∧ no-boost` in the final top-K ordering. ([FR-RAG-02](../../context.md#fr-rag-02))
7. Structured log events `rag.boost.start` / `rag.boost.applied` / `rag.boost.graph-unavailable` / `rag.boost.no-active-note` emit through [F01 Logger](../plugin-bootstrap-logging/feature.md) with counts only (`oneHopSize`, `twoHopSize`, `activeTagsSize`, `rowsBoostedOneHop`, `rowsBoostedTwoHop`, `rowsBoostedTag`); active note path and tag strings never appear above `debug` per [NFR-LOG-04](../../context.md#nfr-log-04) and [code style — Logging](../../../../standards/code-style.md#logging). ([FR-RAG-04](../../context.md#fr-rag-04))
8. Vitest suite covers `Scorer.applyBoosts` pure-math matrix, `GraphTraversal.neighbors1h/neighbors2h` disjoint-set invariant, one-query-one-traversal call count (`GraphTraversal.neighbors1h` invoked once per query regardless of row count), absent-active-note byte-identity vs [F31](../rag-cosine-search/feature.md), graph-cache-unavailable degrade path, and the boost-then-top-K ordering assertion. ([FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03), [FR-RAG-04](../../context.md#fr-rag-04))

## Dependencies

- [F34 graph-cache-symmetric](../graph-cache-symmetric/feature.md) — supplies the `GraphCache.neighbors(path): ReadonlySet<string>` + `has`/`size` read surface this feature's `GraphTraversal` shells into for 1-hop and 2-hop traversal per [FR-IDX-10](../../context.md#fr-idx-10) and [FR-IDX-11](../../context.md#fr-idx-11); the symmetric forward-plus-back merge invariant in [F34](../graph-cache-symmetric/feature.md) is what lets 1-hop/2-hop neighborhoods be computed from `neighbors(activeNotePath)` without directional reasoning; the `size() === 0` seam drives the unavailable-degrade path.
- [F31 rag-cosine-search](../rag-cosine-search/feature.md) — supplies `RAGEngine.query` + the pure `Scorer.cosine` base this feature composes `applyBoosts` over per [FR-RAG-01](../../context.md#fr-rag-01)/[FR-RAG-06](../../context.md#fr-rag-06), and reserves the "graph + tag score boosts composed over `Scorer.cosine`" downstream-consumer contract called out in its `## Dependencies` section; the boost pass slots in between `Scorer.cosine` and the min-heap per [FR-RAG-06](../../context.md#fr-rag-06), and inherits the same-file overlap merge per [FR-RAG-07](../../context.md#fr-rag-07) unchanged.
- Drives requirements [FR-RAG-02](../../context.md#fr-rag-02), [FR-RAG-03](../../context.md#fr-rag-03), [FR-RAG-04](../../context.md#fr-rag-04); touches [context.md Glossary — GraphRAG](../../context.md#glossary) and intersects with the [context.md Graph boost on non-indexed sources open question](../../context.md#open-questions) (non-indexed neighbors surface here as opaque paths that never match a scored row, so they boost nothing by construction).
- Upstream caller: [F10 agent-controller-core](../agent-controller-core/feature.md) `AgentRunner` already queries `RAGEngine` per [architecture §5.2 Chat Turn](../../../../architecture/architecture.md#52-chat-turn-no-tools) and supplies the `activeNotePath`/`activeNoteTags` from the Focused Context payload emitted by [F08 editor-bridge-focused-context](../editor-bridge-focused-context/feature.md). Downstream consumers tracked in [features-index.md](../../features-index.md): [F50 perf-scale-10k-vault](../../features-index.md) (keeps boost pass under the ≤200 ms budget at scale).

- [Architecture §3.3 Domain / Core — GraphTraversal](../../../../architecture/architecture.md#33-domain--core-pure) — pins `GraphTraversal` as the pure k-hop computation tier over [F34](../graph-cache-symmetric/feature.md)'s adjacency.

- [Architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules) — binds `FR-RAG-*` to `RAGEngine` + `Scorer` + `GraphTraversal`.
- [Tech stack — Retrieval Layer](../../../../standards/tech-stack.md#retrieval-layer) — `Map<string, Set<string>>` adjacency fed through `GraphTraversal`.
- [Code style — TypeScript](../../../../standards/code-style.md#typescript) — `GRAPH_BOOST_1H/2H/TAG_SHARED` as `as const` literals; `ReadonlySet` on traversal outputs.
- [Code style — Async & Concurrency](../../../../standards/code-style.md#async--concurrency) — one traversal per query, never per row; synchronous pure math.
- [Code style — Logging](../../../../standards/code-style.md#logging) — `rag.boost.*` counts only above `debug`.
- [Code style — Testing (Vitest + msw)](../../../../standards/code-style.md#testing-vitest--msw) — pure-math fixtures; no IndexedDB or network needed for `Scorer`/`GraphTraversal` unit suite.

## Open questions

- **Tag-shared `1.1×` as multiplier vs `+0.1·rawScore` as additive ([FR-RAG-04](../../context.md#fr-rag-04))** — the SRS states "default 1.1×, additive with graph boost"; this feature encodes the semantics as `final = rawScore · graphBoost + 0.1 · rawScore` so a non-neighbour tag-shared row lands at `rawScore · 1.1` (consistent with a 1.1× multiplier) and a 1-hop tag-shared row lands at `rawScore · 1.6` (the two boosts stack additively, not multiplicatively). Verifier to confirm this is the intended semantics vs an alternate `final = rawScore · graphBoost · tagSharedMultiplier` reading.
- **Non-indexed 1-hop neighbors ([FR-RAG-02](../../context.md#fr-rag-02) vs [FR-IDX-05](../../context.md#fr-idx-05), inherits [F34 open question](../graph-cache-symmetric/feature.md))** — `.canvas` (pre-[F36](../../features-index.md)) / PDF targets appear in [F34](../graph-cache-symmetric/feature.md)'s adjacency but have no chunks in `VectorStore`. Current design lets the intersection-with-scored-rows silently drop them. Verifier to confirm silent-drop is acceptable or require a `rag.boost.non-indexed-neighbor` debug log.
- **Active note's own chunks boost semantics** — the active note's own path is excluded from both `oneHop` and `twoHop` (per `GraphTraversal` construction), so its chunks receive no graph boost but **can** receive the tag-shared additive (because a note shares every tag with itself). Is this desired (active note surfaces via cosine plus tag-affinity) or should the active note's own chunks be excluded entirely from RAG results? Verifier to confirm; this slice currently ships the "cosine + tag-shared only" semantics.
