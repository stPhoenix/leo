# F29 — Embeddings + IndexedDB store

## Purpose

Wire the downstream half of the [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) → [F28 chunking-metadata](../chunking-metadata/feature.md) → embed → persist pipeline: after F27's `processPath(path)` seam has produced a `Chunk[]` via F28's pure chunker, this feature calls `EmbeddingClient.embed(chunks.map(c => c.text)): number[][]` against the [ProviderManager](../../../../architecture/architecture.md#34-adapters) embedding model configured per [FR-PROV-08](../../context.md#fr-prov-08) to realise [FR-IDX-08](../../context.md#fr-idx-08), then upserts every `{chunk, vector}` pair together with the `IndexHeader` `{model, dim, version}` record into an IndexedDB database opened at the [Obsidian origin](../../../../standards/tech-stack.md#storage-layout) under the logical namespace `<vault>/.leo/index/` per [NFR-DATA-02](../../context.md#nfr-data-02) to realise [FR-IDX-09](../../context.md#fr-idx-09). On every plugin load the store runs a header + object-store integrity check; any schema-version mismatch, missing object store, dim-count drift, or low-level `IDBDatabase` open error is classified as corruption, surfaces a blocking `Notice` + inline "Index appears corrupted — rebuild?" dialog routed through the [F13](../ui-visual-states-notifications/feature.md) notifications channel, clears the damaged database, and asks [F27](../vault-indexer-dirty-queue/feature.md) to rebuild from a clean slate per [NFR-REL-03](../../context.md#nfr-rel-03). Model-swap reindex orchestration, the dirty queue itself, chunking, RAG retrieval, and the visible settings/status-bar controls all live in sibling features and are explicitly out of scope here.

## Scope

### In scope

- `EmbeddingClient` adapter per [architecture §3.4](../../../../architecture/architecture.md#34-adapters) implementing `embed(texts: string[]): Promise<number[][]>` via a thin `fetch` wrapper over the [LM Studio `/v1/embeddings` endpoint](../../../../standards/tech-stack.md#retrieval-layer) per [FR-IDX-08](../../context.md#fr-idx-08) and [FR-PROV-08](../../context.md#fr-prov-08); consumes the settings-configured embedding model + endpoint URL (the same base URL as chat but a separate `embeddingModel` key), batched request shape `{ model, input: texts }`, response mapped to `data[i].embedding: number[]`, retry + timeout inherited from the ProviderManager [FR-PROV-05](../../context.md#fr-prov-05)/[FR-PROV-06](../../context.md#fr-prov-06) policy, `AbortSignal` passed through from the caller for [FR-IDX-12](../../context.md#fr-idx-12) cancellation friendliness.
- Batched embedding call path per [architecture §5.4 Lazy Indexing](../../../../architecture/architecture.md#54-lazy-indexing): `VaultIndexer` hands `Chunk[]` to `EmbeddingClient.embed(chunk.text[])`, the adapter splits into `EMBED_BATCH_SIZE=32 as const` sub-batches, calls `/v1/embeddings` once per sub-batch, and returns a single ordered `number[][]` preserving input order (assertion: `vectors.length === chunks.length`). All constants are `as const` per [code style — TypeScript](../../../../standards/code-style.md#typescript); batch size is a single source of truth.
- `VectorStore` adapter per [architecture §3.4](../../../../architecture/architecture.md#34-adapters) exposing `upsert(path, chunks, vectors)`, `deleteByPath(path)`, `listHeader() / writeHeader(h)`, and an internal `getAll()` seam [F31](../../features-index.md) will call; persistence through the [`idb` library](../../../../standards/code-style.md#indexeddb-idb) per [FR-IDX-09](../../context.md#fr-idx-09) and [NFR-DATA-02](../../context.md#nfr-data-02).
- IndexedDB schema (`schemaVersion=1 as const`) per [architecture §3.4 VectorStore row](../../../../architecture/architecture.md#34-adapters): database name `leo-index` (per-vault isolation provided by Obsidian's origin scoping — see the [storage-layout note](../../../../standards/tech-stack.md#storage-layout) "IndexedDB lives in Obsidian's origin"); object stores `header` (keyPath `"key"`, single row id `"header"`, value shape `{model: string, dim: number, version: number}`), `vectors` (keyPath `"id"` = `` `${path}#${line_start}-${line_end}` ``, value shape `{id, path, line_start, line_end, heading_path, frontmatter_tags, inline_tags, text, vector: number[]}`), plus a `by-path` index on `vectors.path` to power fast `deleteByPath`. The logical mount point `<vault>/.leo/index/` from [NFR-DATA-02](../../context.md#nfr-data-02) is reflected in the database name and header-key prefix — no filesystem bytes are written under that path because IndexedDB physically lives in the Obsidian origin.
- Upsert flow per `Chunk` row (`VaultIndexer` → `VectorStore.upsert(path, chunks, vectors)`): open one `readwrite` transaction spanning `vectors`, call `put` per `{chunk, vector}` pair keyed by the composite id, flush, report success; `deleteByPath(path)` opens the `by-path` index and `delete`s every hit in one transaction so stale chunks from a pre-edit revision are evicted atomically. Both methods are `async` and return typed `Result<void>` per [code style — Error Handling](../../../../standards/code-style.md#error-handling).
- `IndexHeader` read/write per [FR-IDX-09](../../context.md#fr-idx-09): `listHeader()` returns `{model, dim, version} | null`; `writeHeader({model, dim, version})` overwrites the single `header` row. Header is written once on first embed (when `listHeader() === null`) with `model` = the settings embedding-model id, `dim` = `vectors[0].length` from the first successful batch, `version` = `schemaVersion`. Subsequent writes only happen during the F30-owned reindex flow.
- Corruption detection on load per [NFR-REL-03](../../context.md#nfr-rel-03): on `Plugin.onload`, `VectorStore.verify()` opens the database and runs five invariant checks — (1) database opens without `IDBDatabase.onerror`, (2) both `header` and `vectors` object stores exist, (3) `header.version === schemaVersion`, (4) `header.dim === listHeader().dim` matches the dim of a sampled `vectors` row (when any exists), (5) sampled `vectors` row passes Zod-schema validation per [code style — Zod](../../../../standards/code-style.md#zod--tool-schemas). Any failure → `CorruptIndexError` with a `reason` tag; no auto-silent recovery.
- Corruption-recovery UX per [NFR-REL-03](../../context.md#nfr-rel-03) routed through the [F13](../ui-visual-states-notifications/feature.md) notifications channel: emit a persistent status-bar warning + one `Notice` + one inline dialog "Index appears corrupted — rebuild?" with `[Rebuild now] [Later]`; Rebuild deletes the database via `indexedDB.deleteDatabase("leo-index")`, re-creates the empty schema, and signals [F27](../vault-indexer-dirty-queue/feature.md) to enqueue the whole vault. Later keeps the database closed and disables RAG (`RAGEngine.available = false`) until the user re-triggers rebuild from the UI feature ([F30](../../features-index.md)). UI copy, exact visual state, and settings affordances are owned by [F30](../../features-index.md); this feature provides the API surface only.
- Structured log events per [NFR-LOG-04](../../context.md#nfr-log-04) via the [F01 Logger](../plugin-bootstrap-logging/feature.md): `index.embed.batch`, `index.embed.failure`, `index.store.upsert`, `index.store.header.write`, `index.store.verify.pass`, `index.store.verify.fail` (with `reason`), `index.store.corruption.rebuild`; chunk text and vector payloads never leak above `debug` per [NFR-LOG-01](../../context.md#nfr-log-01).
- Vitest coverage per [NFR-TEST-01](../../context.md#nfr-test-01) and [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw): `msw` fixture server for `/v1/embeddings` covering batch-split, ordered return, retry-on-5xx; `fake-indexeddb` for `VectorStore.upsert` + `deleteByPath` + composite-key collision on re-upsert; header write/read round-trip; `verify()` corruption-matrix (missing store, schema-version mismatch, dim mismatch, Zod shape failure) each producing a distinct tagged `CorruptIndexError`; header + store survive plugin reload simulation.
- Seam for [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md): this feature populates the two adapter singletons behind F27's `processPath` drain tick (embed + upsert happen once per path after F28 returns `Chunk[]`); pre-existing chunks for the same path are cleared via `deleteByPath(path)` before the upsert so renames/edits never leave orphan rows per [architecture §5.4](../../../../architecture/architecture.md#54-lazy-indexing).

- Query embedding, cosine search, top-K, overlap-merge, graph boosts, tag filters, and the `search_vault` tool per [FR-RAG-01](../../context.md#fr-rag-01)–[FR-RAG-08](../../context.md#fr-rag-08) → ships with [F31 rag-cosine-search](../../features-index.md), F32, F33, F35.

## Acceptance criteria

1. `EmbeddingClient.embed(texts: string[]): Promise<number[][]>` calls the [LM Studio `/v1/embeddings`](../../../../standards/tech-stack.md#retrieval-layer) endpoint with `{model: settings.embeddingModel, input: batch}`, splits input into `EMBED_BATCH_SIZE=32` sub-batches, returns vectors in input order with `vectors.length === texts.length`, and propagates an `AbortSignal` into the underlying `fetch`. ([FR-IDX-08](../../context.md#fr-idx-08))
2. `VectorStore.upsert(path, chunks, vectors)` opens a single `readwrite` IndexedDB transaction, first calls `deleteByPath(path)` through the `by-path` index, then `put`s one row per `{chunk, vector}` pair into the `vectors` store keyed by `` `${path}#${line_start}-${line_end}` ``, under database `leo-index` with `schemaVersion=1` per [NFR-DATA-02](../../context.md#nfr-data-02). ([FR-IDX-09](../../context.md#fr-idx-09), [NFR-DATA-02](../../context.md#nfr-data-02))
3. On the first successful embed of a fresh database, `VectorStore.writeHeader({model, dim, version})` persists `{model: settings.embeddingModel, dim: vectors[0].length, version: schemaVersion}` into the single `header` row; `listHeader()` returns that exact record on subsequent reads. ([FR-IDX-09](../../context.md#fr-idx-09))
4. On `Plugin.onload`, `VectorStore.verify()` returns `ok` when the database opens, both object stores exist, `header.version === schemaVersion`, `header.dim` matches a sampled row's `vector.length`, and that row passes Zod validation; any other outcome resolves to `CorruptIndexError` with a reason tag (`open-failed | missing-store | version-mismatch | dim-mismatch | shape-invalid`). ([NFR-REL-03](../../context.md#nfr-rel-03))
5. On `CorruptIndexError`, the store fires a `corruption` event routed through the [F13](../ui-visual-states-notifications/feature.md) notifications channel, disables RAG (`available = false`) until rebuild, and exposes `rebuild()` that calls `indexedDB.deleteDatabase("leo-index")`, re-creates the empty schema, re-writes an empty header slot, and returns so [F27](../vault-indexer-dirty-queue/feature.md) can re-enqueue the vault. ([NFR-REL-03](../../context.md#nfr-rel-03))
6. Structured log events `index.embed.batch`, `index.embed.failure`, `index.store.upsert`, `index.store.header.write`, `index.store.verify.pass`, `index.store.verify.fail`, `index.store.corruption.rebuild` are emitted via the [F01 Logger](../plugin-bootstrap-logging/feature.md) with key/value fields (`path`, `batchSize`, `reason`, etc.), and chunk text + vector bytes never log above `debug` per [NFR-LOG-04](../../context.md#nfr-log-04). ([FR-IDX-08](../../context.md#fr-idx-08), [FR-IDX-09](../../context.md#fr-idx-09))
7. Vitest suite per [NFR-TEST-01](../../context.md#nfr-test-01) covers `EmbeddingClient` batch-split + ordered return + retry against an `msw` `/v1/embeddings` fixture, `VectorStore.upsert` + `deleteByPath` + composite-key collision round-trip against `fake-indexeddb`, header write/read, and the five corruption-matrix paths (`open-failed`, `missing-store`, `version-mismatch`, `dim-mismatch`, `shape-invalid`) each producing a distinct `CorruptIndexError.reason`. ([FR-IDX-08](../../context.md#fr-idx-08), [FR-IDX-09](../../context.md#fr-idx-09), [NFR-REL-03](../../context.md#nfr-rel-03))

## Open questions

- **`EMBED_BATCH_SIZE` exact value** — 32 texts per `/v1/embeddings` call balances latency vs throughput on typical LM Studio servers; SRS does not pin it. Verifier to confirm or move to a settings knob.
- **Header-mismatch (model swap) responsibility split** — [FR-IDX-01](../../context.md#fr-idx-01) "now / later / revert model" UI + settings wiring is owned by [F30](../../features-index.md); this feature detects the mismatch via `listHeader()` during `verify()`. Verifier to confirm the dividing line so F30 can own the reindex prompt without this feature re-implementing it.
- **Per-vault DB isolation** — IndexedDB is scoped by Obsidian origin, which is shared across vaults on a single install. Proposal: namespace the DB name as `` `leo-index:${app.appId}` `` if `app.appId` is stable; verifier to confirm the Obsidian API guarantee.
