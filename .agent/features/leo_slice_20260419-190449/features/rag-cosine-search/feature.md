# F31 — RAG cosine search + top-K

## Purpose

Stand up the first end-to-end RAG retrieval path on top of the [F29 embeddings-indexeddb-store](../embeddings-indexeddb-store/feature.md) `VectorStore` + `EmbeddingClient` seams so that [`AgentRunner.query`](../../../../architecture/architecture.md#32-agent-layer) can ask a new `RAGEngine.query(text, {k?}): RAGHit[]` for the top-K most similar chunks of any user message. The engine embeds the query through the same [LM Studio `/v1/embeddings`](../../../../standards/tech-stack.md#retrieval-layer) adapter, scans the persisted `vectors` rows with the [pure `Scorer` cosine similarity math](../../../../architecture/architecture.md#33-domain--core-pure) (linear-scan v1 per [architecture §3.4 VectorStore row](../../../../architecture/architecture.md#34-adapters)), returns `{path, line_start, line_end, score}` tuples sorted desc and truncated at `K=10` per [FR-RAG-01](../../context.md#fr-rag-) and [FR-RAG-06](../../context.md#fr-rag-), and merges same-file adjacent or overlapping hits into a single widened range per [FR-RAG-07](../../context.md#fr-rag-) so that `ContextAssembler` consumers do not duplicate prose. The linear-scan performance envelope is deliberately budgeted against [NFR-PERF-03](../../context.md#nfr-perf-03) (≤200 ms on a 10 k-note vault) using the one-shot `VectorStore.getAll()` seam already exposed by [F29](../embeddings-indexeddb-store/feature.md), validated per [architecture §5.2 Chat Turn](../../../../architecture/architecture.md#52-chat-turn-no-tools) which places RAG on the hot path. Exclude-list filtering, tag filters, the `search_vault` tool, graph boosts, and any HNSW / worker-thread perf polish all ship in sibling features and are explicitly out of scope here.

## Scope

### In scope

- `RAGEngine.query(text: string, opts?: {k?: number; signal?: AbortSignal}): Promise<RAGHit[]>` per [architecture §11 Mapping SRS FR → Modules](../../../../architecture/architecture.md#11-mapping-srs-fr--modules) with `RAGHit = {path: string; line_start: number; line_end: number; score: number}` per [FR-RAG-06](../../context.md#fr-rag-); default `K = DEFAULT_TOP_K = 10 as const` per [code style — TypeScript](../../../../standards/code-style.md#typescript); mounted at `src/rag/RAGEngine.ts` per [architecture §9 Project File Layout](../../../../architecture/architecture.md#9-project-file-layout-proposed).
- Query-embedding path per [FR-RAG-01](../../context.md#fr-rag-) and [architecture §5.2 Chat Turn](../../../../architecture/architecture.md#52-chat-turn-no-tools): delegates to the [F29 `EmbeddingClient.embed([text])`](../embeddings-indexeddb-store/feature.md) single-item batch and unwraps `vectors[0]`; passes the caller's `AbortSignal` through per [architecture §10 Concurrency & Lifecycle Rules](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules); asserts `queryVector.length === (await store.listHeader()).dim` before scoring (dim drift → `RAGUnavailableError` with `reason: "header-mismatch"`).
- Linear-scan cosine similarity per [architecture §3.3 Domain / Core — Scorer](../../../../architecture/architecture.md#33-domain--core-pure) and [FR-RAG-01](../../context.md#fr-rag-): pulls every row in one shot via the internal [`VectorStore.getAll()`](../embeddings-indexeddb-store/feature.md) seam already reserved for F31, computes `score = Scorer.cosine(queryVector, row.vector)` as a pure function over `number[]` pairs; no in-place mutation of row state; empty store returns `[]` immediately.
- Top-K selection per [FR-RAG-06](../../context.md#fr-rag-): maintains a size-`k` min-heap (insertion-ordered tiebreak) while iterating `getAll()` so the final sort cost is `O(N log K)` instead of `O(N log N)`; output is sorted by `score` desc, then stable by original `(path, line_start)` for deterministic Vitest fixtures per [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw).
- Same-file overlap merge per [FR-RAG-07](../../context.md#fr-rag-): after top-K selection, group hits by `path`, sort each group by `line_start` asc, and merge any two consecutive chunks whose ranges overlap **or abut** (`b.line_start <= a.line_end + 1`); the merged hit takes `line_start = min`, `line_end = max`, and `score = max(a.score, b.score)` (no averaging — highest-similarity chunk wins per [FR-RAG-06](../../context.md#fr-rag-)'s "return … with score" contract); post-merge the list is re-sorted by `score` desc and **not** re-truncated to K (merged ranges only reduce count).
- Result payload per [FR-RAG-06](../../context.md#fr-rag-): strictly `{path, line_start, line_end, score}` — no `text`, no `chunkId`, no `heading_path`; downstream [ContextAssembler](../../../../architecture/architecture.md#33-domain--core-pure) re-reads prose via [VaultAdapter](../../../../architecture/architecture.md#34-adapters), keeping `RAGHit` narrow and PII-light for [NFR-LOG-01](../../context.md#nfr-log-01) logging.
- Unavailable-state handling per [NFR-REL-03](../../context.md#nfr-rel-03) (inherited from F29): when `VectorStore.available === false` (post-corruption + pre-rebuild) `RAGEngine.query` resolves to `[]` with a single `rag.query.unavailable` log event per [F01 Logger](../plugin-bootstrap-logging/feature.md); callers treat "no results" identically to a healthy empty store so the chat turn in [architecture §5.2](../../../../architecture/architecture.md#52-chat-turn-no-tools) never fails closed.
- Performance envelope per [NFR-PERF-03](../../context.md#nfr-perf-03): the linear-scan + heap path must complete in ≤200 ms on a `fake-indexeddb` fixture seeded with 10 000 rows × 1024-dim `Float32`-backed `number[]`; a Vitest bench asserts the budget and logs `rag.query.ms` per [NFR-LOG-04](../../context.md#nfr-log-04); deeper profiling / `hnswlib-wasm` swap explicitly deferred per [architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points).
- Structured log events per [NFR-LOG-04](../../context.md#nfr-log-04) via the [F01 Logger](../plugin-bootstrap-logging/feature.md): `rag.query.start` (`{qLen, k}`), `rag.query.embed.ms`, `rag.query.scan.ms` (`{rows}`), `rag.query.result` (`{hits, topScore}`), `rag.query.unavailable`, `rag.query.merge` (`{before, after}`); query text and chunk bodies never log above `debug` per [NFR-LOG-01](../../context.md#nfr-log-01).
- Vitest coverage per [NFR-TEST-01](../../context.md#nfr-test-01) and [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw): `msw` fixture for `/v1/embeddings` single-item call; `fake-indexeddb` seeded via [F29](../embeddings-indexeddb-store/feature.md)'s `upsert` path with 10 k rows for the perf bench; pure-math tests on `Scorer.cosine` (colinear = 1, orthogonal = 0, zero-vector guard); top-K correctness vs a reference `Array.sort` implementation; overlap-merge matrix (disjoint, touching, overlapping, identical, three-way chain); `AbortSignal` propagation aborts mid-scan; dim-mismatch → `RAGUnavailableError`.

### Out of scope

- Exclude-list glob filtering over `path` prior to scoring and its indexing-time counterpart per [FR-RAG-08](../../context.md#fr-rag-) → ships with [F32 rag-exclude-list](../../features-index.md).
- Optional tag filter (frontmatter + inline) at query time and the `search_vault(query, tags?)` tool registration per [FR-RAG-05](../../context.md#fr-rag-) + [FR-AGENT-02](../../context.md#fr-agent-02) → ships with [F33 rag-tag-filter-search-vault](../../features-index.md).
- 1-hop (1.5×) / 2-hop (1.2×) graph boosts and tag-shared (1.1×) additive boost per [FR-RAG-02](../../context.md#fr-rag-)–[FR-RAG-04](../../context.md#fr-rag-) → ships with [F35 graphrag-boosts](../../features-index.md); `Scorer` in this feature only computes raw cosine.
- `hnswlib-wasm` swap, worker-thread offloading, 1 M-row scale, and profiling-driven tuning per [architecture §8 Extension Points](../../../../architecture/architecture.md#8-extension-points) → ships with [F50 perf-scale-10k-vault](../../features-index.md); this feature only proves the v1 linear-scan budget.

## Acceptance criteria

1. `RAGEngine.query(text, {k})` embeds the query via [F29 `EmbeddingClient`](../embeddings-indexeddb-store/feature.md) (`vectors[0]` of a single-item batch), scans every row returned by `VectorStore.getAll()`, and returns at most `k` (default `DEFAULT_TOP_K=10`) `{path, line_start, line_end, score}` tuples sorted by `score` desc. ([FR-RAG-01](../../context.md#fr-rag-), [FR-RAG-06](../../context.md#fr-rag-))
2. Cosine similarity is computed by the pure `Scorer.cosine(a: number[], b: number[]): number` function, returns `1` for colinear vectors, `0` for orthogonal, and is guarded against zero-vector inputs (returns `0`, never `NaN`). ([FR-RAG-01](../../context.md#fr-rag-))
3. `RAGEngine.query(text)` with `k` unset uses `DEFAULT_TOP_K=10`; the result shape is exactly `{path, line_start, line_end, score}` — any additional keys are a regression. ([FR-RAG-06](../../context.md#fr-rag-))
4. Same-file overlap merge: given two top-K hits on the same `path` whose ranges overlap or abut (`b.line_start <= a.line_end + 1`), the engine emits a single merged hit with `line_start = min`, `line_end = max`, `score = max(a.score, b.score)`, and the final list is re-sorted by `score` desc. ([FR-RAG-07](../../context.md#fr-rag-))
5. On a `fake-indexeddb` fixture seeded with 10 000 × 1024-dim rows, `RAGEngine.query(sample)` completes in ≤200 ms (wall clock via `performance.now()`) and emits a `rag.query.ms` log event. ([NFR-PERF-03](../../context.md#nfr-perf-03))
6. When `VectorStore.available === false` or `store.getAll()` returns `[]`, `RAGEngine.query` resolves to `[]` without throwing; when the embedded query dim `!==` `listHeader().dim` it resolves to `[]` and emits one `rag.query.unavailable` with `reason: "header-mismatch"` log event through the [F01 Logger](../plugin-bootstrap-logging/feature.md). ([FR-RAG-01](../../context.md#fr-rag-))
7. The caller-supplied `AbortSignal` is wired into both the embedding `fetch` and the scan loop; aborting mid-scan rejects the returned promise with the signal's `reason` (or a default `AbortError`) within one `requestIdleCallback` tick. ([NFR-PERF-03](../../context.md#nfr-perf-03))
8. Vitest suite covers: `Scorer.cosine` pure-math cases; top-K correctness vs a reference `Array.sort` on 1 000 random vectors; the overlap-merge matrix (disjoint / touching / overlapping / identical / three-way chain); the 10 k-row perf bench asserting the ≤200 ms budget; `AbortSignal` propagation; the `header-mismatch` unavailable path. ([FR-RAG-01](../../context.md#fr-rag-), [FR-RAG-06](../../context.md#fr-rag-), [FR-RAG-07](../../context.md#fr-rag-), [NFR-PERF-03](../../context.md#nfr-perf-03))

## Open questions

- **Abut-vs-overlap boundary (`+1` gap merge)** — SRS [FR-RAG-07](../../context.md#fr-rag-) says "merge overlapping" but notes chunked by heading may share a boundary line; this feature merges on `b.line_start <= a.line_end + 1`. Verifier to confirm the single-line gap is desired or tighten to strict `<=` overlap.
- **Merged-hit score aggregation** — `max(a.score, b.score)` keeps the strongest signal per [FR-RAG-06](../../context.md#fr-rag-), but an argument exists for `sum` (rewards multi-chunk relevance) or `avg` (normalises size). Verifier to pick one for v1 so [F35](../../features-index.md) boosts compose predictably.
- **10 k-row perf bench stability in CI** — `performance.now()` on `fake-indexeddb` + Vitest may be noisy on shared runners; flake risk for the ≤200 ms gate per [NFR-PERF-03](../../context.md#nfr-perf-03). Verifier to decide between a generous CI-only `2×` budget or moving the bench to a dedicated `pnpm bench` target outside the unit suite.
