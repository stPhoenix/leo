# F44 — PTL retry with group head-truncation

## Purpose

Wrap [F43 autocompaction](../compaction-autocompact/feature.md)'s summarization call in the prompt-too-long (PTL) retry loop specified at [compact.md §13](../../../../srs/compact.md#13-prompt-too-long-recovery): when the streamed summary starts with the `PROMPT_TOO_LONG_ERROR_MESSAGE` sentinel, Leo groups the pre-summary messages by API round per [compact.md §14](../../../../srs/compact.md#14-message-grouping), drops the oldest `dropCount` groups, re-issues the summarization call, and repeats up to `MAX_PTL_RETRIES = 3` before giving up with `ERROR_MESSAGE_PROMPT_TOO_LONG`. `dropCount` resolves to the parsed token gap from the PTL error (drop groups until accumulated tokens ≥ gap) or — when the gap is unparseable — 20% of the current groups (minimum 1, never all). If the truncated head starts with an assistant message, a synthetic `"[earlier conversation truncated for compaction retry]"` user marker (`isMeta: true`) is prepended to honour [compact.md §15 "Role Alternation"](../../../../srs/compact.md#15-api-invariant-preservation). This slice satisfies the PTL-recovery binding at [FR-COMPACT-06](../../context.md#fr-compact-06) and the ≤3 PTL retry latency NFR at [NFR-PERF-07](../../context.md#nfr-perf-07).

## Scope

### In scope

- `truncateHeadForPTLRetry(messages, ptlResponse)` pure function per [compact.md §13 "Algorithm"](../../../../srs/compact.md#13-prompt-too-long-recovery): strips any prior synthetic retry marker, groups via `groupMessagesByApiRound` from [F43](../compaction-autocompact/feature.md) (imported, not re-implemented), returns `null` when fewer than 2 groups remain, computes `dropCount`, slices the head, and prepends the synthetic user marker when required.
- Token-gap parser that extracts the numeric gap from the PTL error text (regex match per [compact.md §13 "Algorithm" step 4](../../../../srs/compact.md#13-prompt-too-long-recovery)); returns `null` on failure so the caller falls back to the 20% rule.
- 20% fallback computation (`dropCount = max(1, floor(0.20 * groups.length))`) with the invariant `dropCount < groups.length` per [compact.md §13 "Algorithm" step 4](../../../../srs/compact.md#13-prompt-too-long-recovery).
- Synthetic marker constant `PTL_TRUNCATION_MARKER = "[earlier conversation truncated for compaction retry]"` attached as a user message with `isMeta: true` only when the post-slice head's first message has role `assistant` per [compact.md §13 "Algorithm" step 6](../../../../srs/compact.md#13-prompt-too-long-recovery); an existing marker from a prior attempt is stripped first so consecutive retries do not stack markers.
- Retry loop in `autoCompactIfNeeded` per [compact.md §13 "Retry Loop"](../../../../srs/compact.md#13-prompt-too-long-recovery): detect PTL via `response.startsWith(PROMPT_TOO_LONG_ERROR_MESSAGE)`, increment `ptlAttempts`, throw `ERROR_MESSAGE_PROMPT_TOO_LONG` on `ptlAttempts > MAX_PTL_RETRIES` or when `truncateHeadForPTLRetry` returns `null`, else re-issue the summarization call with the truncated head and loop.
- `MAX_PTL_RETRIES = 3` constant binding from [compact.md §3 "Retry and Streaming"](../../../../srs/compact.md#3-constants-and-thresholds).
- `ERROR_MESSAGE_PROMPT_TOO_LONG` and `PROMPT_TOO_LONG_ERROR_MESSAGE` string constants sourced verbatim from [compact.md §20](../../../../srs/compact.md#20-error-handling); a Vitest snapshot pins the `ERROR_MESSAGE_PROMPT_TOO_LONG` text to the spec.
- `tengu_compact_ptl_retry` telemetry event per [compact.md §17](../../../../srs/compact.md#17-telemetry-events) emitted on every truncation attempt with `{attempt, droppedMessages, remainingMessages}` via [F01 Logger](../plugin-bootstrap-logging/feature.md).
- `tengu_compact_failed {reason: 'prompt_too_long', preCompactTokenCount}` emitted when retries are exhausted or truncation returns `null`, per [compact.md §17](../../../../srs/compact.md#17-telemetry-events).
- API-invariant preservation across truncation per [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation): because grouping is on API-round boundaries, `tool_use` ↔ `tool_result` pairs within a kept group never split; dangling pairs that would arise from a prior resume are handled by [F42](../compaction-microcompact/feature.md)'s existing pairing checker before the retry re-enters the stream.
- `cacheSafeParams.forkContextMessages` update on every retry per [compact.md §13 "Retry Loop"](../../../../srs/compact.md#13-prompt-too-long-recovery) so subsequent cache-sharing attempts (future [F38](../../features-index.md) cloud providers) see the truncated head.
- AbortController propagation: cancelling the turn-scoped controller from [F10](../agent-controller-core/feature.md) stops any in-flight PTL retry immediately; the loop checks `signal.aborted` before each re-issue and exits with `null`.
- Vitest coverage: token-gap parsing (parseable / unparseable), 20% fallback with 1/5/10/99 groups, marker prepend when head starts with assistant, marker strip when a prior marker exists, `null` return on `groups.length < 2`, three-retry boundary (succeed on attempt 3, throw on attempt 4), abort-mid-retry, `cacheSafeParams.forkContextMessages` update assertion, and a msw-driven end-to-end fixture with two consecutive PTL responses followed by a valid summary.

### Out of scope

- Message grouping itself (`groupMessagesByApiRound`) — owned by [F43](../compaction-autocompact/feature.md) and [compact.md §14](../../../../srs/compact.md#14-message-grouping); this slice imports it.
- Streaming retry counter (`MAX_COMPACT_STREAMING_RETRIES = 2`) — a distinct retry axis owned by [F43](../compaction-autocompact/feature.md) per [compact.md §7 Phase 3 Path B "Streaming retry"](../../../../srs/compact.md#7-layer-3-full-conversation-compaction); PTL retries and streaming retries do not share a counter.
- Circuit breaker (3 consecutive failure disable) — deferred to [F45](../../features-index.md) per [NFR-REL-06](../../context.md#nfr-rel-) and [compact.md §6 "Auto-Compact Execution" step 2](../../../../srs/compact.md#6-layer-2-autocompaction); this slice's exhausted-retries path feeds F45's counter by emitting `tengu_compact_failed` but does not flip the breaker itself.
- Partial compaction's own PTL loop per [compact.md §9 step 4](../../../../srs/compact.md#9-partial-compaction) — partial compaction is out of scope for v1 per [context.md Out of scope](../../context.md#out-of-scope).
- PTL recovery for the main user turn (non-compaction path) — [FR-AGENT-08](../../context.md#fr-agent-) pre-compaction fallback (oldest history → RAG → skill examples) is owned by [F10](../agent-controller-core/feature.md); see [context.md Open questions](../../context.md#open-questions) "PTL error in the main chat path vs compaction" — this slice covers the compaction call only.
- Parsing the full provider error envelope (status codes, nested JSON) — only the summary text's leading sentinel is inspected, matching [compact.md §13 "Retry Loop"](../../../../srs/compact.md#13-prompt-too-long-recovery).
- User-visible surface when retries exhaust — the `ERROR_MESSAGE_PROMPT_TOO_LONG` string is thrown upward; rendering it as a Notice / inline error lives with [F13 ui-visual-states-notifications](../ui-visual-states-notifications/feature.md).

## Acceptance criteria

1. `truncateHeadForPTLRetry(messages, ptlResponse)` returns `null` iff `groupMessagesByApiRound(messages).length < 2` after the prior-marker strip; Vitest asserts the boundary at 1 group (null) and 2 groups (non-null). ([FR-COMPACT-06](../../context.md#fr-compact-06), [compact.md §13 "Algorithm" step 3](../../../../srs/compact.md#13-prompt-too-long-recovery))
2. Prior-attempt synthetic markers are stripped before grouping: if `messages[0]` equals the `PTL_TRUNCATION_MARKER` user message, it is removed before `groupMessagesByApiRound` runs. Vitest confirms consecutive retries do not accumulate markers. ([FR-COMPACT-06](../../context.md#fr-compact-06), [compact.md §13 "Algorithm" step 1](../../../../srs/compact.md#13-prompt-too-long-recovery))
3. `dropCount` derives from the parseable token gap in `ptlResponse` when present: groups are accumulated from the head until the running token sum (via [F41 tokenCountWithEstimation](../token-estimator-3tier/feature.md)) reaches or exceeds the gap, and that accumulated group count becomes `dropCount`. Vitest asserts a fixture with a 15,000-token gap drops exactly the groups required to cross it. ([FR-COMPACT-06](../../context.md#fr-compact-06), [compact.md §13 "Algorithm" step 4](../../../../srs/compact.md#13-prompt-too-long-recovery))
4. When the token gap is unparseable, `dropCount = max(1, floor(0.20 * groups.length))`, clamped such that `dropCount < groups.length` so at least one group is always retained. Vitest matrix: `{1→1 not reached (null path), 2→1, 5→1, 10→2, 99→19, 100→20}`. ([FR-COMPACT-06](../../context.md#fr-compact-06), [compact.md §13 "Algorithm" step 4](../../../../srs/compact.md#13-prompt-too-long-recovery))
5. After slicing, if the first remaining message has role `assistant`, a synthetic user message with content `"[earlier conversation truncated for compaction retry]"` and `isMeta: true` is prepended so the outgoing head starts with a user turn per [compact.md §15 "Role Alternation"](../../../../srs/compact.md#15-api-invariant-preservation); otherwise no marker is added. Vitest covers both branches. ([FR-COMPACT-06](../../context.md#fr-compact-06), [FR-COMPACT-07](../../context.md#fr-compact-07), [compact.md §13 "Algorithm" step 6](../../../../srs/compact.md#13-prompt-too-long-recovery))
6. The retry loop in `autoCompactIfNeeded` detects PTL via `summary.startsWith(PROMPT_TOO_LONG_ERROR_MESSAGE)`, runs `truncateHeadForPTLRetry`, updates `messagesToSummarize` and `cacheSafeParams.forkContextMessages` with the truncated result, and re-issues the summarization stream. Vitest + `msw` feeds two PTL responses then a valid summary and asserts three total stream invocations with strictly shrinking message counts. ([FR-COMPACT-06](../../context.md#fr-compact-06), [compact.md §13 "Retry Loop"](../../../../srs/compact.md#13-prompt-too-long-recovery))
7. When `ptlAttempts > MAX_PTL_RETRIES = 3` or `truncateHeadForPTLRetry` returns `null`, the loop throws `ERROR_MESSAGE_PROMPT_TOO_LONG` verbatim from [compact.md §20](../../../../srs/compact.md#20-error-handling) and emits `tengu_compact_failed {reason: 'prompt_too_long', preCompactTokenCount}`. Vitest asserts exactly three retries occur before the throw on a four-PTL fixture, and that a two-group fixture throws on the first unparseable-gap call without retrying. ([FR-COMPACT-06](../../context.md#fr-compact-06), [NFR-PERF-07](../../context.md#nfr-perf-07), [compact.md §17 `tengu_compact_failed`](../../../../srs/compact.md#17-telemetry-events), [compact.md §20](../../../../srs/compact.md#20-error-handling))
8. Every truncation attempt emits `tengu_compact_ptl_retry {attempt, droppedMessages, remainingMessages}` per [compact.md §17 `tengu_compact_ptl_retry`](../../../../srs/compact.md#17-telemetry-events) through [F01 Logger](../plugin-bootstrap-logging/feature.md); Vitest asserts `attempt` starts at 1, increments monotonically, and `droppedMessages + remainingMessages` equals the pre-truncation total. ([FR-COMPACT-06](../../context.md#fr-compact-06), [compact.md §17](../../../../srs/compact.md#17-telemetry-events))
9. Abort signalling: when the turn-scoped `AbortController` from [F10](../agent-controller-core/feature.md) fires between retries, the loop exits with `null` without issuing further stream calls and without incrementing `ptlAttempts` beyond the current value. Vitest with fake timers asserts zero additional stream invocations after abort. ([NFR-PERF-07](../../context.md#nfr-perf-07), [compact.md §20 "User Abort Handling"](../../../../srs/compact.md#20-error-handling))
10. API invariants from [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation) hold across every retry's truncated head: `tool_use` / `tool_result` pairs never split (grouping is on API-round boundaries), streaming-chunk adjacency is preserved within a kept group, and the first outgoing message has role `user` (either naturally or via the synthetic marker). A shared invariant checker (reused from [F42](../compaction-microcompact/feature.md)) runs in Vitest on every intermediate `messagesToSummarize`. ([FR-COMPACT-06](../../context.md#fr-compact-06), [FR-COMPACT-07](../../context.md#fr-compact-07), [compact.md §15](../../../../srs/compact.md#15-api-invariant-preservation))

## Open questions

- **Token-gap parse regex**: [compact.md §13 "Algorithm" step 4](../../../../srs/compact.md#13-prompt-too-long-recovery) says "if token gap is parseable from error" but does not pin the error-text shape. LM Studio's OpenAI-compatible error envelope varies; Leo must either define a regex list (OpenAI shape, Anthropic shape, Ollama shape) or always fall back to the 20% rule for v1. Proposing always-20% for v1 since the only shipping provider is LM Studio; revisit with [F38](../../features-index.md) when cloud providers land.
- **`cacheSafeParams.forkContextMessages` in Path-B-only v1**: [F43](../compaction-autocompact/feature.md) skips the cache-sharing Path A, so `cacheSafeParams` is not consumed by the stream call in v1. Should the forkContextMessages update still run on every retry (future-proofing) or be skipped? Proposing keep the update — it is a cheap field assignment and keeps the contract aligned with [compact.md §13 "Retry Loop"](../../../../srs/compact.md#13-prompt-too-long-recovery).
- **`isMeta: true` visibility in transcript**: the synthetic marker uses `isMeta: true` so it does not render in the chat transcript, but [FR-CHAT-08](../../context.md#fr-chat-08) conversation persistence has no defined `isMeta` handling (see [context.md Open questions](../../context.md#open-questions) "Conversation storage format"). Does the marker get persisted, or stripped before save? Proposing strip-before-save, consistent with the marker being a per-retry transient.
- **Three-retry ceiling on tiny conversations**: a conversation with exactly 2 groups hits `groups.length < 2` after one truncation, so the retry count never reaches 3. Is that the intended behaviour, or should the loop attempt a forced single-group retry? Proposing intended — matches [compact.md §13 "Algorithm" step 3](../../../../srs/compact.md#13-prompt-too-long-recovery)'s strict `< 2` guard.
- **Interaction with [F45](../../features-index.md) circuit breaker**: this slice emits `tengu_compact_failed {reason: 'prompt_too_long'}`; F45 increments its counter on any `tengu_compact_failed`. A chronic PTL-exhaust pattern will therefore trip the breaker after three consecutive compactions, disabling autocompact — acceptable for v1 or should the PTL reason be excluded from the breaker trigger? Proposing include (matches upstream behaviour at [compact.md §6 "Auto-Compact Execution" step 2](../../../../srs/compact.md#6-layer-2-autocompaction)).
