# F47 — `/context` command + category breakdown + grid

## Purpose

Deliver the user-visible surface of `/context`: register the in-chat slash command and the `Leo: Show context` command-palette entry, invoke the [F46 pipeline](../context-analyzer-pipeline/feature.md) to produce `ContextData`, then render the two-column layout from [context.md §10](../../../../srs/context.md#10-interactive-ui-layout) — responsive grid on the left (5×5 / 10×10 / 5×10 / 20×10 per [§9.1](../../../../srs/context.md#91-grid-dimensions)) with partial-square fullness per [§9.3](../../../../srs/context.md#93-partial-square-fullness), and the fixed-order category breakdown on the right per [§8](../../../../srs/context.md#8-category-definitions--ordering) with deferred categories listed but excluded from the grid and percentage. Satisfies [FR-CTX-01](../../context.md#fr-ctx-01), [FR-CTX-03](../../context.md#fr-ctx-03), [FR-CTX-04](../../context.md#fr-ctx-04), and [NFR-TEST-08](../../context.md#nfr-test-08). Suggestion rendering, status-line integration, and token warnings are carried by the downstream F48; this feature owns only the command-registration path, category-ordering rendering, and grid allocation/visualization.

## Scope

### In scope

- In-chat slash-command registration for `/context` in the composer from [F06](../chat-composer-input/feature.md): exact command name `context`, no aliases, no arguments, no keybindings per [context.md §2](../../../../srs/context.md#2-command-registration); dispatches the interactive variant when the ChatView is visible.
- Obsidian command-palette entry `Leo: Show context` via `Plugin.addCommand({ id: "leo-show-context", name: "Leo: Show context", callback })`, converging on the same handler as the slash command so both paths render the same grid/breakdown UI (resolves [context.md Open Q §FR-UI-04](../../context.md#open-questions)).
- Wiring the command handler to the [F46](../context-analyzer-pipeline/feature.md) pipeline: caller creates a turn-scoped `AbortController`, awaits `analyzeContextUsage(...)`, feeds the resulting `ContextData` to the renderer, disposes the controller on unmount per [Architecture §10](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules).
- Category-breakdown renderer producing the fixed ordering from [context.md §8](../../../../srs/context.md#8-category-definitions--ordering): System prompt → System tools → MCP tools → MCP tools (deferred) → System tools (deferred) → Custom agents → Memory files → Skills → Messages → Autocompact/Manual compact buffer → Free space; deferred rows carry the `isDeferred: true` visual treatment (dimmed, no grid contribution) per [§8 Deferred Categories](../../../../srs/context.md#8-category-definitions--ordering) while still appearing in the list.
- Responsive grid selector mapping `{contextWindow < 1e6 | ≥ 1e6} × {panelWidth < 80ch | ≥ 80ch}` to `{25, 100, 50, 200}` squares per [context.md §9.1](../../../../srs/context.md#91-grid-dimensions). Panel width is measured in characters via the ChatView container's `clientWidth` divided by the rendered monospace glyph advance (resolved through a one-time offscreen-measure helper, re-run on `ResizeObserver` notifications from [F04](../chat-sidebar-view/feature.md)) — the `<80`/`≥80` threshold itself is adapted from [context.md §9.1](../../../../srs/context.md#91-grid-dimensions) "Screen Width" column (resolves [context.md Open Q §FR-CTX-04](../../context.md#open-questions)).
- Square allocation per [context.md §9.2](../../../../srs/context.md#92-square-allocation): `exactSquares = (category.tokens / contextWindow) * TOTAL_SQUARES`; `allocatedSquares = max(1, round(exactSquares))` for non-free, non-deferred categories; free space uses `round(exactSquares)` (can be 0); deferred categories allocate zero squares.
- Partial-square fullness per [context.md §9.3](../../../../srs/context.md#93-partial-square-fullness): `wholeSquares = floor(exactSquares)`, `fractionalPart = exactSquares - wholeSquares`; square `i < wholeSquares` → fullness 1.0; square `i == wholeSquares` → fullness `fractionalPart`; consumed by the visual-symbol gate (≥70% → `◉`, <70% → `◐`) per [§9.5](../../../../srs/context.md#95-visual-symbols).
- Square rendering order per [context.md §9.4](../../../../srs/context.md#94-square-rendering-order): non-reserved/non-free categories first in category order, then free-space squares up to `TOTAL_SQUARES - reservedSquareCount`, then reserved buffer squares (autocompact `⚡` or manual-compact) at the end.
- Deferred-category collapse for Leo v1: Leo does not ship tool-search (no `isDeferred: true` sources in-tree today), so deferred categories are permitted to be absent from `ContextData` entirely; the renderer SHALL NOT render an empty deferred row and SHALL NOT throw when the category is missing. The ordering + visual contract is still specified here so that adding tool-search later is a pure-data change (resolves [context.md Open Q §FR-CTX-03](../../context.md#open-questions)).
- Vitest coverage per [NFR-TEST-08](../../context.md#nfr-test-08): category-ordering snapshot across the eleven positions, grid-dimension selector table (four combinations), square-allocation fixtures covering `max(1, round(...))` floor clamp and free-space-rounds-to-zero edge, partial-square fullness boundary cases at 0, 0.3, 0.7, 0.999, 1.0, reserved-square append order, deferred-category grid-exclusion assertion, and command-registration fixture asserting one slash-command + one palette command converging on the same handler.

### Out of scope

- `analyzeContextUsage`, the post-compact-boundary filter, the microcompact pass, the seven counting operations, and the `ContextData` shape — all owned by [F46](../context-analyzer-pipeline/feature.md).
- Per-category counting rules, memory-file listing, MCP tool enumeration — consumed through `ContextData` fields supplied by F46 and the counting helpers it owns.
- Suggestion generation and rendering (near-capacity, large-tool-results, memory-bloat, etc.) — carried by F48 per [context.md §12](../../../../srs/context.md#12-context-suggestions).
- Status-line integration and token-warning state — carried by F48 per [context.md §13–§14](../../../../srs/context.md#13-token-warnings) and covers [FR-CTX-06](../../context.md#fr-ctx-06).
- Non-interactive markdown output per [context.md §11](../../../../srs/context.md#11-non-interactive-output-markdown) — Leo is Obsidian-only (no REPL); the non-interactive variant is dropped in v1.
- Context-collapse / `projectView` integration — pipeline slot owned by [F46](../context-analyzer-pipeline/feature.md); slot is identity in v1.
- Autocompact buffer *size calculation* and feature-flag gating — owned by [F43](../compaction-autocompact/feature.md); this feature only renders the category and its reserved-square block.
- Visual wireframes, ARIA, keyboard navigation, focus management, color-token mapping, reduced-motion — delivered by the `ui.md` companion.

## Acceptance criteria

1. `/context` typed in the [F06 composer](../chat-composer-input/feature.md) and `Leo: Show context` selected from the command palette BOTH route to a single handler that awaits [F46](../context-analyzer-pipeline/feature.md)'s `analyzeContextUsage(...)` and mounts the interactive grid+breakdown view; a Vitest fixture spies on the handler and asserts identical invocation payloads from both paths. ([FR-CTX-01](../../context.md#fr-ctx-01), [context.md §2](../../../../srs/context.md#2-command-registration))
2. The category-breakdown renderer emits exactly the eleven-row ordering from [context.md §8](../../../../srs/context.md#8-category-definitions--ordering) when all sources are present: System prompt, System tools, MCP tools, MCP tools (deferred), System tools (deferred), Custom agents, Memory files, Skills, Messages, Autocompact/Manual compact buffer, Free space; a Vitest snapshot pins the order. ([FR-CTX-03](../../context.md#fr-ctx-03), [NFR-TEST-08](../../context.md#nfr-test-08))
3. Conditional categories absent from `ContextData` (e.g. no MCP tools loaded, no Custom agents, no Memory files, no Skills, empty Messages) are skipped in-place without shifting the relative order of the remaining rows; a Vitest fixture covers the "only System prompt + Messages + Free space" minimal case and asserts the three rows appear in category order. ([FR-CTX-03](../../context.md#fr-ctx-03))
4. Deferred categories (`isDeferred: true` on the `ContextCategory`) are rendered in the breakdown list with the dimmed "inactive" treatment AND contribute zero squares to the grid AND are excluded from the usage-percentage denominator; a Vitest fixture injects a deferred entry and asserts grid-square-count unchanged vs. the same fixture with the deferred entry removed. ([FR-CTX-03](../../context.md#fr-ctx-03))
5. Grid dimensions resolve to 5×5 / 10×10 / 5×10 / 20×10 across the four `{contextWindow, panelWidth}` quadrants per [context.md §9.1](../../../../srs/context.md#91-grid-dimensions); a Vitest parametrized table drives all four combinations and asserts both rows×cols and total-squares (`{25, 100, 50, 200}`). ([FR-CTX-04](../../context.md#fr-ctx-04), [NFR-TEST-08](../../context.md#nfr-test-08))
6. Square allocation applies `allocatedSquares = max(1, round(exactSquares))` for every non-free, non-deferred category and `round(exactSquares)` for Free space per [context.md §9.2](../../../../srs/context.md#92-square-allocation); Vitest fixtures cover (a) a sub-1-square category getting bumped to 1, (b) a 0-share Free space getting 0 squares, (c) a fractional-heavy 3.6-square allocation rounding to 4. ([FR-CTX-04](../../context.md#fr-ctx-04), [NFR-TEST-08](../../context.md#nfr-test-08))
7. Partial-square fullness follows [context.md §9.3](../../../../srs/context.md#93-partial-square-fullness) exactly: square `i < floor(exactSquares)` returns fullness `1.0`; square `i == floor(exactSquares)` returns `exactSquares - floor(exactSquares)`; boundary values `0, 0.3, 0.7, 0.999, 1.0` are all covered in a Vitest table and the ≥0.7 → `◉` / <0.7 → `◐` gate from [§9.5](../../../../srs/context.md#95-visual-symbols) is asserted against each. ([FR-CTX-04](../../context.md#fr-ctx-04), [NFR-TEST-08](../../context.md#nfr-test-08))
8. Square rendering order is non-reserved/non-free categories in category order → Free-space squares (filling up to `TOTAL_SQUARES - reservedSquareCount`) → reserved buffer squares at the end, per [context.md §9.4](../../../../srs/context.md#94-square-rendering-order); a Vitest fixture with an autocompact buffer asserts the last `reservedSquareCount` squares carry the reserved category and Free space sits immediately before them. ([FR-CTX-04](../../context.md#fr-ctx-04))
9. Panel-width changes from the [F04](../chat-sidebar-view/feature.md) `ResizeObserver` cause the grid dimensions to be re-selected on the next render; a Vitest fixture simulates a ≥80ch → <80ch resize and asserts the grid switches from 10×10 to 5×5 (or 20×10 → 5×10 at ≥1M) without remounting the breakdown list or re-calling F46. ([FR-CTX-04](../../context.md#fr-ctx-04))
10. A `/context` invocation that rejects (F46 failure or abort) shows the F13 error surface and does not render a partial grid; the turn-scoped `AbortController` is disposed on unmount and no post-abort writes happen. Verified by a Vitest fixture injecting a rejected `analyzeContextUsage`. ([FR-CTX-01](../../context.md#fr-ctx-01), [Architecture §10](../../../../architecture/architecture.md#10-concurrency--lifecycle-rules))

## Open questions

- **Panel-width chars-per-line heuristic (FR-CTX-04)**: [context.md §9.1](../../../../srs/context.md#91-grid-dimensions) specifies thresholds in "columns" (terminal cells). Leo is a CSS pane. Proposing one-time offscreen-measure of the rendered monospace glyph advance at mount, then `clientWidth / advance` ≷ 80 to pick the breakpoint, refreshed on `ResizeObserver`. Verifier to confirm vs. a fixed px threshold (e.g. `<560px → <80ch`).
- **Deferred categories without tool-search (FR-CTX-03)**: Leo v1 ships no tool-search, so "MCP tools (deferred)" and "System tools (deferred)" have no source and will be absent from `ContextData`. Proposing absent-is-legal (renderer no-ops on missing rows) so future tool-search work is a pure-data change. Alternative: always render the two deferred rows with "0 tokens" and the dimmed treatment to keep the eleven-row layout stable; flagging for verifier.
- **Slash-command dispatch plumbing (FR-CTX-01)**: [F06](../chat-composer-input/feature.md) does not yet own a named "slash command registry" contract; `/context` could either be handled by F06's existing submit path (detect `^/context$` at submit time and route instead of sending) or by a new SlashCommandRegistry. Proposing the former for v1 (single command, no aliases, no arguments per [context.md §2](../../../../srs/context.md#2-command-registration)); registry can arrive if more slash commands land.
- **Free-space rounding edge cases**: [context.md §9.2](../../../../srs/context.md#92-square-allocation) permits Free space to round to 0 squares while §9.4 wants Free space placed between categories and reserved. When Free space is 0, is the "fill up to `TOTAL_SQUARES - reservedSquareCount`" clause still a no-op, or does the last non-reserved category steal the remainder? Proposing no-op (Free space simply contributes 0 squares, categories' rounded allocations fill the remainder); verifier to pin.
- **Reserved-category source selection (§8 Reserved Buffer Rules)**: the renderer needs to know which of "Autocompact buffer" vs "Manual compact buffer" to show. Proposing reading `ContextData.reservedBuffer: { kind: 'autocompact' | 'manual' | 'none', tokens: number }` supplied by F46, not computing it here; needs F46 shape confirmation.
- **Palette-command visibility (FR-CTX-01)**: should `Leo: Show context` be registered globally (visible anywhere in Obsidian) or gated on the ChatView being present? Proposing global with a silent no-op + Notice ("Open Leo chat first") when the view is not mounted, matching [F04 chat-sidebar-view](../chat-sidebar-view/feature.md)'s ribbon-open pattern; alternative is `checkCallback` that hides the entry entirely when the view is closed.
