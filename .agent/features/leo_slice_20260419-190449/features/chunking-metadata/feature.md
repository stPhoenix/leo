# F28 — Chunking + metadata extraction

## Purpose

Deliver the pure-function chunking + metadata-extraction tier the [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md) `processPath` seam calls per markdown path: parse the file into heading-scoped chunks using the Obsidian `metadataCache.getFileCache(file).headings` outline as the default segmentation axis per [FR-IDX-06](../../context.md#fr-idx-06), fall back to a fixed-size sliding-window splitter at ~512 tokens with overlap when no usable headings exist (or when a heading section exceeds the window) per [FR-IDX-06](../../context.md#fr-idx-06), and decorate every emitted chunk with the canonical `{path, line_start, line_end, heading_path, frontmatter_tags, inline_tags}` shape per [FR-IDX-07](../../context.md#fr-idx-07) — `heading_path` reconstructed as the ancestor-chain array from the CM6/Obsidian heading outline, `frontmatter_tags` drained from `metadataCache.getFileCache(file).frontmatter.tags` (string or array, normalized), `inline_tags` from `metadataCache.getFileCache(file).tags[*].tag` with the leading `#` stripped. The tier is pure: it takes `{path, source, metadataCache}` and returns `Chunk[]` with zero IO and zero global state, so it can be Vitest-unit-covered end-to-end per [NFR-TEST-01](../../context.md#nfr-test-01). Embedding + IndexedDB persistence ([F29](../../features-index.md)), dirty-queue orchestration ([F27](../vault-indexer-dirty-queue/feature.md)), graph cache ([F34](../../features-index.md)), canvas JSON parsing ([F36](../../features-index.md)), and exclude-list filtering ([F32](../../features-index.md)) are all explicitly downstream or adjacent.

## Scope

### In scope

- `Chunker` pure module (no IO, no Obsidian singletons bound — all host input passed as arguments) exposing `chunk(input: ChunkerInput): Chunk[]` per [architecture §3.3 Domain / Core](../../../../architecture/architecture.md#33-domain--core-pure) and [architecture §8 Extension Points — Chunker interface](../../../../architecture/architecture.md#8-extension-points); `ChunkerInput = { path: string; source: string; fileCache: CachedMetadata }` mirrors the [Obsidian metadataCache shape](../../../../standards/tech-stack.md#platform-apis) so F27 can wire it directly behind its `processPath` seam.
- Heading-based default chunking per [FR-IDX-06](../../context.md#fr-idx-06): walk `fileCache.headings` (1-based `position.start.line` / `position.end.line` from Obsidian) and emit one `Chunk` per heading section (from the heading line through the line before the next same-or-shallower heading, or through end-of-file). Sections under `H1 > H2 > H3` inherit the full ancestor chain via `heading_path: string[]`. When a file has zero headings, the whole body is a single pseudo-section that flows into the fixed-size fallback.
- Fixed-size overlapping fallback per [FR-IDX-06](../../context.md#fr-idx-06): when a heading-section body exceeds `~512` tokens (pure `Math.ceil(len/4)` estimator reused from [FR-COMPACT-02](../../context.md#fr-compact-02) Tier-3 to keep this module provider-independent), slide a 512-token window with 64-token overlap (constants `CHUNK_TARGET_TOKENS=512`, `CHUNK_OVERLAP_TOKENS=64` `as const`) across the section's lines; windows snap to whole-line boundaries so `line_start` / `line_end` remain integers (never mid-line splits). Same fallback applies to the zero-heading whole-body path.
- Canonical `Chunk` shape per [FR-IDX-07](../../context.md#fr-idx-07): `type Chunk = { path: string; line_start: number; line_end: number; heading_path: string[]; frontmatter_tags: string[]; inline_tags: string[]; text: string }`; all arrays are `readonly` per [code style — TypeScript](../../../../standards/code-style.md#typescript), `text` carries the exact substring for the line range (used downstream by F29 for embedding), `path` is vault-relative.
- Frontmatter-tag normalization: read `fileCache.frontmatter.tags` when present, accept both `tags: foo` (single string) and `tags: [foo, bar]` (array), trim whitespace, strip leading `#` per [Obsidian convention](../../../../standards/tech-stack.md#platform-apis), drop empties, dedupe preserving first-seen order; identical normalizer applied to the legacy `tag:` key as a secondary fallback. All chunks of the same file carry the same `frontmatter_tags` snapshot.
- Inline-tag extraction: map `fileCache.tags` (Obsidian's `TagCache[]` with `{tag, position}`) and keep only tags whose `position.start.line` / `end.line` fall inside `[line_start, line_end]`, strip the leading `#`, dedupe preserving first-seen order. Tags from frontmatter lines are never counted as inline.
- Heading-path reconstruction: maintain an ancestor stack while walking `fileCache.headings`; on each heading, pop stack entries with `level >= current.level` before pushing, yielding the inclusive ancestor chain (`heading.heading` strings, in document order) on every section emit; the fallback sub-windows inherit the parent section's full `heading_path` verbatim.
- Deterministic output ordering: chunks emitted in document order (top-to-bottom); two runs on the same `(path, source, fileCache)` triple return byte-identical `Chunk[]` — Vitest snapshot covers this invariant per [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw).
- Seam for [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md): `Chunker.chunk` is invoked inside F27's `processPath(path)` callback; this feature returns `Chunk[]` and exits — it never calls the vault adapter, the embedding client, the graph cache, or IndexedDB itself, honoring the [architecture §3.3 core-pure rule](../../../../architecture/architecture.md#33-domain--core-pure).
- Vitest unit coverage per [NFR-TEST-01](../../context.md#nfr-test-01) and [code style — Testing](../../../../standards/code-style.md#testing-vitest--msw): heading-based segmentation with H1/H2/H3 ancestors; heading-path inheritance; zero-heading file → single-body → fixed-size fallback; oversized H1 section forced into fallback; exact `line_start` / `line_end` boundaries; overlap size (64) preserved between adjacent windows; frontmatter `tags:` as string vs array vs `tag:` legacy vs missing; inline tag scoping (frontmatter tags excluded, cross-section tags excluded); `#` stripping + trim + dedupe on both tag sources; deterministic snapshot on the same input; empty-file edge case emits `[]`.

### Out of scope

- Embedding the emitted chunks through `EmbeddingClient` and upserting vectors + chunk metadata + `IndexHeader` into IndexedDB per [FR-IDX-08](../../context.md#fr-idx-08), [FR-IDX-09](../../context.md#fr-idx-09) → ships with [F29 embeddings-indexeddb-store](../../features-index.md); this feature ends at the `Chunk[]` return value.
- `DirtyQueue` orchestration, `vault.on('create'|'modify'|'delete'|'rename')` listener fan-out, header-mismatch `Notice`, and `<vault>/.leo/index/queue.json` persistence per [FR-IDX-01](../../context.md#fr-idx-01)–[FR-IDX-05](../../context.md#fr-idx-05), [FR-IDX-12](../../context.md#fr-idx-12), [NFR-PERF-04](../../context.md#nfr-perf-04) → ships with [F27 vault-indexer-dirty-queue](../vault-indexer-dirty-queue/feature.md); this feature is a pure callee behind F27's `processPath` seam.
- Symmetric graph cache + incremental graph updates built from `metadataCache.resolvedLinks` + `metadataCache.on('resolved')` per [FR-IDX-10](../../context.md#fr-idx-10), [FR-IDX-11](../../context.md#fr-idx-11) → ships with [F34 graph-cache-symmetric](../../features-index.md); this feature never touches `resolvedLinks`.
- `.canvas` JSON node-text parsing per [architecture §3.4](../../../../architecture/architecture.md#34-adapters) → ships with [F36 canvas-file-indexing](../../features-index.md); canvas inputs are rejected upstream by F27's markdown-only filter and never reach the chunker.

## Acceptance criteria

1. `Chunker.chunk({path, source, fileCache})` is a pure function — zero IO, zero Obsidian-singleton access, same input yields byte-identical `Chunk[]` (Vitest snapshot asserts determinism). ([FR-IDX-06](../../context.md#fr-idx-06), [NFR-TEST-01](../../context.md#nfr-test-01))
2. When `fileCache.headings` is non-empty, the chunker emits one `Chunk` per heading section whose `line_start` / `line_end` cover from the heading line through the line before the next same-or-shallower heading (or end-of-file), and every chunk's `heading_path: string[]` is the ancestor chain from H1 down to the section's own heading in document order. ([FR-IDX-06](../../context.md#fr-idx-06))
3. When a heading section (or a zero-heading whole-file body) exceeds `CHUNK_TARGET_TOKENS=512` tokens estimated via `Math.ceil(len/4)`, the chunker falls back to a sliding-window splitter with `CHUNK_OVERLAP_TOKENS=64`, snapping to whole-line boundaries so `line_start` / `line_end` remain integers; adjacent windows share exactly 64 tokens of overlap. ([FR-IDX-06](../../context.md#fr-idx-06))
4. Every emitted `Chunk` conforms to `{path: string, line_start: number, line_end: number, heading_path: readonly string[], frontmatter_tags: readonly string[], inline_tags: readonly string[], text: string}`; `path` is vault-relative; `text` is the exact substring of the line range from `source`. ([FR-IDX-07](../../context.md#fr-idx-07))
5. `frontmatter_tags` are normalized from `fileCache.frontmatter.tags` (accepting both string and `string[]` shapes, plus the legacy `tag:` key as secondary fallback) with leading `#` stripped, trimmed, empties dropped, deduped first-seen order; every chunk from the same file carries the same snapshot. ([FR-IDX-07](../../context.md#fr-idx-07))
6. `inline_tags` are scoped from `fileCache.tags` to the chunk's `[line_start, line_end]` range only (frontmatter-line tags excluded), with `#` stripped, trimmed, deduped first-seen order. ([FR-IDX-07](../../context.md#fr-idx-07))
7. Vitest suite covers heading-based segmentation (H1/H2/H3 ancestry), zero-heading → fallback, oversized heading forced into fallback, overlap size preservation, frontmatter `tags:` as string vs array vs legacy `tag:` vs missing, inline-tag range scoping + `#` stripping + dedupe, deterministic snapshot, and empty-file `[]` edge case — executable under `vitest run` with no vault or network fixture. ([NFR-TEST-01](../../context.md#nfr-test-01))

## Open questions

- **`CHUNK_OVERLAP_TOKENS` exact value** — 64 tokens balances context continuity against index bloat; SRS does not pin it. Verifier to confirm or adjust in a constants-review pass.
- **Token estimator cohesion with F29 embed tokenizer** — this module uses the [FR-COMPACT-02](../../context.md#fr-compact-02) Tier-3 `len/4` estimator for windowing; F29 may later use the provider's true tokenizer to verify payload size before embed. Proposal: keep the estimator here and let F29 reject oversized chunks on its side; verifier to confirm split.
- **Heading-path string casing / trimming** — raw `heading.heading` strings can carry trailing whitespace or embedded markdown (`*bold*`). Proposal: trim whitespace only, preserve markdown; verifier to confirm policy.
