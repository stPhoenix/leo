### Notes on coverage edges

- **FR-UI-07** (empty / onboarding states) is primarily owned by **F03** (first-launch "Configure LM Studio" CTA + setup wizard); the no-index "Index vault" CTA ships alongside **F30** (indexer UI controls) and empty-thread example prompts ship in **F04** (ChatView shell). The canonical row is F03.
- **Non-functional IDs** are attached to the feature that most naturally owns the invariant or test per rule 10.
- **F19** intentionally has no direct FR in `covers` because its requirements (FR-AGENT-04 / FR-AGENT-06 for write variants) are carried by F16's registry definition; F19 delivers the concrete `create_note` / `append_to_note` implementations and is still a separate slice for implementability.
- **F36, F37, F40, F49, F50** expose no net-new requirement IDs beyond those already covered upstream but are preserved as independent slices because they each deliver observable behavior tied to a distinct SRS phase deliverable (canvas indexing, multi-thread CRUD, user tools, attachments, scale polish).
